## Урок 1.1. Что такое Linux, дистрибутивы, установка (или WSL для Windows)
**Время: 35 минут**

### Цель урока
Понять, что такое Linux, разобраться в понятии дистрибутива и настроить окружение для практики.

### 1. Что такое Linux

Linux — это ядро операционной системы, изначально созданное Линусом Торвальдсом в 1991 году и распространяемое с открытым исходным кодом. В отличие от Windows или macOS, Linux — не единая операционная система от одной компании, а основа, на которой множество разных команд и организаций собирают собственные полноценные ОС.

Linux лежит в основе подавляющего большинства серверов, облачной инфраструктуры, суперкомпьютеров и встраиваемых устройств в мире — понимание Linux является базовым навыком для разработчиков, DevOps-инженеров, системных администраторов и специалистов по кибербезопасности.

### 2. Что такое дистрибутив

Дистрибутив — это полноценная операционная система, собранная на основе ядра Linux с добавлением набора программ, системы установки пакетов и (обычно) графического интерфейса. Самые распространённые дистрибутивы:

- **Ubuntu** — один из самых популярных дистрибутивов, дружелюбный к новичкам, основан на Debian.
- **Debian** — стабильный, консервативный дистрибутив, на котором основано множество других.
- **CentOS / Rocky Linux / AlmaLinux** — распространены на корпоративных серверах, совместимы с Red Hat Enterprise Linux.
- **Fedora** — дистрибутив с более новыми версиями программ, спонсируется Red Hat.
- **Alpine Linux** — минималистичный дистрибутив, часто используется как база для Docker-контейнеров.

### 3. Ядро vs дистрибутив: аналогия

Можно представить ядро Linux как «двигатель», а дистрибутив — как полноценный «автомобиль», построенный вокруг этого двигателя разными производителями со своими особенностями, инструментами и внешним видом.

### 4. Способы получить доступ к Linux для обучения

- **WSL (Windows Subsystem for Linux)** — самый удобный способ для пользователей Windows: позволяет запускать полноценный Linux прямо внутри Windows без установки отдельной системы.
- **Виртуальная машина** (VirtualBox, VMware) — установка Linux в изолированном виртуальном окружении поверх текущей ОС.
- **macOS/Linux уже содержат Unix-подобный терминал**, во многом схожий с Linux для базовых команд.
- **Облачный сервер** — аренда небольшого виртуального сервера для удалённой практики.

### 5. Установка WSL на Windows

Команда, выполняемая в PowerShell от имени администратора:

```bash
wsl --install
```

Эта команда установит WSL и дистрибутив Ubuntu по умолчанию. После установки и перезагрузки Ubuntu запускается как обычное приложение из меню «Пуск».

### 6. Первая проверка после установки

```bash
lsb_release -a     # информация об установленном дистрибутиве
uname -r              # версия ядра Linux
whoami                # имя текущего пользователя
```

### Резюме урока

- Linux — это ядро операционной системы, на основе которого создаются различные полноценные дистрибутивы (Ubuntu, Debian, CentOS и др.).
- Дистрибутив — это ядро Linux вместе с набором программ и системой управления пакетами.
- Для практики на Windows самый удобный способ — установка WSL, позволяющая запускать Linux прямо внутри Windows.

## Урок 1.2. Терминал и структура файловой системы Linux
**Время: 35 минут**

### Цель урока
Познакомиться с терминалом и понять принципы организации файловой системы Linux.

### 1. Что такое терминал

Терминал (командная строка, shell) — текстовый интерфейс для взаимодействия с операционной системой через ввод команд. В Linux самой распространённой командной оболочкой является **bash** (Bourne Again Shell), хотя существуют и альтернативы (zsh, fish).

### 2. Приглашение командной строки (prompt)

Типичное приглашение в терминале Linux выглядит так:

```
user@hostname:~$
```

Где `user` — имя пользователя, `hostname` — имя компьютера, `~` — текущая папка (символ `~` обозначает домашнюю директорию пользователя), `$` — символ обычного пользователя (у администратора root обычно используется `#`).

### 3. Единое дерево файловой системы

В отличие от Windows, где у каждого диска есть отдельная буква (C:, D:), в Linux вся файловая система представляет собой единое дерево, начинающееся с корневой папки, обозначаемой символом `/`.

```
/
├── home/       # домашние папки пользователей
├── etc/         # конфигурационные файлы всей системы
├── var/         # изменяемые данные: логи, кэш, временные файлы служб
├── usr/          # установленные программы, библиотеки, документация
├── bin/           # основные системные исполняемые команды
├── tmp/           # временные файлы, очищаемые при перезагрузке
└── root/          # домашняя папка администратора (root)
```

### 4. Домашняя папка пользователя

Каждый пользователь Linux имеет собственную домашнюю папку, обычно расположенную по пути `/home/имя_пользователя`. Символ `~` в терминале — это сокращение для пути к домашней папке текущего пользователя.

### 5. Абсолютные и относительные пути

- **Абсолютный путь** — полный путь от корня файловой системы, например `/home/alina/projects/notes.txt`.
- **Относительный путь** — путь относительно текущей папки, например `projects/notes.txt`, если пользователь уже находится в `/home/alina`.

### 6. Скрытые файлы и папки

Файлы и папки, имя которых начинается с точки (например, `.bashrc`, `.ssh`), считаются скрытыми и не отображаются в обычном списке содержимого папки — такие файлы часто используются для хранения конфигурации программ и системы.

### Резюме урока

- Терминал — текстовый интерфейс взаимодействия с Linux, самая распространённая командная оболочка — bash.
- Файловая система Linux представляет собой единое дерево, начинающееся с корня `/`, в отличие от системы дисков с буквами в Windows.
- Символ `~` обозначает домашнюю папку текущего пользователя, скрытые файлы и папки в Linux начинаются с точки.

## Урок 1.3. Навигация: pwd, cd, ls, mkdir, rmdir
**Время: 30 минут**

### Цель урока
Освоить базовые команды навигации по файловой системе Linux.

### 1. Команда pwd — где я нахожусь

`pwd` (print working directory) выводит полный путь к текущей папке:

```bash
pwd
# /home/alina/projects
```

### 2. Команда cd — переход между папками

```bash
cd projects              # перейти в папку projects (относительный путь)
cd /home/alina/projects   # перейти по абсолютному пути
cd ..                        # перейти на уровень выше
cd ../..                      # перейти на два уровня выше
cd ~                          # перейти в домашнюю папку
cd -                          # вернуться в предыдущую папку, где был до этого
cd                             # без аргументов - тоже переход в домашнюю папку
```

### 3. Команда ls — просмотр содержимого папки

```bash
ls                # список файлов и папок в текущей директории
ls -l               # подробный список: права доступа, владелец, размер, дата изменения
ls -a               # включая скрытые файлы (начинающиеся с точки)
ls -la               # комбинация -l и -a
ls -lh               # -l с "человекочитаемым" форматом размера файлов (КБ, МБ, ГБ)
ls /home            # список содержимого конкретной указанной папки
```

### 4. Команда mkdir — создание папок

```bash
mkdir new_folder                      # создать одну папку
mkdir folder1 folder2 folder3          # создать сразу несколько папок
mkdir -p projects/app/src/utils   # создать вложенные папки сразу, даже если промежуточные не существуют
```

Без флага `-p` попытка создать вложенную папку внутри ещё не существующей родительской папки завершится ошибкой.

### 5. Команда rmdir — удаление пустых папок

```bash
rmdir empty_folder   # удаляет только ПУСТУЮ папку
```

Если папка содержит файлы, `rmdir` завершится ошибкой — для удаления непустых папок используется команда `rm -r` (Урок 2.1).

### 6. Автодополнение через Tab

Одна из самых полезных возможностей терминала — автодополнение имён файлов и папок при нажатии клавиши `Tab`. Это не только ускоряет ввод команд, но и снижает число опечаток — рекомендуется вырабатывать привычку постоянно использовать `Tab` при наборе путей.

### 7. История команд

Стрелки «вверх»/«вниз» в терминале позволяют быстро пролистывать ранее введённые команды текущей и предыдущих сессий, а команда `history` выводит список последних выполненных команд.

### Резюме урока

- `pwd` показывает текущую папку, `cd` — перемещает между папками, `ls` — показывает содержимое папки.
- `mkdir -p` создаёт вложенные папки сразу, `rmdir` удаляет только пустые папки.
- Автодополнение через `Tab` и история команд (стрелки, `history`) значительно ускоряют работу в терминале.

# Модуль 2. Работа с файлами и правами доступа

## Урок 2.1. Работа с файлами: touch, cp, mv, rm, cat, less
**Время: 40 минут**

### Цель урока
Научиться создавать, копировать, перемещать, удалять и просматривать файлы через терминал.

### 1. Создание пустого файла: touch

```bash
touch notes.txt          # создаёт пустой файл (или обновляет время изменения существующего)
touch file1.txt file2.txt   # можно создать сразу несколько файлов
```

### 2. Создание файла с содержимым: echo и перенаправление

```bash
echo "Привет, Linux!" > notes.txt    # создаёт файл с указанным содержимым (перезаписывает, если существует)
echo "Ещё строка" >> notes.txt         # добавляет строку в конец файла (не перезаписывает)
```

Оператор `>` перенаправляет вывод команды в файл, полностью его перезаписывая; `>>` — добавляет в конец, сохраняя существующее содержимое.

### 3. Просмотр содержимого файла: cat, less

```bash
cat notes.txt      # выводит всё содержимое файла целиком
less notes.txt        # постраничный просмотр (удобно для больших файлов)
```

Внутри `less`: `Пробел` — следующая страница, `b` — предыдущая страница, `/слово` — поиск по тексту, `q` — выход.

### 4. Просмотр части файла: head, tail

```bash
head notes.txt          # первые 10 строк файла
head -n 5 notes.txt      # первые 5 строк
tail notes.txt             # последние 10 строк
tail -f app.log              # непрерывный просмотр добавляемых строк в реальном времени (удобно для логов)
```

### 5. Копирование файлов и папок: cp

```bash
cp notes.txt notes_backup.txt        # копирование файла с новым именем
cp notes.txt documents/                 # копирование файла в другую папку с тем же именем
cp -r project/ project_backup/       # копирование папки целиком (флаг -r - рекурсивно)
```

### 6. Перемещение и переименование: mv

```bash
mv notes.txt documents/          # переместить файл в другую папку
mv old_name.txt new_name.txt     # переименовать файл (перемещение "на месте" с новым именем)
```

### 7. Удаление файлов и папок: rm

```bash
rm notes.txt              # удалить файл
rm -r old_project/          # удалить папку рекурсивно, вместе со всем содержимым
rm -rf old_project/          # удалить без запроса подтверждения
```

> **Важное предупреждение:** в Linux нет «корзины» для команды `rm` — удалённые файлы восстановить стандартными средствами невозможно. Флаг `-rf` особенно опасен: он удаляет рекурсивно и без единого подтверждения. Перед выполнением команд с `rm -rf` всегда стоит перепроверить путь.

### Резюме урока

- `touch` создаёт пустой файл, `echo "текст" > файл` — файл с содержимым (`>` перезаписывает, `>>` добавляет).
- `cat`/`less` показывают содержимое файла целиком или постранично, `head`/`tail` — начало или конец файла.
- `cp`, `mv`, `rm` — копирование, перемещение и удаление; `rm` удаляет файлы безвозвратно, без возможности восстановления.

## Урок 2.2. Редакторы nano и vim: базовые операции
**Время: 40 минут**

### Цель урока
Освоить базовые операции в двух самых распространённых текстовых редакторах терминала Linux.

### 1. Зачем нужны текстовые редакторы в терминале

Часто требуется отредактировать файл прямо на сервере, где нет графического интерфейса, — для этого используются текстовые редакторы, работающие непосредственно в терминале. Два самых распространённых — nano (простой) и vim (мощный, но сложный для новичков).

### 2. Редактор nano — для начала

```bash
nano notes.txt
```

Nano открывает файл для редактирования с подсказками по нижнему краю экрана (символ `^` означает клавишу `Ctrl`):

- `Ctrl+O` — сохранить файл (Write Out)
- `Ctrl+X` — выйти из редактора
- `Ctrl+K` — вырезать строку
- `Ctrl+U` — вставить вырезанное
- `Ctrl+W` — найти текст в файле

Nano — интуитивно понятный редактор, подходящий для новичков и для быстрых, несложных правок конфигурационных файлов.

### 3. Редактор vim — более мощный, но сложнее

Vim работает по принципу «режимов» — в отличие от nano, где можно сразу печатать текст, vim по умолчанию открывается в «нормальном режиме», в котором нажатия клавиш интерпретируются как команды, а не как ввод текста:

```bash
vim notes.txt
```

### 4. Основные режимы vim

- **Нормальный режим (Normal mode)** — режим по умолчанию, используется для перемещения по файлу и выполнения команд.
- **Режим вставки (Insert mode)** — режим для непосредственного набора текста, включается клавишей `i`.
- **Командный режим (Command mode)** — используется для сохранения, выхода и других операций, включается клавишей `:`.

### 5. Базовые команды vim

```
i           - войти в режим вставки (можно печатать текст)
Esc         - вернуться в нормальный режим из любого другого
:w          - сохранить файл (write)
:q          - выйти (quit), если нет несохранённых изменений
:wq         - сохранить и выйти
:q!         - выйти БЕЗ сохранения изменений
dd          - удалить текущую строку (в нормальном режиме)
/слово      - найти слово в файле
```

### 6. Частая шутка про vim

В сообществе разработчиков распространена шутка о том, что самая сложная задача с vim — это из него выйти, если случайно попал в него, не зная команд. Помни: `Esc`, затем `:q!` — всегда сработает для выхода без сохранения изменений.

### 7. Какой редактор выбрать

Для новичков, только начинающих работать с терминалом, рекомендуется начинать с **nano** — он значительно проще для базовых правок файлов. Владение базовыми командами **vim** тем не менее полезно, так как он часто установлен по умолчанию на серверах, где nano может отсутствовать.

### Резюме урока

- Nano — простой, интуитивно понятный редактор терминала, подходящий для новичков (`Ctrl+O` — сохранить, `Ctrl+X` — выйти).
- Vim работает через систему режимов (нормальный, вставки, командный) и требует отдельного изучения базовых команд.
- Для быстрого выхода из vim без сохранения: нажать `Esc`, затем ввести `:q!`.

## Урок 2.3. Права доступа и пользователи: chmod, chown, sudo
**Время: 35 минут**

### Цель урока
Понять систему прав доступа в Linux и научиться управлять ими.

### 1. Три типа прав доступа к файлу

- **r (read)** — право на чтение содержимого файла.
- **w (write)** — право на изменение файла.
- **x (execute)** — право на выполнение файла (актуально для программ и скриптов).

### 2. Три категории пользователей для каждого файла

- **owner (владелец)** — пользователь, создавший файл.
- **group (группа)** — группа пользователей, связанная с файлом.
- **others (остальные)** — все прочие пользователи системы.

### 3. Просмотр прав доступа: ls -l

```bash
ls -l
# -rwxr-xr-- 1 alina staff 220 Jul 12 10:00 script.sh
```

Первый символ обозначает тип файла (`-` — обычный файл, `d` — папка), следующие 9 символов делятся на три группы по три символа: права владельца (`rwx`), группы (`r-x`), остальных (`r--`).

### 4. Изменение прав доступа: chmod

Символьный способ:

```bash
chmod +x script.sh          # добавить право на выполнение для всех категорий
chmod u+w file.txt            # добавить право записи только владельцу (u - user)
chmod g-w file.txt             # убрать право записи у группы (g - group)
```

Числовой способ (более распространённый): каждому праву соответствует число — r=4, w=2, x=1, их сумма определяет набор прав для каждой из трёх категорий:

```bash
chmod 755 script.sh    # владелец: 7 (rwx), группа: 5 (r-x), остальные: 5 (r-x)
chmod 644 notes.txt     # владелец: 6 (rw-), группа: 4 (r--), остальные: 4 (r--)
```

### 5. Изменение владельца: chown

```bash
chown alina file.txt              # изменить владельца файла на пользователя alina
chown alina:developers file.txt   # изменить и владельца, и группу одновременно
```

### 6. Команда sudo

Многие операции (установка программ, изменение системных файлов, управление другими пользователями) требуют прав суперпользователя (root). Вместо постоянной работы под учётной записью root (что небезопасно), используется команда `sudo`, временно повышающая права только для одной конкретной команды:

```bash
sudo apt update                  # обновление списка пакетов с правами администратора
sudo chown root file.txt           # смена владельца файла на root
```

После ввода `sudo` система обычно запрашивает пароль текущего пользователя для подтверждения.

### 7. Просмотр текущего пользователя и его группы

```bash
whoami        # имя текущего пользователя
groups         # список групп, в которые входит текущий пользователь
id               # подробная информация: UID, GID, группы
```

### Резюме урока

- Права доступа делятся на чтение (r), запись (w), выполнение (x) для трёх категорий: владелец, группа, остальные.
- `chmod` изменяет права доступа (символьным или числовым способом), `chown` — владельца и группу файла.
- `sudo` временно выполняет одну команду с правами администратора, что безопаснее постоянной работы под root.

## Урок 3.1. Управление процессами: ps, top, kill
**Время: 35 минут**

### Цель урока
Научиться просматривать запущенные процессы и управлять ими.

### 1. Что такое процесс

Процесс — это выполняющаяся в системе программа. Каждый процесс имеет уникальный числовой идентификатор — PID (Process ID), используемый для управления им.

### 2. Просмотр процессов: ps

```bash
ps                  # процессы, запущенные в текущем терминале
ps aux              # ВСЕ процессы в системе с подробной информацией
ps aux | grep nginx   # поиск конкретного процесса по имени через комбинацию с grep
```

Столбцы вывода `ps aux` включают: пользователя-владельца процесса, PID, использование CPU и памяти, время запуска, саму команду.

### 3. Интерактивный просмотр: top

```bash
top
```

`top` показывает список процессов в реальном времени, обновляющийся с определённым интервалом, с сортировкой по использованию ресурсов. Полезные клавиши внутри `top`: `q` — выход, `k` — завершить процесс по PID, `M` — отсортировать по использованию памяти.

### 4. Завершение процесса: kill

```bash
kill 1234         # отправить процессу с PID 1234 сигнал завершения (SIGTERM - "мягкое" завершение)
kill -9 1234       # принудительное завершение (SIGKILL), если процесс не отвечает на обычный сигнал
```

Разница между `kill` и `kill -9`: обычный `kill` даёт процессу возможность корректно завершить свою работу (сохранить данные, закрыть соединения), тогда как `kill -9` завершает процесс немедленно, без каких-либо «прощальных» действий — используется как крайняя мера, если процесс завис.

### 5. Завершение процесса по имени: pkill, killall

```bash
pkill firefox         # завершить процесс(ы) по имени
killall chrome          # завершить все процессы с указанным именем
```

### 6. Фоновые процессы

```bash
long_running_script.sh &     # запуск команды в фоновом режиме (символ &)
jobs                            # список фоновых задач текущей сессии терминала
fg %1                            # вернуть задачу номер 1 на передний план
```

### 7. Практический пример: поиск и завершение зависшего процесса

```bash
ps aux | grep python     # найти все процессы, связанные с python
kill 5678                    # завершить конкретный найденный процесс по его PID
```

### Резюме урока

- Каждый процесс имеет уникальный PID, используемый для управления через команды вроде `kill`.
- `ps aux` показывает статичный список всех процессов, `top` — обновляющийся в реальном времени.
- `kill -9` принудительно завершает зависший процесс, не давая ему возможности корректно закончить работу — используется как крайняя мера.

## Урок 3.2. Менеджеры пакетов: apt, yum (обзор)
**Время: 35 минут**

### Цель урока
Понять принцип работы менеджеров пакетов Linux и научиться устанавливать программы.

### 1. Что такое менеджер пакетов

Менеджер пакетов — инструмент для установки, обновления и удаления программ в Linux, автоматически разрешающий зависимости между пакетами (то есть автоматически устанавливающий все дополнительные библиотеки, необходимые для работы нужной программы).

### 2. APT — менеджер пакетов для Debian/Ubuntu

```bash
sudo apt update                     # обновить локальный список доступных пакетов и их версий
sudo apt upgrade                     # обновить все установленные пакеты до последних версий
sudo apt install имя_пакета      # установить программу
sudo apt remove имя_пакета       # удалить программу (конфигурационные файлы остаются)
sudo apt purge имя_пакета        # удалить программу вместе со всеми конфигурационными файлами
```

Команду `sudo apt update` рекомендуется выполнять регулярно перед установкой новых программ — она не устанавливает и не обновляет ничего сама, а лишь обновляет информацию о доступных версиях пакетов.

### 3. YUM/DNF — менеджеры пакетов для CentOS/Fedora/RHEL

```bash
sudo yum install имя_пакета     # установка (более старые версии CentOS)
sudo dnf install имя_пакета      # установка (современный менеджер, преемник yum)
sudo dnf update                       # обновление всех пакетов
```

### 4. Поиск пакетов

```bash
apt search текст_запроса     # поиск пакета по названию/описанию (Debian/Ubuntu)
apt show имя_пакета            # подробная информация о конкретном пакете
```

### 5. Установка из других источников (обзорно)

Помимо стандартных пакетов менеджера, программы также можно устанавливать: скачивая и устанавливая `.deb`-файлы напрямую (для Debian/Ubuntu), через универсальные форматы вроде Snap или Flatpak, или собирая программу из исходного кода — эти способы выходят за рамки базового курса, но полезно знать об их существовании.

### 6. Проверка установленных пакетов

```bash
apt list --installed        # список всех установленных пакетов
dpkg -l | grep имя_пакета   # проверка, установлен ли конкретный пакет (Debian/Ubuntu)
```

### 7. Практический пример: установка Git

```bash
sudo apt update
sudo apt install git
git --version    # проверка успешности установки
```

### Резюме урока

- Менеджер пакетов автоматизирует установку, обновление и удаление программ, включая разрешение зависимостей между ними.
- `apt` используется в Debian/Ubuntu, `yum`/`dnf` — в CentOS/Fedora/RHEL — разные семейства дистрибутивов используют разные менеджеры пакетов.
- Перед установкой новых программ через `apt` рекомендуется сначала выполнить `sudo apt update` для обновления списка доступных версий.

## Урок 3.3. Основы bash-скриптов и переменных окружения
**Время: 45 минут**

### Цель урока
Научиться писать простые bash-скрипты для автоматизации повторяющихся задач.

### 1. Что такое bash-скрипт

Bash-скрипт — текстовый файл с последовательностью команд терминала, который можно выполнить одним запуском вместо ручного ввода каждой команды по отдельности. Это один из базовых инструментов автоматизации в Linux.

### 2. Первый скрипт

```bash
#!/bin/bash
echo "Привет из bash-скрипта!"
echo "Текущая папка: $(pwd)"
echo "Сегодняшняя дата: $(date)"
```

Первая строка `#!/bin/bash` (называемая «шебанг», shebang) указывает системе, каким именно интерпретатором нужно выполнять этот файл.

### 3. Запуск скрипта

```bash
chmod +x script.sh     # выдать файлу право на выполнение
./script.sh               # запустить скрипт (точка со слэшем означает "в текущей папке")
```

### 4. Переменные в bash

```bash
#!/bin/bash
name="Алина"
echo "Привет, $name!"

age=20
echo "Через год будет $((age + 1)) лет"
```

Важная особенность синтаксиса bash: нельзя ставить пробелы вокруг знака `=` при присваивании переменной — `name = "Алина"` (с пробелами) приведёт к ошибке.

### 5. Условия в bash

```bash
#!/bin/bash
count=5

if [ $count -gt 3 ]; then
    echo "count больше 3"
else
    echo "count не больше 3"
fi
```

Обрати внимание на пробелы внутри квадратных скобок `[ $count -gt 3 ]` — они обязательны в синтаксисе bash.

### 6. Циклы в bash

```bash
#!/bin/bash
for i in 1 2 3 4 5; do
    echo "Число: $i"
done

for file in *.txt; do
    echo "Найден файл: $file"
done
```

### 7. Переменные окружения

Переменные окружения — специальные переменные, доступные всем процессам операционной системы, часто используемые для хранения путей и конфигурации:

```bash
echo $HOME       # домашняя папка текущего пользователя
echo $PATH        # список папок, в которых система ищет исполняемые команды
echo $USER         # имя текущего пользователя

export API_KEY="secret_value"   # создание новой переменной окружения
echo $API_KEY
```

### 8. Практический пример: скрипт резервного копирования

```bash
#!/bin/bash
BACKUP_DIR="backups"
DATE=$(date +%Y-%m-%d)

mkdir -p $BACKUP_DIR
cp -r project/ "$BACKUP_DIR/project_$DATE"

echo "Резервная копия создана: $BACKUP_DIR/project_$DATE"
```

Этот скрипт автоматически создаёт папку с резервной копией проекта, добавляя текущую дату к её названию.

### Резюме урока

- Bash-скрипт объединяет последовательность команд терминала в единый исполняемый файл, начинающийся с шебанга `#!/bin/bash`.
- Синтаксис bash требует внимательности к пробелам: их нельзя ставить вокруг `=` при присваивании, но обязательно внутри `[ ]` в условиях.
- Переменные окружения (`$HOME`, `$PATH` и др.) доступны всем процессам системы и часто используются для хранения путей и конфигурации.
