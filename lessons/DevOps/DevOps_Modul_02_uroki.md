# Модуль 2. Linux и командная строка

## Урок 2.1. Установка и знакомство с Linux (дистрибутивы, терминал)
**Время: 40 минут**

### Цель урока
Понять, что такое Linux, познакомиться с основными дистрибутивами и настроить окружение для практики.

### 1. Почему DevOps начинается с Linux

Подавляющее большинство серверов, облачной инфраструктуры и контейнеров работает на базе Linux. Уверенная работа в терминале Linux — базовый навык, без которого невозможна дальнейшая работа с Docker, Kubernetes, облачными платформами и CI/CD системами.

### 2. Что такое дистрибутив Linux

Linux — это ядро операционной системы. На его основе создаются дистрибутивы — полноценные операционные системы с ядром Linux и набором дополнительного ПО. Самые распространённые дистрибутивы в серверной и DevOps-среде:

- **Ubuntu / Debian** — один из самых популярных дистрибутивов, дружелюбный для новичков, использует пакетный менеджер `apt`.
- **CentOS / RHEL (Red Hat)** — распространены в корпоративной среде, используют пакетный менеджер `yum`/`dnf`.
- **Alpine Linux** — минималистичный дистрибутив, часто используется как база для Docker-образов из-за небольшого размера.

### 3. Способы получить доступ к Linux для практики

- **Windows Subsystem for Linux (WSL)** — позволяет запускать полноценный Linux прямо внутри Windows, самый удобный способ для большинства новичков на Windows.
- **Виртуальная машина** (VirtualBox, VMware) — установка полноценной Linux-системы в отдельном виртуальном окружении.
- **macOS/Linux уже содержит Unix-подобный терминал**, который во многом схож с Linux для базовых команд.
- **Облачный сервер** — можно арендовать небольшой виртуальный сервер у облачного провайдера и практиковаться удалённо.

### 4. Установка WSL (для пользователей Windows)

```bash
wsl --install
```

Эта команда, выполненная в PowerShell от имени администратора, установит WSL и дистрибутив Ubuntu по умолчанию. После установки и перезагрузки Ubuntu запускается как обычное приложение.

### 5. Первый запуск терминала

После установки открой терминал (в Ubuntu/WSL, macOS Terminal или Linux) и выполни:

```bash
whoami        # покажет имя текущего пользователя
pwd            # покажет текущую рабочую папку
uname -a       # покажет информацию о системе и ядре
```

### 6. Структура терминальной строки-приглашения (prompt)

Типичная строка приглашения в Linux выглядит так:

```
user@hostname:~/projects$
```

Где `user` — имя пользователя, `hostname` — имя компьютера/сервера, `~/projects` — текущая папка (`~` означает домашнюю директорию), `$` — символ, означающий, что это обычный пользователь (у администратора обычно `#`).

### Резюме урока

- Linux — ядро операционной системы, на основе которого созданы разные дистрибутивы (Ubuntu, CentOS, Alpine и др.).
- Практиковаться в Linux на Windows удобнее всего через WSL.
- Строка приглашения терминала содержит имя пользователя, имя компьютера и текущую директорию.

## Урок 2.2. Файловая система и базовые команды (ls, cd, pwd, mkdir)
**Время: 45 минут**

### Цель урока
Изучить структуру файловой системы Linux и освоить базовые команды навигации.

### 1. Структура файловой системы Linux

В отличие от Windows, где у каждого диска есть своя буква (C:, D:), в Linux вся файловая система представляет собой единое дерево, начинающееся с корня `/`:

```
/
├── home/       # домашние папки пользователей
├── etc/        # конфигурационные файлы системы
├── var/        # изменяемые данные (логи и т.д.)
├── usr/        # установленные программы и библиотеки
├── bin/        # основные системные исполняемые файлы
└── tmp/        # временные файлы
```

### 2. Команда pwd — узнать текущую папку

```bash
pwd
# /home/user/projects
```

### 3. Команда cd — переход между папками

```bash
cd projects           # перейти в папку projects (относительный путь)
cd /home/user/projects  # перейти по абсолютному пути
cd ..                     # перейти на уровень выше
cd ~                       # перейти в домашнюю папку
cd -                       # вернуться в предыдущую папку
```

### 4. Команда ls — просмотр содержимого папки

```bash
ls              # список файлов и папок
ls -l            # подробный список (права, размер, дата)
ls -a            # включая скрытые файлы (начинающиеся с точки)
ls -la           # комбинация -l и -a
```

### 5. Команда mkdir — создание папок

```bash
mkdir new_folder                    # создать одну папку
mkdir -p projects/app/src           # создать вложенные папки сразу
```

### 6. Практический пример: создание структуры проекта

```bash
mkdir my_project
cd my_project
mkdir src tests docs
ls -l
```

Этот набор команд создаёт папку проекта с тремя подпапками и выводит их список.

### Резюме урока

- Файловая система Linux представляет собой единое дерево, начинающееся с корня `/`.
- `pwd` показывает текущую папку, `cd` — перемещает между папками, `ls` — показывает содержимое.
- `mkdir -p` создаёт сразу несколько вложенных папок одной командой.

## Урок 2.3. Работа с файлами: cp, mv, rm, cat, nano/vim
**Время: 45 минут**

### Цель урока
Научиться создавать, копировать, перемещать, удалять и просматривать файлы через терминал.

### 1. Создание файлов

```bash
touch file.txt              # создать пустой файл
echo "Привет, Linux" > file.txt   # создать файл с содержимым (перезаписывает)
echo "Ещё строка" >> file.txt      # добавить строку в конец файла (не перезаписывает)
```

### 2. Просмотр содержимого файла

```bash
cat file.txt        # вывести всё содержимое файла
less file.txt         # постраничный просмотр (удобно для больших файлов)
head file.txt          # первые 10 строк файла
tail file.txt           # последние 10 строк файла
tail -f app.log        # непрерывный просмотр добавляемых строк (удобно для логов)
```

### 3. Копирование файлов и папок: cp

```bash
cp file.txt file_copy.txt         # копирование файла
cp -r project/ project_backup/    # копирование папки (флаг -r - рекурсивно)
```

### 4. Перемещение и переименование: mv

```bash
mv file.txt documents/          # переместить файл в другую папку
mv old_name.txt new_name.txt    # переименовать файл (перемещение "на месте")
```

### 5. Удаление файлов и папок: rm

```bash
rm file.txt              # удалить файл
rm -r old_project/       # удалить папку рекурсивно
rm -rf old_project/      # удалить без подтверждения (использовать с осторожностью!)
```

> Важное предупреждение: команда `rm` в Linux не имеет «корзины» — удалённые файлы восстановить стандартными средствами невозможно. Особенно осторожно нужно быть с `rm -rf`, которая удаляет без каких-либо подтверждений.

### 6. Текстовые редакторы в терминале: nano и vim

`nano` — простой редактор, удобный для новичков:

```bash
nano file.txt
# Ctrl+O - сохранить, Ctrl+X - выйти
```

`vim` — значительно более мощный, но сложный в освоении редактор, широко распространённый в серверной среде:

```bash
vim file.txt
# i - войти в режим редактирования
# Esc - выйти из режима редактирования
# :wq - сохранить и выйти
# :q! - выйти без сохранения
```

### Резюме урока

- `cat`, `less`, `head`, `tail` — основные команды для просмотра содержимого файлов, `tail -f` особенно полезна для просмотра логов в реальном времени.
- `cp`, `mv`, `rm` — команды копирования, перемещения и удаления, при этом `rm` удаляет файлы безвозвратно.
- `nano` — простой редактор для новичков, `vim` — более мощный, но требующий отдельного изучения инструмент, часто встречающийся на серверах.

## Урок 2.4. Права доступа и пользователи (chmod, chown, sudo)
**Время: 40 минут**

### Цель урока
Понять систему прав доступа в Linux и научиться управлять правами файлов и пользователями.

### 1. Три типа прав доступа

Каждый файл в Linux имеет права на три вида действий:

- **r (read)** — чтение.
- **w (write)** — запись/изменение.
- **x (execute)** — выполнение (для программ и скриптов).

### 2. Три категории пользователей

- **owner (владелец)** — пользователь, создавший файл.
- **group (группа)** — группа пользователей, связанная с файлом.
- **others (остальные)** — все прочие пользователи системы.

### 3. Просмотр прав через ls -l

```bash
ls -l
# -rwxr-xr-- 1 user group 220 Jul 12 10:00 script.sh
```

Первые 10 символов расшифровываются так: тип файла (`-` для обычного файла, `d` для папки), затем по три символа прав для владельца (`rwx`), группы (`r-x`) и остальных (`r--`).

### 4. Изменение прав: chmod

```bash
chmod +x script.sh          # добавить право на выполнение всем
chmod u+w file.txt           # добавить право записи только владельцу
chmod 755 script.sh           # установить права числовым способом
```

Числовой способ основан на сумме значений: r=4, w=2, x=1. Например, `755` означает: владелец — 7 (4+2+1 = rwx), группа — 5 (4+1 = r-x), остальные — 5 (r-x).

### 5. Изменение владельца: chown

```bash
chown user2 file.txt              # изменить владельца файла
chown user2:group2 file.txt        # изменить владельца и группу
```

### 6. Команда sudo — выполнение от имени администратора

Многие операции (установка пакетов, изменение системных файлов) требуют прав суперпользователя (root). Вместо постоянной работы под root-пользователем (что небезопасно), используется команда `sudo`, временно повышающая права для одной конкретной команды:

```bash
sudo apt update              # обновление списка пакетов от имени администратора
sudo chown root file.txt       # смена владельца файла на root
```

### Резюме урока

- Права доступа в Linux делятся на чтение (r), запись (w) и выполнение (x) для трёх категорий: владелец, группа, остальные.
- `chmod` изменяет права доступа к файлу, `chown` — его владельца.
- `sudo` временно выполняет отдельную команду с правами администратора, что безопаснее постоянной работы под root.

## Урок 2.5. Процессы и управление службами (ps, top, systemctl)
**Время: 40 минут**

### Цель урока
Научиться просматривать запущенные процессы, управлять ими и работать с системными службами.

### 1. Что такое процесс

Процесс — это выполняющаяся программа. У каждого процесса есть уникальный идентификатор (PID — Process ID), который можно использовать для управления им.

### 2. Просмотр процессов: ps

```bash
ps                # процессы текущего терминала
ps aux            # все процессы в системе с подробной информацией
ps aux | grep nginx   # поиск конкретного процесса (например, nginx)
```

### 3. Интерактивный просмотр: top / htop

`top` показывает список процессов в реальном времени с информацией об использовании CPU и памяти:

```bash
top
# q - выход
```

`htop` — более удобная, визуально приятная альтернатива (может потребовать отдельной установки).

### 4. Завершение процесса: kill

```bash
kill 1234           # отправить сигнал завершения процессу с PID 1234
kill -9 1234         # принудительное завершение (SIGKILL), если процесс не отвечает
```

### 5. Фоновые процессы

```bash
long_running_script.sh &     # запуск в фоновом режиме
jobs                            # список фоновых задач текущей сессии
fg %1                           # вернуть задачу 1 на передний план
```

### 6. Управление службами через systemctl

В современных дистрибутивах Linux (использующих systemd) службы (например, веб-сервер nginx или база данных) управляются через команду `systemctl`:

```bash
sudo systemctl status nginx      # статус службы
sudo systemctl start nginx        # запустить службу
sudo systemctl stop nginx          # остановить службу
sudo systemctl restart nginx       # перезапустить службу
sudo systemctl enable nginx        # включить автозапуск при загрузке системы
```

### Резюме урока

- Каждый запущенный процесс имеет уникальный PID, по которому им можно управлять (например, через `kill`).
- `ps aux` и `top`/`htop` — основные инструменты просмотра запущенных процессов и потребления ресурсов.
- `systemctl` — стандартный инструмент управления системными службами в современных дистрибутивах Linux.

## Урок 2.6. Основы bash-скриптов и переменных окружения
**Время: 45 минут**

### Цель урока
Научиться писать простые bash-скрипты для автоматизации повторяющихся задач в терминале.

### 1. Что такое bash-скрипт

Bash-скрипт — это текстовый файл с последовательностью команд терминала, которые можно выполнить одним запуском, вместо того чтобы вводить их вручную по одной. Bash-скрипты — один из базовых инструментов автоматизации в DevOps.

### 2. Первый скрипт

```bash
#!/bin/bash
echo "Привет из bash-скрипта!"
echo "Текущая папка: $(pwd)"
```

Первая строка `#!/bin/bash` (шебанг) указывает системе, каким интерпретатором нужно выполнять этот файл.

### 3. Запуск скрипта

```bash
chmod +x script.sh     # даём файлу право на выполнение
./script.sh              # запускаем скрипт
```

### 4. Переменные в bash

```bash
#!/bin/bash
name="Алина"
echo "Привет, $name!"

age=20
echo "Через год будет $((age + 1)) лет"
```

> Важно: в bash нельзя ставить пробелы вокруг знака `=` при присваивании переменной.

### 5. Условия и циклы в bash

```bash
#!/bin/bash
count=5

if [ $count -gt 3 ]; then
    echo "Count больше 3"
else
    echo "Count не больше 3"
fi

for i in 1 2 3 4 5; do
    echo "Число: $i"
done
```

### 6. Переменные окружения

Переменные окружения — специальные переменные, доступные всем процессам операционной системы, часто используются для хранения конфигурации (пути, ключи API и т.д.):

```bash
echo $HOME          # домашняя папка пользователя
echo $PATH           # список папок, где система ищет исполняемые файлы

export API_KEY="secret_value"    # установка новой переменной окружения
echo $API_KEY
```

### 7. Практический пример: скрипт резервного копирования

```bash
#!/bin/bash
BACKUP_DIR="backups"
DATE=$(date +%Y-%m-%d)

mkdir -p $BACKUP_DIR
cp -r project/ "$BACKUP_DIR/project_$DATE"

echo "Резервная копия создана: $BACKUP_DIR/project_$DATE"
```

### Резюме урока

- Bash-скрипты позволяют автоматизировать последовательности команд терминала в единый исполняемый файл.
- Скрипт запускается через `./имя_скрипта.sh` после выдачи прав на выполнение через `chmod +x`.
- Переменные окружения (`$HOME`, `$PATH` и др.) доступны всем процессам системы и часто используются для хранения конфигурации в DevOps-инструментах.
