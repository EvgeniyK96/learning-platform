## Урок 1.1. Что такое JavaScript и где он применяется
**Время: 30 минут**

### Цель урока
Понять, что такое JavaScript, чем он отличается от других языков и где применяется в современной разработке.

### 1. Что такое JavaScript

JavaScript — это язык программирования, изначально созданный для того, чтобы делать веб-страницы интерактивными: реагировать на действия пользователя, изменять содержимое страницы без перезагрузки, проверять данные форм. Сегодня JavaScript — один из самых востребованных языков программирования в мире.

Важно не путать JavaScript с Java — это два совершенно разных языка с похожим названием по историческим причинам маркетинга 1990-х годов.

### 2. Особенности JavaScript

- **Интерпретируемый язык.** Код выполняется браузером (или средой Node.js) без отдельного этапа компиляции.
- **Динамическая типизация.** Тип переменной определяется автоматически.
- **Язык браузера по умолчанию.** JavaScript — единственный язык программирования, который «из коробки» понимают все современные веб-браузеры.
- **Универсальность.** С появлением Node.js JavaScript вышел за пределы браузера и используется на сервере, в мобильной разработке, для написания скриптов.

### 3. Где применяется JavaScript

- **Фронтенд веб-разработка.** Интерактивность сайтов: выпадающие меню, формы, анимации, динамическое обновление данных. Основные фреймворки: React, Vue, Angular.
- **Бэкенд-разработка.** С помощью Node.js на JavaScript можно писать серверную часть приложений.
- **Мобильная разработка.** React Native и подобные технологии позволяют писать мобильные приложения на JavaScript.
- **Настольные приложения.** Такие приложения, как VS Code и Slack, написаны с использованием технологии Electron, основанной на JavaScript.
- **Игры в браузере.** Существуют библиотеки (Phaser и др.) для разработки 2D-игр прямо в браузере.

### 4. Как выполняется JavaScript-код

В браузере JavaScript выполняется специальным движком: например, V8 в Google Chrome и Node.js, SpiderMonkey в Firefox. Именно движок V8 сделал возможным вынос JavaScript за пределы браузера — на его основе построен Node.js.

### 5. Почему JavaScript — хороший выбор для изучения

- Это единственный язык, который выполняется в браузере «из коробки» — без него невозможна фронтенд-разработка.
- Огромный спрос на рынке труда, множество вакансий как для фронтенд-, так и для фулстек-разработчиков.
- Быстрый визуальный результат: уже после нескольких уроков можно увидеть, как код меняет реальную веб-страницу.
- Большое и активное сообщество, множество готовых библиотек и инструментов.

### Резюме урока

- JavaScript — язык программирования, изначально созданный для интерактивности веб-страниц.
- Сегодня используется не только в браузере, но и на сервере (Node.js), в мобильной и настольной разработке.
- Код JavaScript выполняется специальным движком (например, V8), без отдельного этапа компиляции.

## Урок 1.2. Настройка окружения: браузер, VS Code, расширение Live Server
**Время: 35 минут**

### Цель урока
Установить и настроить всё необходимое для комфортного написания и запуска JavaScript-кода.

### 1. Что нужно для старта

В отличие от многих языков программирования, для JavaScript не требуется устанавливать отдельный компилятор или интерпретатор — достаточно современного браузера (Chrome, Firefox, Edge), в который уже встроен движок для выполнения JS-кода.

### 2. Установка VS Code

1. Перейди на сайт code.visualstudio.com и скачай установщик для своей операционной системы.
2. Установи программу по инструкциям установщика.
3. Запусти VS Code.

### 3. Установка расширения Live Server

Расширение Live Server позволяет автоматически открывать HTML-файл в браузере и обновлять страницу при каждом изменении кода — это значительно ускоряет разработку.

1. Открой раздел расширений в VS Code (`Ctrl+Shift+X`).
2. В поиске введи «Live Server».
3. Установи расширение от Ritwick Dey.

### 4. Создание первого проекта

1. Создай папку `js_course`.
2. Открой её в VS Code через «File → Open Folder».
3. Создай файл `index.html` со следующим минимальным содержимым:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Мой первый JS-проект</title>
</head>
<body>
    <h1>Привет, JavaScript!</h1>
    <script src="script.js"></script>
</body>
</html>
```

4. Создай рядом файл `script.js`.

### 5. Запуск через Live Server

Кликни правой кнопкой мыши на файл `index.html` в VS Code и выбери «Open with Live Server». Страница откроется в браузере по адресу вида `http://127.0.0.1:5500`, и будет автоматически обновляться при сохранении изменений в файлах.

### 6. Панель инструментов разработчика (DevTools)

Один из главных инструментов JS-разработчика — встроенная в браузер панель разработчика:

- Открывается клавишей `F12` или сочетанием `Ctrl+Shift+I`.
- Вкладка **Console** — здесь можно видеть вывод `console.log()` и результаты ошибок.
- Вкладка **Elements** — показывает HTML-структуру страницы в реальном времени.
- Вкладка **Sources** — позволяет отлаживать JS-код с точками останова.

### Резюме урока

- Для JavaScript не нужен отдельный компилятор — достаточно браузера и текстового редактора.
- Live Server автоматически обновляет страницу при изменении кода, экономя время разработки.
- Инструменты разработчика (DevTools) в браузере — ключевой инструмент отладки и просмотра результатов работы кода.

## Урок 1.3. Подключение JS к HTML-странице, консоль разработчика
**Время: 35 минут**

### Цель урока
Научиться подключать JavaScript-код к HTML-странице разными способами и работать с консолью разработчика.

### 1. Способы подключения JavaScript к странице

**Способ 1. Встроенный код внутри тега `<script>`:**

```html
<body>
    <script>
        console.log("Привет из встроенного скрипта!");
    </script>
</body>
```

**Способ 2. Подключение внешнего файла (рекомендуемый способ):**

```html
<body>
    <script src="script.js"></script>
</body>
```

Разделение HTML и JS на отдельные файлы делает код более организованным и позволяет переиспользовать один и тот же JS-файл на разных страницах.

### 2. Где размещать тег script

Тег `<script>` лучше размещать перед закрывающим тегом `</body>`, а не в `<head>`. Причина: браузер читает HTML-документ сверху вниз, и если скрипт пытается обратиться к элементам страницы, которые ещё не загружены (потому что находятся ниже в файле), это приведёт к ошибке.

```html
<body>
    <h1>Заголовок</h1>
    <p>Текст страницы</p>
    <script src="script.js"></script>   <!-- в самом конце -->
</body>
```

### 3. Атрибут defer

Альтернативный способ — разместить скрипт в `<head>` с атрибутом `defer`, который откладывает выполнение скрипта до полной загрузки страницы:

```html
<head>
    <script src="script.js" defer></script>
</head>
```

### 4. Консоль разработчика

Консоль (Console) — один из главных инструментов при разработке на JavaScript. Через неё можно:

- Видеть результат вызова `console.log()`.
- Видеть сообщения об ошибках в коде.
- Выполнять произвольный JS-код прямо в браузере для тестирования.

```javascript
console.log("Обычное сообщение");
console.warn("Предупреждение");
console.error("Сообщение об ошибке");
```

### 5. Первый скрипт

```javascript
// script.js
console.log("Скрипт подключён и работает!");
alert("Добро пожаловать!");
```

`alert()` показывает всплывающее окно с сообщением — удобно для самых первых экспериментов, но в реальной разработке используется редко, так как блокирует страницу до нажатия «ОК».

### 6. Типичные ошибки новичков

- Указывают неверный путь к файлу в атрибуте `src` — скрипт не подключается, и в консоли появляется ошибка `404 Not Found`.
- Размещают `<script>` в начале `<body>` без атрибута `defer` и получают ошибку при попытке обратиться к ещё не загруженным элементам страницы.

### Резюме урока

- Подключать JavaScript к странице можно встроенным кодом или (предпочтительнее) внешним файлом через `<script src="...">`.
- Скрипт стоит размещать в конце `<body>` или использовать атрибут `defer`.
- Консоль разработчика (`F12`) — основной инструмент для просмотра вывода и ошибок JS-кода.

## Урок 1.4. Первая программа и вывод данных (console.log, alert)
**Время: 30 минут**

### Цель урока
Написать первую программу на JavaScript и разобраться в основных способах вывода данных.

### 1. Функция console.log()

`console.log()` — самая часто используемая функция при разработке и отладке на JavaScript. Она выводит переданные данные в консоль разработчика.

```javascript
console.log("Hello, World!");
console.log(42);
console.log(true);
```

### 2. Вывод нескольких значений

```javascript
let name = "Алина";
let age = 19;

console.log("Имя:", name, "Возраст:", age);
```

### 3. Функция alert()

`alert()` показывает всплывающее окно поверх страницы:

```javascript
alert("Добро пожаловать на сайт!");
```

Такое окно блокирует выполнение остальной части страницы до тех пор, пока пользователь не нажмёт «ОК» — из-за этого `alert()` редко используется в реальных проектах, но полезен для самых первых экспериментов.

### 4. Функции prompt() и confirm()

`prompt()` показывает окно с полем ввода и возвращает введённый пользователем текст:

```javascript
let userName = prompt("Как тебя зовут?");
console.log("Привет, " + userName);
```

`confirm()` показывает окно с кнопками «ОК»/«Отмена» и возвращает `true` или `false`:

```javascript
let isConfirmed = confirm("Ты уверен?");
console.log(isConfirmed);
```

### 5. Комментарии в коде

```javascript
// Это однострочный комментарий

/*
Это многострочный
комментарий
*/

console.log("Код после комментария");
```

### 6. Практический пример

```javascript
let name = prompt("Введите имя:");
let age = prompt("Введите возраст:");

console.log(`Привет, ${name}! Тебе ${age} лет.`);
```

(Здесь используется шаблонная строка — подробнее о ней в Уроке 2.5.)

### Резюме урока

- `console.log()` — основной инструмент вывода данных при разработке и отладке.
- `alert()`, `prompt()`, `confirm()` — встроенные способы взаимодействия с пользователем через всплывающие окна браузера, используются в основном в учебных целях.
- Комментарии (`//` и `/* */`) помогают документировать код без влияния на его выполнение.
