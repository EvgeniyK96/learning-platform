## Урок 2.1. Переменные: var, let, const
**Время: 40 минут**

### Цель урока
Понять разницу между тремя способами объявления переменных в JavaScript и научиться выбирать подходящий в каждой ситуации.

### 1. Три способа объявления переменных

```javascript
var oldWay = "старый способ";
let modernWay = "современный способ";
const constantValue = "неизменяемое значение";
```

### 2. let — переменная, которую можно менять

```javascript
let age = 20;
age = 21;         // можно переприсвоить
console.log(age); // 21
```

### 3. const — константа

`const` используется для значений, которые не должны меняться после присваивания:

```javascript
const birthYear = 2005;
birthYear = 2006;   // TypeError: Assignment to constant variable
```

> Важно: `const` запрещает переприсваивание переменной, но если в `const` хранится объект или массив, их внутреннее содержимое всё же можно изменять (подробнее в Модуле 5).

### 4. Почему var считается устаревшим

`var` был единственным способом объявления переменных до 2015 года. У него есть особенности, которые часто приводят к ошибкам:

- `var` не учитывает блочную область видимости (scope внутри `{}`), в отличие от `let` и `const`.
- Переменные, объявленные через `var`, можно случайно переобъявить с тем же именем без ошибки.

```javascript
if (true) {
    var x = 10;
}
console.log(x);   // 10 - var "утекает" из блока if

if (true) {
    let y = 10;
}
console.log(y);   // ReferenceError - let не утекает из блока
```

В современном коде рекомендуется использовать `let` и `const`, а `var` — избегать.

### 5. Какой способ выбирать

Общепринятая практика: по умолчанию использовать `const`, а `let` — только тогда, когда переменная действительно должна изменяться в процессе работы программы. Это делает код более предсказуемым — сразу видно, какие переменные могут менять значение.

### 6. Правила именования переменных

- Может содержать буквы, цифры, `$` и `_`, но не может начинаться с цифры.
- Регистрозависимость: `age` и `Age` — разные переменные.
- Стиль именования в JavaScript — `camelCase`: `userName`, а не `user_name`.

```javascript
let userName = "Данияр";   // хорошо, camelCase
let user_name = "Данияр";  // работает, но не соответствует конвенции JS
```

### Резюме урока

- `let` — для переменных, значение которых будет меняться, `const` — для неизменяемых значений.
- `var` считается устаревшим способом из-за особенностей области видимости и не рекомендуется в новом коде.
- По умолчанию стоит использовать `const`, переключаясь на `let` только при необходимости.

## Урок 2.2. Типы данных и оператор typeof
**Время: 40 минут**

### Цель урока
Изучить основные типы данных JavaScript и научиться определять тип значения с помощью оператора `typeof`.

### 1. Примитивные типы данных

| Тип | Пример | Описание |
|---|---|---|
| `number` | `10`, `3.14` | Числа (целые и дробные — единый тип) |
| `string` | `"текст"` | Строки |
| `boolean` | `true`, `false` | Логический тип |
| `undefined` | — | Переменная объявлена, но значение не присвоено |
| `null` | — | Специальное значение «пустоты», отсутствия значения |

### 2. Оператор typeof

```javascript
console.log(typeof 10);          // "number"
console.log(typeof "текст");     // "string"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" - историческая особенность/баг языка
```

> Особенность: `typeof null` возвращает `"object"` — это известная историческая ошибка в дизайне языка, сохранённая для обратной совместимости.

### 3. Отличие undefined от null

```javascript
let a;                 // переменная объявлена, но значение не присвоено
console.log(a);        // undefined

let b = null;           // переменной явно присвоено "ничего"
console.log(b);         // null
```

`undefined` обычно означает, что JavaScript ещё не присвоил значение, а `null` — что разработчик сам явно указал «отсутствие значения».

### 4. Числа в JavaScript

В отличие от многих языков, в JavaScript нет отдельных типов для целых и дробных чисел — всё это единый тип `number`:

```javascript
let integer = 10;
let float = 3.14;
console.log(typeof integer);   // "number"
console.log(typeof float);     // "number"
```

Особые числовые значения:

```javascript
console.log(10 / 0);       // Infinity
console.log(-10 / 0);      // -Infinity
console.log("текст" * 2);  // NaN (Not a Number)
console.log(typeof NaN);   // "number" (да, NaN тоже "число")
```

### 5. Строки

```javascript
let single = 'одинарные кавычки';
let double = "двойные кавычки";
let backtick = `обратные кавычки (шаблонная строка)`;
```

### 6. Составные типы (кратко — подробно в Модуле 5)

Помимо примитивных типов, в JavaScript есть составные (объектные) типы — массивы (`array`) и объекты (`object`), которые технически оба относятся к типу `object`:

```javascript
console.log(typeof []);           // "object"
console.log(typeof {});           // "object"
console.log(typeof function(){}); // "function"
```

### Резюме урока

- Основные примитивные типы: `number`, `string`, `boolean`, `undefined`, `null`.
- `typeof` позволяет проверить тип значения, но имеет историческую особенность — `typeof null` возвращает `"object"`.
- В JavaScript нет разделения на целые и дробные числа — все числа относятся к единому типу `number`.

## Урок 2.3. Операторы: арифметические, сравнения, логические
**Время: 40 минут**

### Цель урока
Изучить основные группы операторов JavaScript и научиться комбинировать условия с помощью логических операторов.

### 1. Арифметические операторы

| Оператор | Значение | Пример | Результат |
|---|---|---|---|
| `+` | сложение | `5 + 2` | `7` |
| `-` | вычитание | `5 - 2` | `3` |
| `*` | умножение | `5 * 2` | `10` |
| `/` | деление | `5 / 2` | `2.5` |
| `%` | остаток от деления | `5 % 2` | `1` |
| `**` | возведение в степень | `5 ** 2` | `25` |

### 2. Операторы инкремента и декремента

```javascript
let count = 5;
count++;             // эквивалент count = count + 1
console.log(count);  // 6
count--;
console.log(count);  // 5
```

### 3. Операторы сравнения

```javascript
console.log(5 > 3);    // true
console.log(5 < 3);    // false
console.log(5 >= 5);   // true
console.log(5 <= 4);   // false
```

Особое внимание стоит уделить операторам равенства — в JavaScript их два вида:

```javascript
console.log(5 == "5");    // true (сравнивает значения, приводя типы)
console.log(5 === "5");   // false (строгое сравнение, учитывает тип)
```

`===` (строгое равенство) считается более безопасным и предсказуемым выбором и рекомендуется к использованию по умолчанию вместо `==`.

### 4. Логические операторы

| Оператор | Значение |
|---|---|
| `&&` | логическое И |
| `\|\|` | логическое ИЛИ |
| `!` | логическое НЕ |

```javascript
let age = 20;
let hasId = true;

console.log(age >= 18 && hasId);   // true
console.log(age < 18 || hasId);     // true
console.log(!hasId);                 // false
```

### 5. Приоритет операторов

Как и в математике, у операторов есть приоритет выполнения. Скобки `()` позволяют явно задать нужный порядок:

```javascript
let result = (2 + 3) * 4;   // 20, а не 14
```

### 6. Оператор нулевого слияния (??) и опциональная цепочка

Современный JavaScript (ES2020+) добавил несколько полезных операторов:

```javascript
let username = null;
let displayName = username ?? "Гость";   // "Гость", если username null или undefined
console.log(displayName);
```

### Резюме урока

- Строгое равенство `===` предпочтительнее нестрогого `==`, так как не выполняет неявное приведение типов.
- Логические операторы `&&`, `||`, `!` позволяют комбинировать несколько условий.
- Приоритет операторов можно явно задавать через скобки.

## Урок 2.4. Приведение типов и особенности сравнения (==, ===)
**Время: 35 минут**

### Цель урока
Понять механизм приведения типов в JavaScript и научиться избегать типичных ошибок, связанных с ним.

### 1. Явное приведение типов

```javascript
let str = "42";
let num = Number(str);        // явное преобразование в число
console.log(num, typeof num); // 42 "number"

let numToStr = String(42);
console.log(numToStr, typeof numToStr);  // "42" "string"

let bool = Boolean(1);
console.log(bool);   // true
```

### 2. Функции parseInt и parseFloat

```javascript
console.log(parseInt("42px"));     // 42 (извлекает число из начала строки)
console.log(parseFloat("3.14м"));  // 3.14
console.log(Number("42px"));         // NaN (Number требует полностью корректную строку)
```

### 3. Неявное приведение типов

JavaScript часто автоматически приводит типы при выполнении операций — это может приводить к неожиданным результатам, если не понимать механику:

```javascript
console.log("5" + 3);      // "53" (число превращается в строку)
console.log("5" - 3);      // 2 (строка превращается в число)
console.log("5" * "2");    // 10
console.log(true + 1);     // 2 (true превращается в 1)
console.log("" + null);    // "null"
```

Правило: оператор `+` со строкой всегда выполняет конкатенацию (склеивание), а остальные арифметические операторы (`-`, `*`, `/`) пытаются привести операнды к числам.

### 4. Falsy и truthy значения

При приведении к `boolean` некоторые значения считаются «ложными» (falsy), все остальные — «истинными» (truthy):

**Falsy-значения:** `false`, `0`, `""` (пустая строка), `null`, `undefined`, `NaN`.

Всё остальное, включая пустой массив `[]` и пустой объект `{}`, считается truthy (в отличие, например, от Python).

```javascript
if ("") {
    console.log("Это не выполнится");
}

if ("текст") {
    console.log("А это выполнится");
}
```

### 5. Почему === безопаснее ==

```javascript
console.log(0 == "");           // true - неожиданно!
console.log(0 == "0");          // true
console.log("" == "0");         // false - при этом это не транзитивно!
console.log(null == undefined); // true
console.log(null === undefined);// false
```

Из-за подобных неочевидных случаев в реальной разработке практически всегда используется строгое сравнение `===` и `!==`.

### 6. Практический пример

```javascript
let userInput = prompt("Введите число:");
let number = Number(userInput);

if (isNaN(number)) {
    console.log("Это не число!");
} else {
    console.log(`Квадрат числа: ${number ** 2}`);
}
```

### Резюме урока

- JavaScript активно выполняет неявное приведение типов, что может приводить к неожиданным результатам.
- Оператор `+` со строкой выполняет конкатенацию, остальные арифметические операторы приводят операнды к числам.
- Строгое сравнение `===` рекомендуется использовать по умолчанию, чтобы избежать сюрпризов нестрогого сравнения `==`.

## Урок 2.5. Шаблонные строки (template literals)
**Время: 35 минут**

### Цель урока
Научиться использовать шаблонные строки для удобного форматирования текста и вставки значений переменных.

### 1. Что такое шаблонные строки

Шаблонные строки (template literals) — это строки, заключённые в обратные кавычки (`` ` ``), которые позволяют вставлять значения переменных и выражений прямо внутрь текста.

```javascript
let name = "Алина";
let age = 20;

let message = `Меня зовут ${name}, мне ${age} лет`;
console.log(message);
```

### 2. Сравнение со «старым» способом конкатенации

```javascript
// Старый способ
let oldMessage = "Меня зовут " + name + ", мне " + age + " лет";

// Современный способ
let newMessage = `Меня зовут ${name}, мне ${age} лет`;
```

Шаблонные строки значительно улучшают читаемость кода, особенно при большом количестве вставляемых значений.

### 3. Вставка выражений

Внутри `${}` можно писать любые JavaScript-выражения, а не только имена переменных:

```javascript
let price = 1500;
console.log(`Цена со скидкой 10%: ${price * 0.9} тг`);

let a = 5, b = 3;
console.log(`Сумма: ${a + b}, произведение: ${a * b}`);
```

### 4. Многострочные строки

Одно из главных удобств шаблонных строк — возможность писать текст на нескольких строках без специальных символов переноса:

```javascript
let poem = `Первая строка
Вторая строка
Третья строка`;

console.log(poem);
```

При использовании обычных кавычек для этого потребовалось бы явно вставлять символ `\n`.

### 5. Вызов функций внутри шаблонных строк

```javascript
function getGreeting(name) {
    return `Привет, ${name}!`;
}

console.log(`Сообщение: ${getGreeting("Данияр")}`);
```

### 6. Практический пример: чек из магазина

```javascript
let items = [
    { name: "Хлеб", price: 350 },
    { name: "Молоко", price: 620 }
];

let total = items.reduce((sum, item) => sum + item.price, 0);

let receipt = items.map(item => `${item.name}: ${item.price} тг`).join("\n");
console.log(`${receipt}\n-----\nИтого: ${total} тг`);
```

### Резюме урока

- Шаблонные строки заключаются в обратные кавычки и позволяют вставлять переменные и выражения через `${...}`.
- Значительно упрощают форматирование текста по сравнению с конкатенацией через `+`.
- Поддерживают многострочный текст без дополнительных символов переноса строки.
