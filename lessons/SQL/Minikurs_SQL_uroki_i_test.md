# Модуль 1. Основы реляционных баз данных и SELECT

## Урок 1.1. Что такое реляционная база данных, таблицы, ключи
**Время: 35 минут**

### Цель урока
Понять базовые концепции реляционных баз данных: таблицы, строки, столбцы, первичные и внешние ключи.

### 1. Что такое база данных

База данных — это организованная структура для хранения, изменения и извлечения данных. В отличие от хранения данных в обычных файлах (например, CSV или текстовых), базы данных обеспечивают эффективный поиск, целостность данных и одновременный доступ множества пользователей или приложений.

### 2. Что такое реляционная база данных

Реляционная база данных организует данные в виде связанных между собой таблиц (отсюда и название — от англ. relation, «отношение», в математическом смысле таблицы). SQL (Structured Query Language — «язык структурированных запросов») — стандартный язык для взаимодействия с реляционными базами данных: создания таблиц, добавления, изменения, удаления и выборки данных.

### 3. Структура таблицы

Таблица состоит из строк (records, «записей») и столбцов (columns, «полей»). Каждый столбец имеет определённое имя и тип данных, каждая строка представляет собой одну отдельную запись:

```
Таблица students
┌────┬──────────┬──────┬────────────┐
│ id │  name    │ age  │   course   │
├────┼──────────┼──────┼────────────┤
│ 1  │  Алина   │  20  │ SQL с нуля │
│ 2  │  Данияр  │  19  │ SQL с нуля │
└────┴──────────┴──────┴────────────┘
```

### 4. Первичный ключ (Primary Key)

Первичный ключ — столбец (или комбинация столбцов), однозначно идентифицирующий каждую строку в таблице. Значения первичного ключа обязаны быть уникальными и не могут быть пустыми (`NULL`). В примере выше столбец `id` — типичный первичный ключ.

### 5. Внешний ключ (Foreign Key)

Внешний ключ — столбец, ссылающийся на первичный ключ другой (или той же самой) таблицы, устанавливающий связь между двумя таблицами:

```
Таблица orders
┌────┬─────────────┬────────────┐
│ id │ student_id  │  book_name │
├────┼─────────────┼────────────┤
│ 1  │      1        │  Гарри Поттер │
│ 2  │      1        │  Властелин колец │
└────┴─────────────┴────────────┘
```

Столбец `student_id` в таблице `orders` — внешний ключ, ссылающийся на `id` в таблице `students`, тем самым связывая каждый заказ с конкретным студентом.

### 6. Популярные реляционные СУБД

СУБД (система управления базами данных) — программное обеспечение для работы с базами данных. Самые распространённые реляционные СУБД: **PostgreSQL**, **MySQL**, **SQLite** (лёгкая встроенная база данных, удобная для обучения и небольших приложений), **Microsoft SQL Server**, **Oracle Database**. Синтаксис SQL, изучаемый в этом курсе, в основном универсален для всех перечисленных систем, с незначительными различиями в деталях.

### 7. Проектирование таблиц: типы данных столбцов

Каждый столбец таблицы имеет определённый тип данных, схожий с типами в языках программирования:

| Тип SQL | Назначение |
|---|---|
| `INTEGER` | целое число |
| `REAL` / `FLOAT` | дробное число |
| `TEXT` / `VARCHAR` | текстовые данные |
| `DATE` / `DATETIME` | дата и время |
| `BOOLEAN` | логическое значение |

### Резюме урока

- Реляционная база данных организует данные в виде связанных таблиц, состоящих из строк и столбцов.
- Первичный ключ однозначно идентифицирует каждую строку таблицы, внешний ключ устанавливает связь между таблицами.
- SQL — универсальный язык для взаимодействия с реляционными базами данных, с небольшими различиями в синтаксисе между конкретными СУБД (PostgreSQL, MySQL, SQLite и др.).

## Урок 1.2. Базовый запрос SELECT, WHERE
**Время: 45 минут**

### Цель урока
Освоить базовый запрос SELECT для выборки данных из таблицы и фильтрацию результатов через WHERE.

### 1. Базовый синтаксис SELECT

```sql
SELECT * FROM students;
```

Эта команда выбирает **все** столбцы (`*`) из таблицы `students`. Точка с запятой `;` в конце традиционно обозначает завершение SQL-инструкции.

### 2. Выбор конкретных столбцов

```sql
SELECT name, age FROM students;
```

Указание конкретных нужных столбцов вместо `*` — хорошая практика: это уменьшает объём передаваемых данных и делает запрос более понятным для читающего его человека.

### 3. Фильтрация результатов: WHERE

```sql
SELECT * FROM students WHERE age > 19;
```

`WHERE` фильтрует строки, оставляя в результате только те, что удовлетворяют указанному условию.

### 4. Операторы сравнения в WHERE

```sql
SELECT * FROM students WHERE age = 20;
SELECT * FROM students WHERE age != 20;      -- или <>
SELECT * FROM students WHERE age > 19;
SELECT * FROM students WHERE age >= 20;
SELECT * FROM students WHERE age < 22;
```

### 5. Логические операторы: AND, OR, NOT

```sql
SELECT * FROM students WHERE age > 18 AND course = 'SQL с нуля';
SELECT * FROM students WHERE age < 18 OR age > 25;
SELECT * FROM students WHERE NOT course = 'SQL с нуля';
```

### 6. Оператор LIKE для поиска по шаблону текста

```sql
SELECT * FROM students WHERE name LIKE 'А%';       -- имена, начинающиеся на "А"
SELECT * FROM students WHERE name LIKE '%ина';      -- имена, заканчивающиеся на "ина"
SELECT * FROM students WHERE name LIKE '%ани%';    -- имена, содержащие "ани" в любом месте
```

Символ `%` в шаблоне `LIKE` обозначает «любое количество любых символов» (включая ноль символов).

### 7. Операторы IN и BETWEEN

```sql
SELECT * FROM students WHERE course IN ('SQL с нуля', 'Python с нуля');   -- вместо нескольких OR
SELECT * FROM students WHERE age BETWEEN 18 AND 25;                              -- включительно с обеих сторон
```

### 8. Проверка на отсутствие значения: IS NULL

```sql
SELECT * FROM students WHERE course IS NULL;         -- строки, где course не заполнен
SELECT * FROM students WHERE course IS NOT NULL;    -- строки, где course заполнен
```

Важно: для проверки на `NULL` (отсутствие значения) нельзя использовать обычное сравнение `= NULL` — оно всегда даст ложный результат; необходимо использовать именно `IS NULL`/`IS NOT NULL`.

### Резюме урока

- `SELECT столбцы FROM таблица` — базовый синтаксис выборки данных, `WHERE` фильтрует строки по заданному условию.
- Операторы `AND`, `OR`, `NOT` комбинируют несколько условий, `LIKE` с символом `%` выполняет поиск по текстовому шаблону.
- Для проверки на отсутствие значения используется `IS NULL`/`IS NOT NULL`, а не обычное сравнение с `NULL`.

## Урок 1.3. Сортировка и ограничение результатов: ORDER BY, LIMIT
**Время: 35 минут**

### Цель урока
Научиться сортировать результаты запроса и ограничивать их количество.

### 1. Сортировка результатов: ORDER BY

```sql
SELECT * FROM students ORDER BY age;         -- по возрастанию (ASC - по умолчанию)
SELECT * FROM students ORDER BY age DESC;   -- по убыванию
```

`ASC` (ascending, по возрастанию) — значение по умолчанию, поэтому его обычно не указывают явно. `DESC` (descending, по убыванию) необходимо указывать явно, если требуется обратный порядок.

### 2. Сортировка по нескольким столбцам

```sql
SELECT * FROM students ORDER BY course, age DESC;
```

Такой запрос сначала сортирует записи по значению столбца `course` (по алфавиту, по возрастанию), а внутри каждой группы с одинаковым `course` — по `age` в порядке убывания.

### 3. Ограничение количества результатов: LIMIT

```sql
SELECT * FROM students LIMIT 5;    -- показать только первые 5 строк результата
```

`LIMIT` особенно полезен при работе с большими таблицами — например, для предпросмотра данных или реализации постраничной навигации (пагинации) без загрузки всей таблицы целиком.

### 4. Пропуск строк: OFFSET

```sql
SELECT * FROM students LIMIT 5 OFFSET 10;   -- пропустить первые 10 строк, затем показать следующие 5
```

Комбинация `LIMIT` и `OFFSET` — стандартный способ реализации пагинации (постраничного вывода) в реальных приложениях: например, для отображения «страницы 3» результатов при 10 записях на страницу используется `LIMIT 10 OFFSET 20`.

### 5. Топ-N записей: практический паттерн

Комбинация `ORDER BY` и `LIMIT` — очень частый и полезный паттерн для получения «топ-N» записей по какому-либо критерию:

```sql
SELECT * FROM students ORDER BY age DESC LIMIT 3;   -- 3 самых старших студента
```

### 6. Порядок написания частей запроса SELECT

Важно соблюдать правильный порядок ключевых слов в запросе — SQL требует строгого следования синтаксической структуре:

```sql
SELECT столбцы
FROM таблица
WHERE условие
ORDER BY столбец
LIMIT число;
```

Например, `WHERE` всегда должен идти перед `ORDER BY`, а `ORDER BY` — перед `LIMIT`; изменение этого порядка приведёт к синтаксической ошибке.

### 7. Практический пример: топ-3 самых молодых студентов на конкретном курсе

```sql
SELECT name, age
FROM students
WHERE course = 'SQL с нуля'
ORDER BY age ASC
LIMIT 3;
```

### Резюме урока

- `ORDER BY столбец [ASC|DESC]` сортирует результаты запроса, по умолчанию — по возрастанию.
- `LIMIT` ограничивает количество возвращаемых строк, `OFFSET` пропускает указанное количество строк — вместе они реализуют пагинацию.
- SQL требует строгого порядка ключевых слов в запросе: `SELECT → FROM → WHERE → ORDER BY → LIMIT`.

## Урок 1.4. Агрегатные функции и группировка: COUNT, SUM, GROUP BY
**Время: 45 минут**

### Цель урока
Научиться вычислять агрегированные показатели (суммы, средние значения, количество) и группировать данные для анализа.

### 1. Что такое агрегатные функции

Агрегатные функции выполняют вычисление над набором строк, возвращая единое итоговое значение — например, сумму, среднее значение, количество записей.

### 2. Основные агрегатные функции

```sql
SELECT COUNT(*) FROM students;              -- количество всех строк в таблице
SELECT SUM(age) FROM students;               -- сумма значений столбца age по всем строкам
SELECT AVG(age) FROM students;               -- среднее значение
SELECT MIN(age) FROM students;                -- минимальное значение
SELECT MAX(age) FROM students;               -- максимальное значение
```

### 3. Подсчёт с условием

```sql
SELECT COUNT(*) FROM students WHERE course = 'SQL с нуля';
```

`WHERE` применяется до вычисления агрегатной функции — сначала отфильтровываются нужные строки, затем над ними выполняется агрегация.

### 4. Группировка данных: GROUP BY

`GROUP BY` группирует строки с одинаковыми значениями указанного столбца, позволяя применить агрегатную функцию отдельно к каждой такой группе:

```sql
SELECT course, COUNT(*) 
FROM students 
GROUP BY course;
```

Этот запрос вернёт количество студентов, сгруппированных по каждому уникальному значению `course` — например: `SQL с нуля - 15`, `Python с нуля - 22`.

### 5. Несколько агрегатных функций в одном запросе с GROUP BY

```sql
SELECT course, COUNT(*) as student_count, AVG(age) as average_age
FROM students
GROUP BY course;
```

Ключевое слово `AS` присваивает вычисленному столбцу более понятное («псевдоним», alias) имя в результатах — без этого столбец с агрегатной функцией отображался бы под менее читаемым техническим именем вроде `COUNT(*)`.

### 6. Фильтрация групп: HAVING

`WHERE` фильтрует отдельные строки **до** группировки, а `HAVING` фильтрует уже сформированные **группы** — например, чтобы оставить только те курсы, где студентов больше определённого количества:

```sql
SELECT course, COUNT(*) as student_count
FROM students
GROUP BY course
HAVING COUNT(*) > 10;
```

Использовать `WHERE` с агрегатной функцией (например, `WHERE COUNT(*) > 10`) — синтаксическая ошибка; для фильтрации по результату агрегатной функции обязательно нужен именно `HAVING`.

### 7. Полный порядок ключевых слов запроса с группировкой

```sql
SELECT столбцы, агрегатная_функция(столбец)
FROM таблица
WHERE условие_для_строк
GROUP BY столбец
HAVING условие_для_групп
ORDER BY столбец
LIMIT число;
```

### 8. Практический пример: средний возраст студентов по курсам, где записано больше 5 человек

```sql
SELECT course, COUNT(*) as total, AVG(age) as avg_age
FROM students
GROUP BY course
HAVING COUNT(*) > 5
ORDER BY avg_age DESC;
```

### Резюме урока

- Агрегатные функции (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`) вычисляют единое значение по набору строк.
- `GROUP BY` группирует строки по значению указанного столбца, позволяя применить агрегатные функции отдельно к каждой группе.
- `WHERE` фильтрует строки до группировки, `HAVING` — уже сформированные группы после применения агрегатных функций.

## Урок 2.1. Добавление, изменение и удаление данных: INSERT, UPDATE, DELETE
**Время: 50 минут**

### Цель урока
Освоить основные команды изменения данных в таблицах базы данных.

### 1. Добавление новой записи: INSERT

```sql
INSERT INTO students (name, age, course)
VALUES ('Ерлан', 21, 'SQL с нуля');
```

Столбец `id`, если он настроен как автоинкрементный первичный ключ (автоматически увеличивающийся при каждой новой записи), обычно не указывается вручную при вставке — база данных сгенерирует его значение автоматически.

### 2. Добавление нескольких записей одной командой

```sql
INSERT INTO students (name, age, course) VALUES
    ('Алина', 20, 'SQL с нуля'),
    ('Данияр', 19, 'Python с нуля'),
    ('Аружан', 22, 'SQL с нуля');
```

### 3. Изменение существующих данных: UPDATE

```sql
UPDATE students
SET age = 21
WHERE name = 'Алина';
```

> **Критически важно:** команда `UPDATE` без условия `WHERE` изменит **абсолютно все** строки таблицы — это одна из самых частых и опасных ошибок при работе с SQL. Перед выполнением `UPDATE` в реальной базе данных рекомендуется сначала выполнить эквивалентный `SELECT` с тем же условием `WHERE`, чтобы убедиться, что затронуты именно нужные строки.

### 4. Изменение нескольких столбцов одновременно

```sql
UPDATE students
SET age = 22, course = 'Python с нуля'
WHERE name = 'Ерлан';
```

### 5. Удаление данных: DELETE

```sql
DELETE FROM students
WHERE name = 'Ерлан';
```

> **Так же критически важно:** команда `DELETE FROM таблица` без условия `WHERE` удалит **абсолютно все** строки таблицы (хотя саму структуру таблицы при этом не тронет — для полного удаления самой таблицы используется отдельная команда `DROP TABLE`). Это ещё более опасная и необратимая ошибка, чем аналогичная проблема с `UPDATE`.

### 6. Практика безопасной работы с UPDATE/DELETE

Хорошая практика перед выполнением потенциально опасного `UPDATE` или `DELETE` в реальной рабочей базе данных: сначала выполнить `SELECT` с тем же самым условием `WHERE`, внимательно проверить, что в результате отображаются именно те строки, которые предполагается изменить или удалить, и только затем заменить `SELECT *` на `UPDATE`/`DELETE` с тем же условием.

### 7. Практический пример: полный цикл изменения данных

```sql
-- Сначала проверяем, какие строки будут затронуты
SELECT * FROM students WHERE age < 18;

-- Убедившись, что выбраны правильные строки, удаляем их
DELETE FROM students WHERE age < 18;
```

### Резюме урока

- `INSERT INTO таблица (столбцы) VALUES (значения)` добавляет новую запись, можно добавить сразу несколько записей одной командой.
- `UPDATE таблица SET столбец = значение WHERE условие` изменяет существующие записи — отсутствие `WHERE` изменит все строки таблицы.
- `DELETE FROM таблица WHERE условие` удаляет записи — отсутствие `WHERE` удалит все строки; перед выполнением `UPDATE`/`DELETE` рекомендуется сначала проверить затрагиваемые строки через `SELECT` с тем же условием.

## Урок 2.2. Связи между таблицами и объединение: JOIN (INNER, LEFT)
**Время: 60 минут**

### Цель урока
Изучить оператор JOIN для объединения данных из нескольких связанных таблиц в одном запросе.

### 1. Зачем нужен JOIN

В реляционных базах данных данные обычно распределены по нескольким связанным таблицам (Урок 1.1) для избежания дублирования и обеспечения целостности данных. `JOIN` позволяет объединить данные из нескольких таблиц в результате одного запроса на основе связи между ними (обычно через первичный и внешний ключи).

### 2. Пример связанных таблиц

```sql
-- Таблица students
-- id | name
-- 1  | Алина
-- 2  | Данияр

-- Таблица orders
-- id | student_id | book_name
-- 1  |     1      | Гарри Поттер
-- 2  |     1      | Властелин колец
-- 3  |     2      | Дюна
```

### 3. INNER JOIN — только совпадающие записи

`INNER JOIN` возвращает только те строки, для которых найдено соответствие в обеих объединяемых таблицах:

```sql
SELECT students.name, orders.book_name
FROM students
INNER JOIN orders ON students.id = orders.student_id;
```

Результат этого запроса:
```
name    | book_name
Алина   | Гарри Поттер
Алина   | Властелин колец
Данияр  | Дюна
```

Условие `ON students.id = orders.student_id` определяет, по какому именно столбцу связаны две таблицы — обычно это первичный ключ одной таблицы, соответствующий внешнему ключу другой.

### 4. LEFT JOIN — все записи из «левой» таблицы

`LEFT JOIN` возвращает **все** строки из первой («левой») таблицы, даже если для них нет совпадения во второй таблице — в этом случае недостающие значения из второй таблицы заполняются `NULL`:

```sql
SELECT students.name, orders.book_name
FROM students
LEFT JOIN orders ON students.id = orders.student_id;
```

Если, например, в таблице `students` есть студент без единого заказа в таблице `orders`, `INNER JOIN` полностью исключил бы его из результата, а `LEFT JOIN` показал бы его с `NULL` в столбце `book_name`.

### 5. Разница между INNER JOIN и LEFT JOIN: наглядное сравнение

| | INNER JOIN | LEFT JOIN |
|---|---|---|
| Строки без совпадения в правой таблице | Исключаются из результата | Включаются, с NULL в столбцах правой таблицы |
| Типичное применение | Когда нужны только полностью связанные записи | Когда важно видеть все записи «главной» таблицы, даже без связанных данных |

### 6. Использование сокращений (алиасов) для имён таблиц

Для более коротких и читаемых запросов таблицам часто присваивают короткие алиасы:

```sql
SELECT s.name, o.book_name
FROM students s
INNER JOIN orders o ON s.id = o.student_id;
```

### 7. Объединение более чем двух таблиц

`JOIN` можно применять последовательно для объединения данных сразу из нескольких связанных таблиц:

```sql
SELECT s.name, o.book_name, p.amount
FROM students s
INNER JOIN orders o ON s.id = o.student_id
INNER JOIN payments p ON o.id = p.order_id;
```

### Резюме урока

- `JOIN` объединяет данные из нескольких связанных таблиц в результате одного запроса на основе указанного условия связи (обычно первичный/внешний ключ).
- `INNER JOIN` возвращает только строки, для которых найдено совпадение в обеих таблицах, `LEFT JOIN` — все строки левой таблицы, даже без совпадения (с `NULL` вместо недостающих значений).
- Алиасы таблиц (например, `students s`) делают запросы с объединением нескольких таблиц более короткими и читаемыми.

## Урок 2.3. Подзапросы (subqueries)
**Время: 45 минут**

### Цель урока
Изучить подзапросы — механизм использования результата одного запроса внутри другого запроса.

### 1. Что такое подзапрос

Подзапрос (subquery) — это SQL-запрос, вложенный внутрь другого запроса, результат которого используется как часть внешнего запроса. Подзапросы заключаются в круглые скобки.

### 2. Подзапрос в условии WHERE

```sql
SELECT name FROM students
WHERE course IN (
    SELECT course FROM students WHERE age > 25
);
```

Такой запрос сначала (концептуально) выполняет внутренний подзапрос — находит все курсы, где есть студенты старше 25 лет, а затем внешний запрос выбирает имена всех студентов, обучающихся на любом из этих курсов.

### 3. Скалярный подзапрос (возвращающий одно значение)

```sql
SELECT name, age
FROM students
WHERE age > (SELECT AVG(age) FROM students);
```

Этот запрос находит студентов, чей возраст больше среднего возраста всех студентов — подзапрос `(SELECT AVG(age) FROM students)` возвращает единственное числовое значение, с которым и производится сравнение.

### 4. Подзапрос в FROM (производная таблица)

Подзапрос можно использовать вместо обычной таблицы в предложении `FROM` — результат подзапроса трактуется как временная («производная») таблица для дальнейшей работы с ней:

```sql
SELECT course, avg_age
FROM (
    SELECT course, AVG(age) as avg_age
    FROM students
    GROUP BY course
) as course_stats
WHERE avg_age > 20;
```

### 5. Подзапрос vs JOIN: когда что использовать

Многие задачи можно решить как через подзапрос, так и через `JOIN` — часто это вопрос читаемости кода и, в некоторых случаях, эффективности выполнения запроса. Общее эмпирическое правило: `JOIN` обычно предпочтительнее, когда нужно вывести данные сразу из нескольких таблиц одновременно, тогда как подзапросы часто удобнее для проверки условий («есть ли», «сколько», «больше среднего») без необходимости выводить данные из второй таблицы напрямую.

### 6. Оператор EXISTS

```sql
SELECT name FROM students s
WHERE EXISTS (
    SELECT 1 FROM orders o WHERE o.student_id = s.id
);
```

`EXISTS` проверяет, возвращает ли подзапрос хотя бы одну строку, и часто эффективнее, чем `IN` с большим подзапросом, особенно на больших объёмах данных — эта тема подробнее раскрывается в следующем уроке об оптимизации.

### 7. Практический пример: студенты без единого заказа

```sql
SELECT name FROM students
WHERE id NOT IN (
    SELECT student_id FROM orders
);
```

### Резюме урока

- Подзапрос — SQL-запрос, вложенный в круглых скобках внутрь другого запроса, результат которого используется во внешнем запросе.
- Подзапросы могут возвращать одно значение (скалярный подзапрос), список значений (для использования с `IN`) или целую производную таблицу (в `FROM`).
- Оператор `EXISTS` проверяет наличие хотя бы одной подходящей строки в подзапросе и часто эффективнее аналогичной проверки через `IN`.

## Урок 2.4. Индексы и основы оптимизации запросов
**Время: 45 минут**

### Цель урока
Понять, что такое индексы в базах данных, и получить базовые представления об оптимизации SQL-запросов.

### 1. Проблема медленных запросов на больших таблицах

Без дополнительной оптимизации база данных при выполнении запроса с условием `WHERE` вынуждена последовательно просматривать (сканировать) каждую строку таблицы, проверяя, удовлетворяет ли она условию — этот процесс называется полным сканированием таблицы (full table scan) и становится крайне медленным при работе с таблицами, содержащими миллионы записей.

### 2. Что такое индекс

Индекс — специальная структура данных базы данных (обычно основанная на сбалансированном дереве), позволяющая находить нужные строки значительно быстрее, чем при полном сканировании таблицы — примерно так же, как алфавитный указатель в конце книги позволяет быстро найти нужную страницу, не читая всю книгу подряд.

### 3. Создание индекса

```sql
CREATE INDEX idx_students_age ON students(age);
```

Эта команда создаёт индекс по столбцу `age` таблицы `students`, значительно ускоряя запросы, использующие условие `WHERE age = ...` или `WHERE age > ...` для этого столбца.

### 4. На какие столбцы стоит создавать индексы

Индексы особенно полезны для столбцов, часто используемых в условиях `WHERE`, в качестве ключей соединения `JOIN` (обычно первичные и внешние ключи), а также в `ORDER BY`. Первичные ключи, как правило, автоматически индексируются самой СУБД при создании таблицы.

### 5. Компромисс: индексы не бесплатны

Индексы значительно ускоряют операции чтения (`SELECT`), но замедляют операции записи (`INSERT`, `UPDATE`, `DELETE`) — при каждом изменении данных базе данных приходится дополнительно обновлять и все связанные с изменённым столбцом индексы. Кроме того, каждый индекс занимает дополнительное место на диске. По этой причине не стоит бездумно создавать индексы на все подряд столбцы таблицы — только на действительно часто используемые в запросах для фильтрации, соединения и сортировки.

### 6. Анализ выполнения запроса: EXPLAIN

Большинство СУБД предоставляют команду `EXPLAIN`, показывающую, каким образом база данных фактически планирует выполнить конкретный запрос — используется ли при этом индекс или происходит полное сканирование таблицы:

```sql
EXPLAIN SELECT * FROM students WHERE age > 20;
```

Анализ вывода `EXPLAIN` — важный практический навык при диагностике медленных запросов в реальных приложениях, хотя детальная интерпретация результатов выходит за рамки этого вводного мини-курса.

### 7. Базовые рекомендации по написанию эффективных запросов

- Выбирай только действительно необходимые столбцы вместо `SELECT *`, особенно в таблицах с большим количеством столбцов.
- Создавай индексы на столбцы, часто используемые в `WHERE`, `JOIN` и `ORDER BY`.
- Избегай применения функций непосредственно к индексированному столбцу внутри условия `WHERE` (например, `WHERE UPPER(name) = 'АЛИНА'`) — это может помешать базе данных использовать существующий индекс для этого столбца.
- Используй `LIMIT` при предварительном исследовании данных в больших таблицах, чтобы не загружать весь результат целиком без необходимости.

### Резюме урока

- Индекс — специальная структура данных, ускоряющая поиск строк по значению определённого столбца, аналогично алфавитному указателю в книге.
- Индексы ускоряют операции чтения, но замедляют операции записи и занимают дополнительное место — их стоит создавать выборочно, только для действительно часто используемых в запросах столбцов.
- `EXPLAIN` показывает план фактического выполнения запроса базой данных, включая то, используется ли при этом индекс.
