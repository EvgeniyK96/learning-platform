## Урок 1.1. Что такое C++, чем компилируемый язык отличается от интерпретируемого
**Время: 35 минут**

### Цель урока
Понять, что такое C++, его место среди языков программирования, и разобраться в разнице между компилируемыми и интерпретируемыми языками.

### 1. Что такое C++

C++ — это язык программирования общего назначения, созданный Бьёрном Страуструпом в начале 1980-х годов как расширение языка C с поддержкой объектно-ориентированного программирования (отсюда и название: `++` — оператор инкремента в самом C++). C++ сочетает высокую производительность, близкую к «железу», с возможностями высокоуровневого программирования.

### 2. Компилируемые vs интерпретируемые языки

- **Интерпретируемые языки** (Python, JavaScript) выполняются построчно специальной программой-интерпретатором непосредственно во время запуска.
- **Компилируемые языки** (C++, C, Rust) сначала полностью переводятся («компилируются») специальной программой-компилятором в машинный код — набор инструкций, которые процессор может выполнить напрямую, без посредников. Только после этого получившийся исполняемый файл запускается.

```
Python:  код.py → интерпретатор → результат (сразу при запуске)
C++:     код.cpp → компилятор → исполняемый файл (.exe) → запуск → результат
```

### 3. Преимущества компилируемого подхода

- **Производительность.** Скомпилированная программа выполняется значительно быстрее, так как процессор работает напрямую с машинным кодом, без накладных расходов на интерпретацию.
- **Обнаружение ошибок до запуска.** Многие ошибки (например, несоответствие типов) компилятор находит на этапе компиляции, ещё до фактического запуска программы.

### 4. Недостатки компилируемого подхода

- Более долгий цикл разработки: после каждого изменения кода нужно заново компилировать программу перед тестированием.
- Скомпилированный исполняемый файл обычно привязан к конкретной операционной системе и архитектуре процессора (хотя сам код можно перекомпилировать под другую платформу).

### 5. Где применяется C++ сегодня

- **Разработка игр.** Большинство крупных игровых движков (Unreal Engine и др.) написаны на C++ из-за требований к максимальной производительности.
- **Системное программирование.** Операционные системы, драйверы, компиляторы других языков.
- **Высоконагруженные и высокопроизводительные системы.** Финансовые торговые системы, базы данных, обработка больших массивов данных в реальном времени.
- **Встраиваемые системы.** Программирование микроконтроллеров и устройств с ограниченными ресурсами.
- **Инфраструктурное ПО.** Множество крупных проектов с открытым исходным кодом (например, часть Chrome, MySQL) написаны на C++.

### 6. Почему стоит изучать C++, несмотря на сложность

C++ считается более сложным для изучения языком по сравнению с Python или JavaScript, так как требует более глубокого понимания того, как работает память компьютера. Однако именно это делает C++ отличным способом по-настоящему разобраться в принципах работы компьютеров «изнутри» — знания, которые остаются полезными независимо от того, на каких языках предстоит работать в будущем.

### Резюме урока

- C++ — компилируемый язык программирования общего назначения, ориентированный на высокую производительность.
- В компилируемых языках код переводится в машинный код целиком перед запуском, в отличие от построчного выполнения интерпретируемых языков.
- C++ широко применяется в разработке игр, системном программировании и других задачах, требующих максимальной производительности.

## Урок 1.2. Установка компилятора (MinGW/GCC) и VS Code
**Время: 45 минут**

### Цель урока
Установить и настроить компилятор C++ и текстовый редактор для написания и компиляции кода.

### 1. Что такое компилятор

Компилятор — программа, переводящая исходный код C++ (текстовый файл `.cpp`) в исполняемый машинный код. Самые распространённые компиляторы C++: **GCC** (GNU Compiler Collection, кроссплатформенный), **Clang** (используется в том числе в macOS), **MSVC** (компилятор от Microsoft для Windows).

### 2. Установка компилятора на Windows: MinGW

MinGW — сборка компилятора GCC для Windows.

1. Скачать установщик MSYS2 с сайта msys2.org (рекомендуемый современный способ получить MinGW).
2. Установить MSYS2, следуя инструкциям.
3. В открывшемся терминале MSYS2 выполнить:
```bash
pacman -S mingw-w64-ucrt-x86_64-gcc
```
4. Добавить путь к папке `bin` компилятора в переменную окружения PATH операционной системы.

### 3. Установка компилятора на macOS

```bash
xcode-select --install
```

Эта команда установит инструменты командной строки Xcode, включающие компилятор Clang, совместимый с командами GCC.

### 4. Установка компилятора на Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install build-essential
```

### 5. Проверка установки

```bash
g++ --version
```

Если компилятор установлен корректно, команда выведет номер его версии.

### 6. Установка VS Code и расширения C/C++

1. Установить VS Code с сайта code.visualstudio.com (см. также Урок 1.2 курса Python — процесс аналогичен).
2. Открыть раздел расширений (`Ctrl+Shift+X`).
3. Установить расширение «C/C++» от Microsoft — оно добавляет подсветку синтаксиса, автодополнение и поддержку отладки.

### 7. Первая компиляция

Создай файл `hello.cpp` со следующим содержимым:

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
```

Скомпилируй и запусти через терминал:

```bash
g++ hello.cpp -o hello
./hello        # на Windows: hello.exe
```

Команда `g++ hello.cpp -o hello` компилирует файл `hello.cpp` в исполняемый файл с именем `hello`, который затем запускается отдельной командой.

### Резюме урока

- Для компиляции C++ кода нужен отдельно установленный компилятор (MinGW/GCC на Windows, Xcode Command Line Tools на macOS, build-essential на Linux).
- VS Code с расширением C/C++ обеспечивает удобную среду разработки с подсветкой синтаксиса и автодополнением.
- Компиляция и запуск программы — два раздельных шага: `g++ файл.cpp -o имя_программы`, затем запуск получившегося исполняемого файла.

## Урок 1.3. Первая программа: компиляция и запуск Hello, World!
**Время: 35 минут**

### Цель урока
Разобрать структуру первой программы на C++ построчно и закрепить процесс компиляции и запуска.

### 1. Разбор программы Hello, World!

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
```

### 2. Директива #include

`#include <iostream>` — это директива препроцессора, подключающая стандартную библиотеку ввода-вывода `iostream`, которая содержит функциональность для работы с `std::cout` (вывод) и `std::cin` (ввод, см. Урок 2.2). Без этого подключения использование `std::cout` привело бы к ошибке компиляции.

### 3. Функция main()

```cpp
int main() {
    // код программы
    return 0;
}
```

Каждая программа на C++ обязательно должна содержать функцию `main()` — именно с неё начинается выполнение программы. Тип `int` перед `main` означает, что функция возвращает целое число — код завершения программы операционной системе (`0` традиционно означает «программа завершилась успешно»).

### 4. Вывод текста: std::cout

```cpp
std::cout << "Hello, World!" << std::endl;
```

`std::cout` («console output») — стандартный поток вывода. Оператор `<<` («вставить в поток») передаёт данные в поток вывода. `std::endl` завершает строку и переносит курсор на новую строку (аналог `\n`).

### 5. Пространство имён std

Приставка `std::` перед `cout` и `endl` указывает, что эти имена принадлежат пространству имён `std` (standard) — стандартной библиотеке C++. Такая приставка помогает избежать конфликтов имён между разными библиотеками.

Альтернативный способ — подключить всё пространство имён `std` целиком через `using namespace std;`, что позволяет писать просто `cout` без приставки, но в реальных проектах явное указание `std::` считается более безопасной практикой.

### 6. Точка с запятой

В отличие от Python, в C++ каждая инструкция обязательно завершается точкой с запятой `;`. Пропуск точки с запятой — одна из самых частых ошибок у новичков, приводящая к ошибке компиляции.

### 7. Компиляция и запуск

```bash
g++ hello.cpp -o hello
./hello
```

### Резюме урока

- Программа на C++ обязательно должна содержать функцию `main()`, с которой начинается выполнение.
- `#include <iostream>` подключает стандартную библиотеку ввода-вывода, необходимую для `std::cout`.
- Каждая инструкция в C++ обязательно завершается точкой с запятой — пропуск её вызывает ошибку компиляции.

## Урок 1.4. Структура программы на C++: #include, main(), точка с запятой
**Время: 30 минут**

### Цель урока
Закрепить понимание базовой структуры программы на C++ и типичных синтаксических требований языка.

### 1. Общая структура программы на C++

```cpp
#include <iostream>    // 1. Подключение библиотек

using namespace std;    // 2. (опционально) Использование пространства имён

int main() {              // 3. Точка входа
    // 4. Тело программы
    cout << "Программа работает" << endl;
    return 0;               // 5. Код завершения
}
```

### 2. Блоки кода и фигурные скобки

В отличие от Python, где границы блоков кода задаются отступами, в C++ блоки кода обозначаются фигурными скобками `{}`. Отступы при этом остаются важными для читаемости кода, но формально не влияют на его выполнение:

```cpp
int main() {
    cout << "Строка 1" << endl;
    cout << "Строка 2" << endl;
    return 0;
}
```

### 3. Комментарии

```cpp
// Однострочный комментарий

/*
Многострочный
комментарий
*/

int main() {
    // Это тоже комментарий
    return 0;
}
```

### 4. Множественные подключения библиотек

```cpp
#include <iostream>
#include <string>
#include <vector>

int main() {
    cout << "Несколько библиотек подключено" << endl;
    return 0;
}
```

### 5. Чувствительность к регистру

C++ чувствителен к регистру символов — `Main`, `MAIN` и `main` являются тремя разными идентификаторами, и только `main` (в нижнем регистре) распознаётся компилятором как точка входа программы.

### 6. Типичные ошибки структуры программы у новичков

```cpp
#include <iostream>

int main() {
    cout << "Ошибка: cout без std:: и без using namespace std" << endl;
    return 0
}
// Ошибка 1: отсутствует std:: перед cout (если не подключено using namespace std)
// Ошибка 2: пропущена точка с запятой после return 0
// Ошибка 3: не хватает закрывающей фигурной скобки
```

### 7. Компиляция с предупреждениями

Хорошей практикой считается компилировать код с включёнными предупреждениями компилятора — это помогает заранее обнаружить потенциальные проблемы:

```bash
g++ -Wall hello.cpp -o hello
```

Флаг `-Wall` («all warnings») включает вывод большинства распространённых предупреждений компилятора.

### Резюме урока

- Типичная структура программы на C++: подключение библиотек → (опционально) using namespace → функция main() → тело программы → return.
- Блоки кода в C++ обозначаются фигурными скобками `{}`, а не отступами, как в Python.
- Компиляция с флагом `-Wall` помогает заранее обнаружить потенциальные проблемы в коде через предупреждения компилятора.
