## Урок 8.1. Стек и куча (stack vs heap)
**Время: 35 минут**

### Цель урока
Понять два основных вида памяти, используемых программами на C++, и разницу между ними.

### 1. Два вида памяти

Программа на C++ использует два принципиально разных вида памяти для хранения данных во время выполнения: **стек (stack)** и **куча (heap)**.

### 2. Стек (stack)

Стек — область памяти, автоматически управляемая компилятором, где хранятся локальные переменные функций. Память в стеке выделяется и освобождается автоматически при входе в функцию/блок кода и выходе из него.

```cpp
void someFunction() {
    int x = 5;          // x создаётся в стеке
    double y = 3.14;    // y создаётся в стеке
}   // при выходе из функции x и y автоматически уничтожаются
```

### 3. Особенности стека

- **Быстрый доступ.** Выделение и освобождение памяти в стеке происходит очень быстро.
- **Автоматическое управление.** Не требует ручного освобождения памяти разработчиком.
- **Ограниченный размер.** Размер стека ограничен (обычно несколько мегабайт) — слишком глубокая рекурсия (Урок 4.4) или слишком большие локальные переменные могут привести к переполнению стека.
- **Ограниченное время жизни.** Переменные в стеке существуют только до выхода из блока кода, в котором они объявлены.

### 4. Куча (heap)

Куча — область памяти, управляемая вручную разработчиком через операторы `new` и `delete` (подробно в Уроке 8.2). Память в куче не освобождается автоматически при выходе из функции.

```cpp
int* createOnHeap() {
    int *pointer = new int(42);   // выделение памяти в куче
    return pointer;                  // память останется существовать после выхода из функции
}
```

### 5. Особенности кучи

- **Гибкость.** Память можно выделять и освобождать в любой момент, независимо от структуры вызовов функций.
- **Больший доступный размер.** Куча значительно больше стека, ограничена в основном общим объёмом доступной оперативной памяти.
- **Ручное управление.** Разработчик обязан сам освобождать память через `delete`, когда она больше не нужна — забытое освобождение приводит к утечке памяти (Урок 8.3).
- **Более медленный доступ** по сравнению со стеком, из-за более сложного механизма управления.

### 6. Сравнительная таблица

| | Стек (stack) | Куча (heap) |
|---|---|---|
| Управление | Автоматическое | Ручное (new/delete) |
| Скорость | Быстрая | Медленнее |
| Размер | Ограничен (обычно МБ) | Большой (ограничен объёмом ОЗУ) |
| Время жизни данных | До выхода из блока кода | Пока явно не освобождена |
| Типичное использование | Локальные переменные, параметры функций | Динамически создаваемые объекты, данные переменного размера |

### 7. Наглядный пример разницы

```cpp
void stackExample() {
    int number = 42;   // в стеке, автоматически уничтожится при выходе из функции
}

void heapExample() {
    int *number = new int(42);   // в куче, останется в памяти, пока явно не удалим
    delete number;                    // теперь корректно освобождаем память
}
```

### Резюме урока

- Стек — быстрая, автоматически управляемая память для локальных переменных с ограниченным временем жизни и размером.
- Куча — более гибкая, но требующая ручного управления память, используемая для динамически создаваемых данных через `new`/`delete`.
- Понимание разницы между стеком и кучей — фундаментальная основа для дальнейшего изучения динамического выделения памяти в C++.

## Урок 8.2. Динамическое выделение памяти: new и delete
**Время: 45 минут**

### Цель урока
Освоить операторы new и delete для ручного управления памятью в куче.

### 1. Оператор new

`new` выделяет память в куче для объекта указанного типа и возвращает указатель на неё:

```cpp
int *pointer = new int;        // выделение памяти для одного int
*pointer = 42;                    // запись значения по этому адресу

int *pointerWithValue = new int(42);   // выделение с одновременной инициализацией

cout << *pointer << endl;   // 42
```

### 2. Оператор delete

Когда динамически выделенная память больше не нужна, её необходимо явно освободить с помощью `delete` — иначе память останется занятой до завершения работы всей программы (утечка памяти):

```cpp
int *pointer = new int(42);
cout << *pointer << endl;
delete pointer;                   // освобождение памяти
pointer = nullptr;               // хорошая практика: обнулить указатель после delete
```

### 3. Динамическое выделение массивов: new[] и delete[]

Для массивов используется отдельный синтаксис с квадратными скобками, важно использовать именно парный `delete[]`, а не обычный `delete`:

```cpp
int size = 5;
int *array = new int[size];    // динамический массив из 5 элементов

for (int i = 0; i < size; i++) {
    array[i] = i * 10;
}

for (int i = 0; i < size; i++) {
    cout << array[i] << " ";
}

delete[] array;    // обязательно delete[] для массивов, а не просто delete
```

### 4. Динамическое выделение объектов класса

```cpp
class Car {
public:
    string brand;
    Car(string b) : brand(b) {}
};

int main() {
    Car *myCar = new Car("Toyota");
    cout << myCar->brand << endl;   // -> используется для доступа к членам через указатель
    delete myCar;
    return 0;
}
```

Обрати внимание на оператор `->` для доступа к членам объекта через указатель — это сокращённая запись для `(*myCar).brand`.

### 5. Зачем вообще нужна динамическая память, если есть стек

Динамическое выделение памяти необходимо, когда: размер данных неизвестен заранее (например, зависит от пользовательского ввода), объект должен «пережить» функцию, в которой он был создан, или требуется создать очень большой объект, который не поместился бы в ограниченный размер стека.

### 6. Практический пример: динамический массив по размеру, введённому пользователем

```cpp
int size;
cout << "Сколько чисел ввести? ";
cin >> size;

int *numbers = new int[size];

for (int i = 0; i < size; i++) {
    cout << "Введите число: ";
    cin >> numbers[i];
}

int sum = 0;
for (int i = 0; i < size; i++) {
    sum += numbers[i];
}

cout << "Сумма: " << sum << endl;
delete[] numbers;
```

Это невозможно реализовать с обычным (статическим) массивом, так как его размер должен быть известен на этапе компиляции, а не определяться во время выполнения программы.

### Резюме урока

- `new` выделяет память в куче и возвращает указатель на неё, `delete` — освобождает эту память.
- Для динамических массивов используется парный синтаксис `new[]`/`delete[]`, а не обычный `new`/`delete`.
- Динамическая память необходима, когда размер данных неизвестен на этапе компиляции или объект должен существовать дольше, чем функция, в которой он создан.

## Урок 8.3. Проблема утечек памяти
**Время: 35 минут**

### Цель урока
Понять, что такое утечка памяти, и научиться распознавать типичные ситуации, приводящие к ней.

### 1. Что такое утечка памяти

Утечка памяти (memory leak) происходит, когда программа выделяет память в куче через `new`, но никогда не освобождает её через `delete` — эта память остаётся «занятой» до завершения работы всей программы, даже если сама программа больше не имеет к ней доступа (потерян единственный указатель на неё).

```cpp
void leakyFunction() {
    int *pointer = new int(42);
    // забыли delete pointer!
}   // указатель pointer уничтожается при выходе из функции, но память в куче остаётся занятой навсегда
```

### 2. Почему утечки памяти опасны

При однократном вызове небольшая утечка может показаться незначительной, но в программах, работающих длительное время (серверы, игры, встроенные системы) и постоянно выделяющих новую память (например, в цикле), утечки постепенно накапливаются и в итоге могут привести к полному исчерпанию доступной памяти и аварийному завершению программы.

### 3. Типичный сценарий утечки в цикле

```cpp
for (int i = 0; i < 1000; i++) {
    int *data = new int(i);
    // используем data...
    // забыли delete data - память "утекает" 1000 раз!
}
```

### 4. Утечка при потере указателя

```cpp
int *pointer = new int(42);
pointer = new int(100);   // старая память (42) потеряна безвозвратно - утечка! Больше нет указателя на неё
delete pointer;              // освобождает только вторую выделенную область (100)
```

### 5. Правило: каждому new должен соответствовать один delete

Золотое правило управления памятью в «классическом» C++: на каждый вызов `new` должен приходиться ровно один соответствующий вызов `delete` (или `new[]`/`delete[]` для массивов). Отслеживание этого соответствия вручную в сложных программах — источник значительной части ошибок, что и стало причиной появления «умных указателей», рассматриваемых в следующем уроке.

### 6. Инструменты для обнаружения утечек памяти (обзорно)

Существуют специализированные инструменты (Valgrind на Linux, встроенные средства анализа памяти в некоторых IDE), позволяющие обнаружить утечки памяти в программе, которые сложно найти простым просмотром кода — эти инструменты активно используются в профессиональной разработке на C++.

### 7. Практический пример утечки и её исправления

```cpp
// Код с утечкой:
void processData(int size) {
    int *data = new int[size];
    // ... обработка данных ...
    // забыли delete[] data;
}

// Исправленный код:
void processDataFixed(int size) {
    int *data = new int[size];
    // ... обработка данных ...
    delete[] data;   // память корректно освобождена
}
```

### Резюме урока

- Утечка памяти происходит, когда выделенная через `new` память никогда не освобождается через `delete`.
- Накопление утечек в долго работающих программах может привести к исчерпанию памяти и аварийному завершению.
- Правило «каждому new — свой delete» — базовая, но требующая внимательности практика управления памятью в классическом C++.

## Урок 8.4. Умные указатели: unique_ptr, shared_ptr
**Время: 55 минут**

### Цель урока
Освоить умные указатели — современный механизм C++ для автоматического управления динамической памятью.

### 1. Идея умных указателей

Умные указатели (smart pointers) — объекты стандартной библиотеки C++ (начиная с C++11), «оборачивающие» обычный указатель и автоматически освобождающие связанную с ним память, когда она становится не нужна — избавляя разработчика от необходимости вручную вызывать `delete` и, соответственно, от риска забыть это сделать (Урок 8.3).

### 2. unique_ptr

`unique_ptr` — умный указатель, обеспечивающий единоличное владение объектом: в каждый момент времени только один `unique_ptr` может «владеть» конкретным объектом в памяти.

```cpp
#include <memory>

void example() {
    unique_ptr<int> pointer = make_unique<int>(42);
    cout << *pointer << endl;   // 42
}   // память автоматически освобождается здесь, delete вызывать не нужно!
```

`make_unique<T>(аргументы)` — рекомендуемый способ создания `unique_ptr`, доступный начиная с C++14.

### 3. unique_ptr нельзя копировать

```cpp
unique_ptr<int> p1 = make_unique<int>(42);
unique_ptr<int> p2 = p1;   // ОШИБКА КОМПИЛЯЦИИ: unique_ptr нельзя копировать
```

Это ограничение — часть самой идеи «единоличного владения»: если бы копирование было разрешено, стало бы неясно, какой именно `unique_ptr` отвечает за освобождение памяти.

### 4. Передача владения через std::move

Владение объектом можно передать от одного `unique_ptr` к другому явно через `std::move`, при этом исходный указатель становится пустым:

```cpp
#include <utility>

unique_ptr<int> p1 = make_unique<int>(42);
unique_ptr<int> p2 = std::move(p1);   // владение перешло к p2

cout << (p1 == nullptr) << endl;   // 1 (p1 теперь пуст)
cout << *p2 << endl;                  // 42
```

### 5. shared_ptr

`shared_ptr` — умный указатель, поддерживающий совместное владение объектом несколькими указателями одновременно. Внутри `shared_ptr` ведёт подсчёт ссылок (reference counting) — объект автоматически удаляется, когда счётчик ссылок на него достигает нуля (то есть больше ни один `shared_ptr` на него не ссылается):

```cpp
#include <memory>

shared_ptr<int> p1 = make_shared<int>(42);
shared_ptr<int> p2 = p1;   // копирование разрешено! теперь оба указателя владеют одним объектом

cout << p1.use_count() << endl;   // 2 (два указателя ссылаются на объект)
```

### 6. Класс Car с умными указателями: практический пример

```cpp
#include <memory>
#include <vector>

class Car {
public:
    string brand;
    Car(string b) : brand(b) {
        cout << "Создан автомобиль: " << brand << endl;
    }
    ~Car() {
        cout << "Уничтожен автомобиль: " << brand << endl;
    }
};

int main() {
    vector<unique_ptr<Car>> garage;
    
    garage.push_back(make_unique<Car>("Toyota"));
    garage.push_back(make_unique<Car>("BMW"));
    
    for (const auto &car : garage) {
        cout << car->brand << endl;
    }
    
    return 0;   // все объекты Car автоматически и корректно уничтожаются здесь, без ручного delete
}
```

### 7. Почему умные указатели — рекомендуемая практика в современном C++

Умные указатели значительно снижают риск утечек памяти и других ошибок управления памятью, автоматизируя вызов `delete` в нужный момент. В современном C++ рекомендуется по умолчанию использовать умные указатели вместо «голых» указателей с ручным `new`/`delete`, обращаясь к последним лишь в специфических ситуациях, требующих более тонкого контроля.

### Резюме урока

- `unique_ptr` обеспечивает единоличное владение объектом и не может быть скопирован, только перемещён через `std::move`.
- `shared_ptr` поддерживает совместное владение несколькими указателями через подсчёт ссылок — объект удаляется автоматически, когда последняя ссылка на него исчезает.
- В современном C++ умные указатели — рекомендуемая практика, значительно снижающая риск утечек памяти по сравнению с ручным управлением через `new`/`delete`.

## Урок 8.5. Практика: работа с динамической памятью
**Время: 45 минут**

### Цель урока
Закрепить материал модуля, решив несколько практических задач с использованием динамической памяти и умных указателей.

### 1. Задача 1. Динамический массив переменного размера

```cpp
#include <iostream>
using namespace std;

int main() {
    int size;
    cout << "Введите размер массива: ";
    cin >> size;
    
    int *numbers = new int[size];
    
    for (int i = 0; i < size; i++) {
        numbers[i] = i * i;
    }
    
    for (int i = 0; i < size; i++) {
        cout << numbers[i] << " ";
    }
    cout << endl;
    
    delete[] numbers;
    return 0;
}
```

### 2. Задача 2. Функция, создающая объект в куче через unique_ptr

```cpp
#include <memory>

unique_ptr<int> createNumber(int value) {
    return make_unique<int>(value);
}

int main() {
    unique_ptr<int> num = createNumber(42);
    cout << *num << endl;
    return 0;
}
```

### 3. Задача 3. Класс с динамическим массивом внутри (обзор проблемы «ручного» подхода)

```cpp
class IntArray {
private:
    int *data;
    int size;
    
public:
    IntArray(int s) : size(s) {
        data = new int[size];
    }
    
    void set(int index, int value) {
        if (index >= 0 && index < size) {
            data[index] = value;
        }
    }
    
    int get(int index) {
        return data[index];
    }
    
    ~IntArray() {          // деструктор - освобождает память при уничтожении объекта
        delete[] data;
    }
};

int main() {
    IntArray arr(5);
    arr.set(0, 100);
    cout << arr.get(0) << endl;
    return 0;
}   // деструктор автоматически вызовется здесь, освободив память
```

Деструктор (метод с именем `~ИмяКласса()`) автоматически вызывается при уничтожении объекта — важный механизм для классов, самостоятельно управляющих динамической памятью.

### 4. Задача 4. Список объектов через vector умных указателей

```cpp
#include <memory>
#include <vector>

class Task {
public:
    string title;
    Task(string t) : title(t) {}
};

int main() {
    vector<shared_ptr<Task>> tasks;
    
    tasks.push_back(make_shared<Task>("Купить молоко"));
    tasks.push_back(make_shared<Task>("Сделать домашнее задание"));
    
    for (const auto &task : tasks) {
        cout << task->title << endl;
    }
    
    return 0;
}
```

### 5. Задача 5. Сравнение подхода: голый указатель vs умный указатель

```cpp
// Рискованный подход (легко забыть delete)
void riskyApproach() {
    int *data = new int[100];
    // ... работа с данными ...
    delete[] data;   // легко забыть на реальном сложном коде с множеством ветвлений
}

// Безопасный подход
#include <vector>
void safeApproach() {
    vector<int> data(100);   // vector сам управляет своей памятью автоматически
    // ... работа с данными ...
}   // память освобождается автоматически, без явного delete
```

### 6. Общая рекомендация современного C++

В большинстве повседневных задач вместо «голых» указателей с ручным `new`/`delete` стоит по умолчанию использовать: `vector` вместо динамических массивов (Модуль 6), умные указатели (`unique_ptr`/`shared_ptr`) вместо голых указателей на отдельные объекты. Прямое использование `new`/`delete` оставляют для специфических низкоуровневых задач или при изучении того, как эти механизмы устроены «под капотом».

### Резюме урока

- Практика с динамической памятью демонстрирует как «классический» подход через `new`/`delete`, так и современные безопасные альтернативы.
- Деструктор класса — специальный метод, автоматически вызываемый при уничтожении объекта, часто используемый для освобождения динамически выделенных ресурсов.
