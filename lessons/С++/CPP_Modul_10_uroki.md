## Урок 10.1. Планирование проекта (например, консольная игра или учётная система)
**Время: 40 минут**

### Цель урока
Научиться формулировать техническое задание для итогового проекта на C++.

### 1. Выбор темы итогового проекта

Учитывая освоенные темы курса (ООП, STL, работа с памятью, файлы, исключения), хорошими темами для итогового проекта являются:

- **Консольная игра** (например, «Крестики-нолики», «Виселица», текстовый квест) с использованием классов для игровых объектов.
- **Система учёта** (библиотека, склад, студенты) с сохранением данных в файл и полной обработкой ошибок.
- **Простой калькулятор матриц** с использованием двумерных массивов/векторов и перегрузки операторов.
- **Телефонный справочник** с использованием `map` для быстрого поиска и файлового хранения данных.

### 2. Структура технического задания

1. **Задача.** Какую проблему решает программа?
2. **Классы.** Какие классы потребуются, как они связаны (наследование, композиция)?
3. **Структуры данных.** Какие контейнеры STL будут использованы?
4. **Хранение данных.** Будут ли данные сохраняться в файл между запусками?
5. **Обработка ошибок.** Какие исключения предполагается обрабатывать?

### 3. Пример технического задания

**Проект:** Система учёта библиотеки

**Задача:** Позволить библиотекарю вести учёт книг: добавление, поиск, выдача и возврат книг.

**Классы:** `Book` (книга: название, автор, доступность), `Library` (управление коллекцией книг).

**Структуры данных:** `vector<Book>` для хранения книг, `map<string, int>` для быстрого поиска книги по названию.

**Хранение данных:** CSV-файл `library.csv`, загружаемый при старте и сохраняемый при каждом изменении.

**Обработка ошибок:** попытка выдать уже выданную книгу, поиск несуществующей книги, ошибки при работе с файлом.

### 4. Оценка объёма задачи

Итоговый проект должен быть реализуем за отведённое время модуля (около 5 часов) и явно опираться минимум на 4–5 тем курса: классы и ООП, STL-контейнеры, работа с файлами, обработка исключений, и по возможности — умные указатели.

### 5. Планирование структуры файлов

Для итогового проекта рекомендуется разделение на несколько файлов, а не один большой `main.cpp`:

```
library_project/
├── main.cpp        # точка входа, меню программы
├── Book.h            # объявление класса Book
├── Book.cpp           # реализация класса Book
├── Library.h          # объявление класса Library
└── Library.cpp         # реализация класса Library
```

### 6. Домашнее задание к уроку

Составь техническое задание для своего итогового проекта по образцу из пункта 3: задача, классы, структуры данных, хранение данных, обработка ошибок.

### Резюме урока

- Итоговый проект должен явно опираться на реальные темы курса: ООП, STL, файлы, исключения.
- Хорошее техническое задание заранее определяет структуру классов и используемые контейнеры STL до начала кодирования.
- Разделение проекта на несколько файлов (`.h`/`.cpp`) — стандартная практика организации кода в реальных C++ проектах.

## Урок 10.2. Проектирование структуры классов
**Время: 60 минут**

### Цель урока
Спроектировать структуру классов итогового проекта до начала написания полной реализации.

### 1. От идеи к структуре классов

После формулировки технического задания следующий шаг — детально спроектировать классы: их атрибуты, методы и связи между ними.

### 2. Заголовочные файлы (.h) и файлы реализации (.cpp)

В реальных C++ проектах принято разделять **объявление** класса (заголовочный файл `.h`) и его **реализацию** (файл `.cpp`) — это позволяет подключать класс в разных частях программы, не дублируя код реализации.

```cpp
// Book.h
#ifndef BOOK_H
#define BOOK_H

#include <string>
using namespace std;

class Book {
private:
    string title;
    string author;
    bool available;
    
public:
    Book(string t, string a);
    
    string getTitle() const;
    string getAuthor() const;
    bool isAvailable() const;
    void setAvailable(bool status);
};

#endif
```

### 3. Разбор защиты от повторного включения (include guard)

Конструкция `#ifndef BOOK_H / #define BOOK_H / ... / #endif` называется include guard — она предотвращает ошибку повторного объявления класса, если один и тот же заголовочный файл случайно подключается несколько раз в разных частях программы (например, косвенно через несколько других заголовочных файлов).

### 4. Реализация в отдельном .cpp файле

```cpp
// Book.cpp
#include "Book.h"

Book::Book(string t, string a) : title(t), author(a), available(true) {}

string Book::getTitle() const { return title; }
string Book::getAuthor() const { return author; }
bool Book::isAvailable() const { return available; }
void Book::setAvailable(bool status) { available = status; }
```

Модификатор `const` после списка параметров метода (`getTitle() const`) указывает, что метод не изменяет состояние объекта — хорошая практика для методов, только читающих данные (геттеров).

### 5. Проектирование класса Library

```cpp
// Library.h
#ifndef LIBRARY_H
#define LIBRARY_H

#include "Book.h"
#include <vector>
#include <string>

class Library {
private:
    vector<Book> books;
    
public:
    void addBook(string title, string author);
    bool borrowBook(string title);
    bool returnBook(string title);
    void showAllBooks() const;
    void saveToFile(string filename) const;
    void loadFromFile(string filename);
};

#endif
```

### 6. Схема взаимодействия классов

```
main.cpp (меню, пользовательский ввод)
   │
   └──> Library (управление коллекцией)
              │
              └──> Book (отдельная запись о книге)
```

### 7. Компиляция многофайлового проекта

При компиляции проекта из нескольких `.cpp` файлов необходимо передать компилятору все файлы реализации сразу:

```bash
g++ main.cpp Book.cpp Library.cpp -o library_app
./library_app
```

### Резюме урока

- Реальные C++ проекты разделяют объявление класса (`.h`) и его реализацию (`.cpp`), что требует использования include guard для предотвращения повторного объявления.
- Проектирование структуры классов и их связей до начала полной реализации помогает избежать архитектурных проблем на позднем этапе разработки.
- Компиляция многофайлового проекта требует передачи компилятору всех участвующих `.cpp` файлов одновременно.

## Урок 10.3. Реализация основного функционала
**Время: 90 минут**

### Цель урока
Реализовать рабочую версию итогового проекта на основе технического задания и структуры классов.

### 1. Полная реализация класса Library

```cpp
// Library.cpp
#include "Library.h"
#include <iostream>
#include <fstream>
#include <algorithm>
using namespace std;

void Library::addBook(string title, string author) {
    books.push_back(Book(title, author));
    cout << "Книга добавлена: " << title << endl;
}

bool Library::borrowBook(string title) {
    for (auto &book : books) {
        if (book.getTitle() == title) {
            if (!book.isAvailable()) {
                cout << "Книга уже выдана" << endl;
                return false;
            }
            book.setAvailable(false);
            cout << "Книга выдана: " << title << endl;
            return true;
        }
    }
    cout << "Книга не найдена" << endl;
    return false;
}

bool Library::returnBook(string title) {
    for (auto &book : books) {
        if (book.getTitle() == title) {
            book.setAvailable(true);
            cout << "Книга возвращена: " << title << endl;
            return true;
        }
    }
    cout << "Книга не найдена" << endl;
    return false;
}

void Library::showAllBooks() const {
    cout << "\n--- Список книг ---" << endl;
    for (const auto &book : books) {
        cout << book.getTitle() << " - " << book.getAuthor()
             << " [" << (book.isAvailable() ? "доступна" : "выдана") << "]" << endl;
    }
}
```

### 2. Реализация сохранения и загрузки

```cpp
void Library::saveToFile(string filename) const {
    ofstream outFile(filename);
    if (!outFile.is_open()) {
        cout << "Ошибка сохранения файла" << endl;
        return;
    }
    
    for (const auto &book : books) {
        outFile << book.getTitle() << ";" << book.getAuthor() << ";"
                << book.isAvailable() << endl;
    }
    outFile.close();
}

void Library::loadFromFile(string filename) {
    ifstream inFile(filename);
    if (!inFile.is_open()) {
        return;   // файла ещё нет - это нормально при первом запуске
    }
    
    string line;
    while (getline(inFile, line)) {
        size_t pos1 = line.find(';');
        size_t pos2 = line.find(';', pos1 + 1);
        
        string title = line.substr(0, pos1);
        string author = line.substr(pos1 + 1, pos2 - pos1 - 1);
        bool available = stoi(line.substr(pos2 + 1));
        
        Book book(title, author);
        book.setAvailable(available);
        books.push_back(book);
    }
    inFile.close();
}
```

### 3. Главный файл программы с меню

```cpp
// main.cpp
#include "Library.h"
#include <iostream>
using namespace std;

int main() {
    Library library;
    string filename = "library.csv";
    library.loadFromFile(filename);
    
    int choice;
    do {
        cout << "\n1. Показать книги\n2. Добавить книгу\n3. Выдать книгу\n4. Вернуть книгу\n5. Выход\n";
        cout << "Выберите действие: ";
        cin >> choice;
        cin.ignore();
        
        string title, author;
        
        switch (choice) {
            case 1:
                library.showAllBooks();
                break;
            case 2:
                cout << "Название: ";
                getline(cin, title);
                cout << "Автор: ";
                getline(cin, author);
                library.addBook(title, author);
                library.saveToFile(filename);
                break;
            case 3:
                cout << "Название книги для выдачи: ";
                getline(cin, title);
                library.borrowBook(title);
                library.saveToFile(filename);
                break;
            case 4:
                cout << "Название книги для возврата: ";
                getline(cin, title);
                library.returnBook(title);
                library.saveToFile(filename);
                break;
            case 5:
                cout << "До свидания!" << endl;
                break;
            default:
                cout << "Некорректный выбор" << endl;
        }
    } while (choice != 5);
    
    return 0;
}
```

### 4. Компиляция и первый запуск

```bash
g++ main.cpp Book.cpp Library.cpp -o library_app
./library_app
```

### 5. Постепенная проверка функционала

Рекомендуется проверять каждую реализованную функцию сразу после её написания (например, сначала только добавление книг, затем сохранение/загрузку, затем выдачу/возврат), а не откладывать всю проверку на самый конец разработки.

### Резюме урока

- Реализация итогового проекта на C++ включает как логику классов (`Library.cpp`, `Book.cpp`), так и пользовательский интерфейс в главном файле (`main.cpp`).
- Компиляция многофайлового проекта требует явного перечисления всех `.cpp` файлов при вызове компилятора.
- Постепенная проверка каждой части функционала сразу после реализации существенно упрощает поиск и исправление ошибок.

## Урок 10.4. Тестирование и отладка
**Время: 60 минут**

### Цель урока
Проверить работу итогового проекта на разных сценариях и исправить найденные ошибки.

### 1. Ручное тестирование по сценариям

- Добавление новой книги — сохраняется ли она корректно в файл?
- Выдача и возврат книги — корректно ли обновляется статус доступности?
- Повторный запуск программы — загружаются ли ранее сохранённые книги?
- Попытка выдать уже выданную книгу — обрабатывается ли эта ситуация корректно?
- Попытка работы с несуществующей книгой — не приводит ли это к падению программы?

### 2. Проверка граничных случаев

- Что произойдёт при пустом названии книги?
- Что произойдёт, если файл данных повреждён или имеет неожиданный формат?
- Что произойдёт при первом запуске, когда файла с данными ещё не существует?
- Корректно ли обрабатывается некорректный пользовательский ввод в меню (например, буквы вместо номера пункта)?

### 3. Типичные ошибки в C++ проектах, которые стоит проверить

```cpp
// Утечка памяти при использовании new без соответствующего delete
// Выход за границы массива/вектора без проверки
// Обращение к разыменованию nullptr-указателя
// Забытая проверка is_open() при работе с файлами
// Смешивание cin >> и getline() без cin.ignore()
```

### 4. Использование отладчика (обзорно)

Расширение C/C++ для VS Code (установленное в Уроке 1.2) включает встроенный отладчик, позволяющий устанавливать точки останова и пошагово выполнять код, наблюдая за значениями переменных — значительно более эффективный способ находить сложные ошибки по сравнению с расстановкой множества `cout` для отладки.

### 5. Компиляция с дополнительными проверками

При отладке полезно компилировать код с дополнительными флагами, помогающими обнаружить проблемы:

```bash
g++ -Wall -Wextra -g main.cpp Book.cpp Library.cpp -o library_app
```

Флаг `-g` добавляет отладочную информацию, необходимую для полноценной работы отладчика, `-Wextra` включает дополнительные предупреждения компилятора помимо базового набора `-Wall`.

### 6. Чек-лист готовности проекта

- [ ] Все функции из технического задания реализованы.
- [ ] Программа не завершается аварийно при некорректном пользовательском вводе.
- [ ] Данные корректно сохраняются и загружаются между запусками программы.
- [ ] Нет очевидных утечек памяти (все `new` имеют соответствующий `delete`, либо используются умные указатели/контейнеры STL).
- [ ] Код скомпилирован без предупреждений при флагах `-Wall -Wextra`.
- [ ] Проект разделён на логические файлы (`.h`/`.cpp`), а не находится в одном файле.

### Резюме урока

- Тестирование C++ проекта должно включать проверку типичных ошибок языка: утечки памяти, выход за границы, некорректная работа с указателями.
- Компиляция с флагами `-Wall -Wextra -g` помогает заранее обнаружить потенциальные проблемы и подготовить код к полноценной отладке.
- Чек-лист готовности — удобный способ систематически проверить проект перед финальной презентацией.
