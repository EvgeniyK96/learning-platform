## Урок 6.1. Контейнер vector
**Время: 50 минут**

### Цель урока
Изучить std::vector — динамический массив стандартной библиотеки C++, решающий главное ограничение обычных массивов (Модуль 5).

### 1. Проблема фиксированного размера обычных массивов

Как было отмечено в Уроке 5.1, размер обычного C++-массива фиксирован при объявлении и не может изменяться. STL (Standard Template Library — стандартная библиотека шаблонов) предоставляет `vector` — контейнер, который может динамически изменять размер во время выполнения программы, аналогично спискам в Python.

### 2. Объявление и инициализация vector

```cpp
#include <vector>

vector<int> numbers;                     // пустой vector
vector<int> scores = {90, 85, 78, 92};   // с начальными значениями
vector<string> names(5);                    // vector из 5 пустых строк
```

Обрати внимание на синтаксис `vector<int>` — угловые скобки указывают тип элементов, которые будет хранить контейнер (это пример использования шаблонов, generic-механизма C++).

### 3. Добавление и удаление элементов

```cpp
vector<int> numbers = {1, 2, 3};

numbers.push_back(4);     // добавить элемент в конец
numbers.pop_back();         // удалить последний элемент

cout << numbers.size() << endl;   // количество элементов
```

### 4. Доступ к элементам

```cpp
vector<int> numbers = {10, 20, 30};

cout << numbers[0] << endl;         // 10 (без проверки границ, как у обычного массива)
cout << numbers.at(0) << endl;      // 10 (с проверкой границ - выбросит исключение при выходе за пределы)
cout << numbers.front() << endl;    // первый элемент
cout << numbers.back() << endl;      // последний элемент
```

Метод `.at()` — более безопасная альтернатива оператору `[]`, так как проверяет корректность индекса и выбрасывает исключение `out_of_range` при выходе за границы, вместо неопределённого поведения.

### 5. Перебор vector

```cpp
vector<int> numbers = {10, 20, 30, 40, 50};

for (int i = 0; i < numbers.size(); i++) {
    cout << numbers[i] << " ";
}

for (int number : numbers) {
    cout << number << " ";
}
```

### 6. Изменение размера и очистка

```cpp
vector<int> numbers = {1, 2, 3};

numbers.resize(5);          // изменить размер (новые элементы = 0 по умолчанию)
numbers.clear();              // удалить все элементы
cout << numbers.empty() << endl;   // проверка, пуст ли vector
```

### 7. Практический пример: сбор оценок студента

```cpp
#include <vector>
#include <iostream>
using namespace std;

int main() {
    vector<double> grades;
    int count;
    
    cout << "Сколько оценок ввести? ";
    cin >> count;
    
    for (int i = 0; i < count; i++) {
        double grade;
        cout << "Введите оценку: ";
        cin >> grade;
        grades.push_back(grade);
    }
    
    double sum = 0;
    for (double grade : grades) {
        sum += grade;
    }
    
    cout << "Средняя оценка: " << sum / grades.size() << endl;
    return 0;
}
```

### Резюме урока

- `vector` — динамический массив стандартной библиотеки C++, автоматически изменяющий свой размер при добавлении/удалении элементов.
- `.push_back()`/`.pop_back()` добавляют и удаляют элементы с конца, `.size()` возвращает текущее количество элементов.
- Метод `.at()` безопаснее оператора `[]` благодаря проверке границ и выбросу исключения при выходе за пределы.

## Урок 6.2. Контейнеры map и set
**Время: 50 минут**

### Цель урока
Изучить ассоциативные контейнеры STL — map (аналог словаря) и set (аналог множества).

### 1. Что такое map

`map` — ассоциативный контейнер, хранящий пары «ключ: значение», аналогичный словарю (`dict`) в Python. Элементы в `map` автоматически упорядочены по ключу.

```cpp
#include <map>

map<string, int> ages;

ages["Алина"] = 20;
ages["Данияр"] = 19;
ages["Аружан"] = 22;

cout << ages["Алина"] << endl;   // 20
```

### 2. Проверка наличия ключа

В отличие от Python, обращение к несуществующему ключу через `[]` не вызывает ошибку, а автоматически создаёт новый элемент с значением по умолчанию — это может привести к неожиданному поведению, если не быть внимательным:

```cpp
map<string, int> ages;
ages["Алина"] = 20;

cout << ages["Данияр"] << endl;   // 0 (создаст новую запись с значением 0!)
```

Правильный способ проверить наличие ключа — метод `.find()` или `.count()`:

```cpp
if (ages.find("Данияр") != ages.end()) {
    cout << "Ключ найден" << endl;
} else {
    cout << "Ключ не найден" << endl;
}

if (ages.count("Алина") > 0) {
    cout << "Алина есть в словаре" << endl;
}
```

### 3. Перебор map

```cpp
map<string, int> ages = {{"Алина", 20}, {"Данияр", 19}};

for (const auto &pair : ages) {
    cout << pair.first << ": " << pair.second << endl;
}
```

`pair.first` — ключ, `pair.second` — значение. Ключевое слово `auto` позволяет компилятору самому определить тип элемента, что удобно для сложных типов вроде пар map.

### 4. Удаление элементов из map

```cpp
map<string, int> ages = {{"Алина", 20}, {"Данияр", 19}};
ages.erase("Данияр");
```

### 5. Что такое set

`set` — контейнер, хранящий только уникальные элементы в отсортированном порядке, аналогичный множеству (`set`) в Python:

```cpp
#include <set>

set<int> numbers = {5, 3, 1, 3, 5, 2};
// автоматически хранит только уникальные значения в отсортированном порядке: {1, 2, 3, 5}

for (int number : numbers) {
    cout << number << " ";
}
// 1 2 3 5
```

### 6. Операции с set

```cpp
set<int> numbers = {1, 2, 3};

numbers.insert(4);
numbers.erase(2);

if (numbers.find(3) != numbers.end()) {
    cout << "3 есть в множестве" << endl;
}

cout << numbers.size() << endl;
```

### 7. Практический пример: подсчёт уникальных слов

```cpp
#include <set>
#include <sstream>

set<string> uniqueWords;
string text = "кот собака кот попугай собака кот";
istringstream stream(text);
string word;

while (stream >> word) {
    uniqueWords.insert(word);
}

cout << "Уникальных слов: " << uniqueWords.size() << endl;
```

### Резюме урока

- `map` хранит пары «ключ: значение», автоматически отсортированные по ключу; обращение к несуществующему ключу через `[]` неявно создаёт новую запись.
- `set` хранит только уникальные элементы в отсортированном порядке.
- Проверка наличия ключа/элемента выполняется через `.find()` или `.count()`, а не прямым обращением через `[]`.

## Урок 6.3. Итераторы
**Время: 40 минут**

### Цель урока
Понять концепцию итераторов — универсального механизма доступа к элементам контейнеров STL.

### 1. Что такое итератор

Итератор — это объект, обеспечивающий унифицированный способ последовательного доступа к элементам контейнера, работающий по аналогии с указателем на элемент внутри контейнера. Итераторы позволяют писать единообразный код для работы с разными типами контейнеров STL (vector, map, set и др.).

### 2. Получение итераторов: begin() и end()

```cpp
vector<int> numbers = {10, 20, 30};

vector<int>::iterator it = numbers.begin();   // итератор на первый элемент
```

`.begin()` возвращает итератор, указывающий на первый элемент контейнера, `.end()` — на «условную» позицию сразу после последнего элемента (не на сам последний элемент!).

### 3. Перебор контейнера через итераторы

```cpp
vector<int> numbers = {10, 20, 30, 40};

for (vector<int>::iterator it = numbers.begin(); it != numbers.end(); it++) {
    cout << *it << " ";   // разыменование итератора для получения значения
}
```

### 4. Использование auto для упрощения синтаксиса

Длинный тип `vector<int>::iterator` неудобно писать вручную — ключевое слово `auto` позволяет компилятору автоматически определить тип:

```cpp
for (auto it = numbers.begin(); it != numbers.end(); it++) {
    cout << *it << " ";
}
```

### 5. Итераторы для map

```cpp
map<string, int> ages = {{"Алина", 20}, {"Данияр", 19}};

for (auto it = ages.begin(); it != ages.end(); it++) {
    cout << it->first << ": " << it->second << endl;
}
```

Стрелка `->` используется для доступа к членам объекта, на который указывает итератор — аналогично тому, как `->` используется с обычными указателями на структуры/классы (подробнее в Модуле 7).

### 6. Связь итераторов и range-based for

Синтаксис `for (int number : numbers)`, знакомый по предыдущим модулям, фактически является удобной «оболочкой» над использованием итераторов — компилятор автоматически разворачивает такой цикл в эквивалентный код с явным использованием `.begin()`/`.end()`. Для большинства практических задач range-based for предпочтительнее из-за более простого и читаемого синтаксиса, но понимание итераторов необходимо для более сложных операций (например, удаления элементов во время перебора, поиска конкретной позиции).

### 7. Практический пример: удаление элемента через итератор

```cpp
vector<int> numbers = {1, 2, 3, 4, 5};

auto it = find(numbers.begin(), numbers.end(), 3);   // требует #include <algorithm>
if (it != numbers.end()) {
    numbers.erase(it);   // удаление элемента по итератору
}
```

### Резюме урока

- Итератор — универсальный механизм последовательного доступа к элементам контейнера, работающий по аналогии с указателем.
- `.begin()` возвращает итератор на первый элемент, `.end()` — на позицию сразу после последнего.
- Ключевое слово `auto` упрощает работу с длинными типами итераторов, а range-based for является удобной «оболочкой» над явным использованием итераторов.

## Урок 6.4. Основные алгоритмы STL (sort, find, count)
**Время: 45 минут**

### Цель урока
Изучить готовые алгоритмы стандартной библиотеки для работы с контейнерами.

### 1. Библиотека algorithm

STL предоставляет обширный набор готовых алгоритмов, работающих с большинством контейнеров через итераторы, доступных после подключения `#include <algorithm>`.

### 2. Сортировка: sort

```cpp
#include <algorithm>
#include <vector>

vector<int> numbers = {5, 3, 1, 4, 2};
sort(numbers.begin(), numbers.end());

for (int n : numbers) cout << n << " ";
// 1 2 3 4 5
```

### 3. Сортировка в обратном порядке

```cpp
sort(numbers.begin(), numbers.end(), greater<int>());
// или
sort(numbers.rbegin(), numbers.rend());   // используя обратные итераторы
```

### 4. Сортировка с собственной функцией сравнения

```cpp
vector<string> words = {"банан", "яблоко", "груша"};

sort(words.begin(), words.end(), [](const string &a, const string &b) {
    return a.length() < b.length();   // сортировка по длине строки
});
```

Конструкция `[](параметры) { тело }` — это лямбда-функция C++ (аналог лямбда-функций в Python и JavaScript), часто используемая как компактный способ задать собственную логику сравнения для алгоритмов STL.

### 5. Поиск элемента: find

```cpp
vector<int> numbers = {10, 20, 30, 40};
auto it = find(numbers.begin(), numbers.end(), 30);

if (it != numbers.end()) {
    cout << "Найдено на позиции: " << (it - numbers.begin()) << endl;
} else {
    cout << "Не найдено" << endl;
}
```

### 6. Подсчёт вхождений: count

```cpp
vector<int> numbers = {1, 2, 2, 3, 2, 4};
int countOfTwos = count(numbers.begin(), numbers.end(), 2);
cout << countOfTwos << endl;   // 3
```

### 7. Другие полезные алгоритмы

```cpp
vector<int> numbers = {5, 3, 8, 1, 9};

cout << *max_element(numbers.begin(), numbers.end()) << endl;   // 9
cout << *min_element(numbers.begin(), numbers.end()) << endl;   // 1

reverse(numbers.begin(), numbers.end());   // разворот контейнера "на месте"

bool allPositive = all_of(numbers.begin(), numbers.end(), [](int n) { return n > 0; });
```

### 8. Практический пример: сортировка и поиск студентов

```cpp
struct Student {
    string name;
    int grade;
};

vector<Student> students = {
    {"Алина", 85}, {"Данияр", 92}, {"Аружан", 78}
};

sort(students.begin(), students.end(), [](const Student &a, const Student &b) {
    return a.grade > b.grade;   // сортировка по убыванию оценки
});

for (const auto &s : students) {
    cout << s.name << ": " << s.grade << endl;
}
```

(Структуры `struct` подробно разберём в контексте Модуля 7.)

### Резюме урока

- Библиотека `<algorithm>` предоставляет готовые алгоритмы (`sort`, `find`, `count` и др.), работающие через итераторы с большинством контейнеров STL.
- Лямбда-функции часто используются для задания собственной логики сравнения или условия внутри алгоритмов STL.
- Использование готовых алгоритмов STL предпочтительнее написания собственных циклов «с нуля» — они хорошо протестированы и часто оптимизированы для производительности.

## Урок 6.5. Практика: обработка данных с помощью STL
**Время: 50 минут**

### Цель урока
Закрепить материал модуля, решив несколько практических задач с использованием контейнеров и алгоритмов STL.

### 1. Задача 1. Удаление дубликатов из vector

```cpp
#include <vector>
#include <set>

vector<int> numbers = {1, 2, 2, 3, 3, 3, 4};
set<int> uniqueSet(numbers.begin(), numbers.end());
vector<int> uniqueNumbers(uniqueSet.begin(), uniqueSet.end());

for (int n : uniqueNumbers) cout << n << " ";
// 1 2 3 4
```

### 2. Задача 2. Подсчёт частоты слов через map

```cpp
#include <map>
#include <sstream>

map<string, int> wordCount;
string text = "кот собака кот попугай собака кот";
istringstream stream(text);
string word;

while (stream >> word) {
    wordCount[word]++;
}

for (const auto &pair : wordCount) {
    cout << pair.first << ": " << pair.second << endl;
}
```

### 3. Задача 3. Топ-3 самых частых слова

```cpp
#include <vector>
#include <algorithm>

vector<pair<string, int>> wordVector(wordCount.begin(), wordCount.end());

sort(wordVector.begin(), wordVector.end(), [](const auto &a, const auto &b) {
    return a.second > b.second;   // сортировка по убыванию частоты
});

for (int i = 0; i < 3 && i < wordVector.size(); i++) {
    cout << wordVector[i].first << ": " << wordVector[i].second << endl;
}
```

### 4. Задача 4. Статистика оценок студентов

```cpp
struct Student {
    string name;
    double grade;
};

vector<Student> students = {
    {"Алина", 85}, {"Данияр", 92}, {"Аружан", 78}, {"Ерлан", 95}
};

double sum = 0;
for (const auto &s : students) {
    sum += s.grade;
}
double average = sum / students.size();

auto topStudent = max_element(students.begin(), students.end(),
    [](const Student &a, const Student &b) { return a.grade < b.grade; });

cout << "Средняя оценка: " << average << endl;
cout << "Лучший студент: " << topStudent->name << " (" << topStudent->grade << ")" << endl;
```

### 5. Задача 5. Группировка студентов по признаку (успевающие/неуспевающие)

```cpp
vector<Student> passing, failing;

for (const auto &s : students) {
    if (s.grade >= 80) {
        passing.push_back(s);
    } else {
        failing.push_back(s);
    }
}

cout << "Успевающих: " << passing.size() << endl;
cout << "Неуспевающих: " << failing.size() << endl;
```

### 6. Общие рекомендации по использованию STL

- Предпочитай `vector` обычным массивам в большинстве случаев — он безопаснее и удобнее.
- Используй `map`/`set`, когда нужен быстрый поиск по ключу или уникальность элементов.
- По возможности используй готовые алгоритмы из `<algorithm>` вместо написания собственных циклов — это делает код короче, понятнее и часто эффективнее.

### Резюме урока

- Контейнеры и алгоритмы STL значительно упрощают решение практических задач по сравнению с обычными массивами и написанием кода «с нуля».
- Комбинация `vector` + `map` + алгоритмы `<algorithm>` покрывает большинство типичных задач обработки данных.
- Знание STL — необходимый навык для написания идиоматичного, современного кода на C++, а не просто «C с классами».
