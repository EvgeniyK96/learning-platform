## Урок 7.1. Классы и объекты, конструкторы
**Время: 50 минут**

### Цель урока
Изучить базовые понятия ООП в контексте синтаксиса C++: классы, объекты и конструкторы.

### 1. Класс и объект (повторение концепции из курса Python)

Класс — шаблон, описывающий данные (атрибуты) и поведение (методы) будущих объектов. Объект — конкретный экземпляр, созданный по этому шаблону. Концепция полностью аналогична ООП в Python, но синтаксис в C++ заметно отличается.

```cpp
class Car {
public:
    string brand;
    string model;
    int year;
};

int main() {
    Car myCar;
    myCar.brand = "Toyota";
    myCar.model = "Camry";
    myCar.year = 2022;
    
    cout << myCar.brand << " " << myCar.model << endl;
    return 0;
}
```

### 2. Модификатор доступа public

Ключевое слово `public:` указывает, что перечисленные после него атрибуты и методы доступны из любого места программы. Более подробно модификаторы доступа разберём в Уроке 7.3.

### 3. Конструктор класса

Конструктор — специальный метод, автоматически вызываемый при создании нового объекта, используемый для инициализации его атрибутов. Имя конструктора всегда совпадает с именем класса:

```cpp
class Car {
public:
    string brand;
    string model;
    int year;
    
    Car(string b, string m, int y) {   // конструктор
        brand = b;
        model = m;
        year = y;
    }
};

int main() {
    Car myCar("Toyota", "Camry", 2022);   // вызов конструктора при создании объекта
    cout << myCar.brand << endl;
    return 0;
}
```

### 4. Список инициализации конструктора (member initializer list)

Более идиоматичный для C++ способ инициализации атрибутов в конструкторе — список инициализации, записываемый после двоеточия перед телом конструктора:

```cpp
class Car {
public:
    string brand;
    string model;
    int year;
    
    Car(string b, string m, int y) : brand(b), model(m), year(y) {
        // тело конструктора может остаться пустым, если инициализация уже произошла в списке
    }
};
```

Такой подход считается более эффективным (особенно для сложных типов) и является предпочтительной практикой в современном C++.

### 5. Конструктор по умолчанию

Если класс не объявляет ни одного конструктора, компилятор автоматически генерирует конструктор по умолчанию, не принимающий параметров:

```cpp
class Point {
public:
    int x;
    int y;
};

int main() {
    Point p;   // используется автоматически сгенерированный конструктор по умолчанию
    p.x = 5;
    p.y = 10;
    return 0;
}
```

Как только объявлен хотя бы один пользовательский конструктор (с параметрами), автоматическая генерация конструктора по умолчанию прекращается — если он всё ещё нужен, его необходимо объявить явно.

### 6. Множественные объекты одного класса

```cpp
Car car1("Toyota", "Camry", 2022);
Car car2("BMW", "X5", 2023);

cout << car1.brand << endl;   // Toyota
cout << car2.brand << endl;   // BMW
```

Каждый объект хранит собственные независимые значения атрибутов, как и в Python.

### Резюме урока

- Класс в C++ объявляется через ключевое слово `class`, атрибуты и методы объявляются внутри фигурных скобок с указанием модификатора доступа (`public` и др.).
- Конструктор — метод с именем, совпадающим с именем класса, автоматически вызываемый при создании объекта для инициализации его атрибутов.
- Список инициализации конструктора (после `:`) — предпочтительный, более эффективный способ инициализации атрибутов в современном C++.

## Урок 7.2. Атрибуты и методы класса
**Время: 40 минут**

### Цель урока
Научиться добавлять в класс методы, описывающие поведение объектов.

### 1. Методы класса

```cpp
class Car {
public:
    string brand;
    string model;
    int mileage;
    
    Car(string b, string m) : brand(b), model(m), mileage(0) {}
    
    void drive(int distance) {
        mileage += distance;
        cout << "Проехали " << distance << " км. Общий пробег: " << mileage << " км" << endl;
    }
    
    string info() {
        return brand + " " + model + ", пробег: " + to_string(mileage) + " км";
    }
};

int main() {
    Car myCar("Toyota", "Camry");
    myCar.drive(150);
    myCar.drive(80);
    cout << myCar.info() << endl;
    return 0;
}
```

`to_string()` — стандартная функция C++ для преобразования числа в строку (аналог `str()` в Python).

### 2. Объявление методов вне тела класса

В более крупных проектах методы часто объявляются внутри класса только своей «сигнатурой», а полная реализация выносится за пределы объявления класса с помощью оператора `::` (оператор области видимости):

```cpp
class Car {
public:
    string brand;
    int mileage;
    
    Car(string b);
    void drive(int distance);   // только объявление метода внутри класса
};

Car::Car(string b) : brand(b), mileage(0) {}   // реализация конструктора вне класса

void Car::drive(int distance) {                   // реализация метода вне класса
    mileage += distance;
}
```

Такой подход особенно полезен при разделении объявления класса (обычно в заголовочном файле `.h`) и его реализации (в файле `.cpp`) — тема, которая подробно изучается при работе с более крупными многофайловыми проектами.

### 3. Методы, вызывающие другие методы

```cpp
class Rectangle {
public:
    double width, height;
    
    Rectangle(double w, double h) : width(w), height(h) {}
    
    double area() {
        return width * height;
    }
    
    double perimeter() {
        return 2 * (width + height);
    }
    
    void describe() {
        cout << "Площадь: " << area() << ", периметр: " << perimeter() << endl;
    }
};
```

### 4. Атрибуты класса (статические члены)

Атрибут, общий для всех объектов класса (а не индивидуальный для каждого объекта), объявляется с ключевым словом `static`:

```cpp
class Car {
public:
    static int totalCars;   // общий для всех объектов счётчик
    string brand;
    
    Car(string b) : brand(b) {
        totalCars++;   // увеличиваем общий счётчик при создании каждого объекта
    }
};

int Car::totalCars = 0;   // обязательная инициализация статического члена вне класса

int main() {
    Car car1("Toyota");
    Car car2("BMW");
    cout << "Всего создано машин: " << Car::totalCars << endl;   // 2
    return 0;
}
```

### 5. Практический пример: класс BankAccount

```cpp
class BankAccount {
public:
    string owner;
    double balance;
    
    BankAccount(string o, double b) : owner(o), balance(b) {}
    
    void deposit(double amount) {
        balance += amount;
        cout << "Баланс пополнен на " << amount << ". Текущий баланс: " << balance << endl;
    }
    
    void withdraw(double amount) {
        if (amount > balance) {
            cout << "Недостаточно средств" << endl;
        } else {
            balance -= amount;
            cout << "Снято " << amount << ". Текущий баланс: " << balance << endl;
        }
    }
};
```

### Резюме урока

- Методы объявляются внутри класса аналогично обычным функциям и могут быть реализованы как внутри класса, так и вынесены за его пределы через оператор `::`.
- `static`-атрибуты являются общими для всех объектов класса, а не индивидуальными для каждого экземпляра.
- Методы одного класса могут свободно вызывать другие методы того же класса без явного указания объекта.

## Урок 7.3. Инкапсуляция, геттеры и сеттеры
**Время: 40 минут**

### Цель урока
Изучить принцип инкапсуляции и модификаторы доступа в C++.

### 1. Три модификатора доступа в C++

| Модификатор | Доступ |
|---|---|
| `public` | доступен из любого места программы |
| `private` | доступен только внутри самого класса |
| `protected` | доступен внутри класса и его наследников (подробнее в Уроке 7.4) |

В отличие от Python, где приватность атрибутов — лишь соглашение (`__attr`), в C++ модификаторы доступа строго проверяются компилятором.

### 2. Приватные атрибуты

```cpp
class BankAccount {
private:
    double balance;   // доступен только внутри класса
    
public:
    string owner;
    
    BankAccount(string o, double b) : owner(o), balance(b) {}
};

int main() {
    BankAccount account("Алина", 1000);
    cout << account.balance << endl;   // ОШИБКА КОМПИЛЯЦИИ: balance является private
    return 0;
}
```

### 3. Геттеры и сеттеры

Для контролируемого доступа к приватным атрибутам используются публичные методы — геттеры (чтение) и сеттеры (изменение с проверкой):

```cpp
class BankAccount {
private:
    double balance;
    
public:
    string owner;
    
    BankAccount(string o, double b) : owner(o), balance(b) {}
    
    double getBalance() {          // геттер
        return balance;
    }
    
    void setBalance(double amount) {   // сеттер с проверкой
        if (amount < 0) {
            cout << "Баланс не может быть отрицательным" << endl;
        } else {
            balance = amount;
        }
    }
};

int main() {
    BankAccount account("Алина", 1000);
    cout << account.getBalance() << endl;   // 1000
    account.setBalance(-500);                    // Баланс не может быть отрицательным
    account.setBalance(2000);
    cout << account.getBalance() << endl;   // 2000
    return 0;
}
```

### 4. Класс по умолчанию: class vs struct

Важная особенность C++: если в `class` не указан модификатор доступа явно, по умолчанию все члены являются `private`. В `struct` (структуре) по умолчанию действует `public`:

```cpp
class Point {
    int x, y;   // по умолчанию private (недоступны снаружи!)
};

struct Point2 {
    int x, y;   // по умолчанию public (доступны снаружи)
};
```

На практике `struct` в современном C++ обычно используется для простых структур данных без сложной логики (только публичные данные), а `class` — когда требуется полноценная инкапсуляция.

### 5. Практический пример: класс Person с валидацией

```cpp
class Person {
private:
    string name;
    int age;
    
public:
    Person(string n, int a) {
        name = n;
        setAge(a);   // используем сеттер даже внутри конструктора для единой валидации
    }
    
    string getName() { return name; }
    int getAge() { return age; }
    
    void setAge(int a) {
        if (a < 0 || a > 150) {
            cout << "Некорректный возраст, установлено значение по умолчанию" << endl;
            age = 0;
        } else {
            age = a;
        }
    }
};
```

### 6. Зачем нужна инкапсуляция на практике

Как и в Python (Модуль 7 курса Python), инкапсуляция защищает данные от некорректных значений, позволяет изменять внутреннюю реализацию класса без影响 на код, использующий его публичный интерфейс, и делает интерфейс класса более понятным для других разработчиков.

### Резюме урока

- В C++ три модификатора доступа: `public` (доступен везде), `private` (только внутри класса), `protected` (класс и наследники).
- В отличие от Python, приватность в C++ строго проверяется компилятором, а не является лишь соглашением об именовании.
- `class` по умолчанию использует `private`, `struct` — `public`; это единственное принципиальное синтаксическое отличие между ними в C++.

## Урок 7.4. Наследование
**Время: 45 минут**

### Цель урока
Изучить наследование классов в C++.

### 1. Базовый синтаксис наследования

```cpp
class Vehicle {
public:
    string brand;
    
    Vehicle(string b) : brand(b) {}
    
    void info() {
        cout << "Транспортное средство: " << brand << endl;
    }
};

class Car : public Vehicle {    // Car наследует от Vehicle
public:
    int doors;
    
    Car(string b, int d) : Vehicle(b), doors(d) {}   // вызов конструктора родителя
};

int main() {
    Car myCar("Toyota", 4);
    myCar.info();                  // унаследованный метод
    cout << myCar.doors << endl;   // 4
    return 0;
}
```

Ключевое слово `public` перед именем родительского класса (`class Car : public Vehicle`) означает публичное наследование — самый распространённый вид, при котором публичные члены родителя остаются публичными в дочернем классе.

### 2. Вызов конструктора родительского класса

Как и в примере выше, конструктор дочернего класса должен явно вызвать конструктор родительского класса в списке инициализации, если у родителя нет конструктора по умолчанию без параметров.

### 3. Переопределение методов: virtual

Чтобы дочерний класс мог переопределить метод родителя с полноценным полиморфным поведением (подробнее в Уроке 7.5), метод родительского класса должен быть объявлен с ключевым словом `virtual`:

```cpp
class Vehicle {
public:
    virtual void move() {
        cout << "Транспортное средство движется" << endl;
    }
};

class Boat : public Vehicle {
public:
    void move() override {   // override явно указывает на переопределение
        cout << "Лодка плывёт по воде" << endl;
    }
};
```

Ключевое слово `override` (появившееся в C++11) не обязательно, но настоятельно рекомендуется — оно явно сообщает компилятору о намерении переопределить метод родителя, и компилятор выдаст ошибку, если сигнатура не совпадает с методом родителя (например, из-за опечатки).

### 4. Многоуровневое наследование

```cpp
class Vehicle {
public:
    string brand;
    Vehicle(string b) : brand(b) {}
};

class Car : public Vehicle {
public:
    int doors;
    Car(string b, int d) : Vehicle(b), doors(d) {}
};

class SportsCar : public Car {
public:
    int topSpeed;
    SportsCar(string b, int d, int speed) : Car(b, d), topSpeed(speed) {}
};

int main() {
    SportsCar ferrari("Ferrari", 2, 340);
    cout << ferrari.brand << " " << ferrari.doors << " " << ferrari.topSpeed << endl;
    return 0;
}
```

### 5. Модификатор protected для наследования

Атрибуты, объявленные как `protected`, недоступны снаружи класса (как `private`), но доступны в дочерних классах (в отличие от `private`):

```cpp
class Vehicle {
protected:
    int mileage;   // доступен в Vehicle и его наследниках, но не снаружи

public:
    Vehicle() : mileage(0) {}
};

class Car : public Vehicle {
public:
    void drive(int distance) {
        mileage += distance;   // доступ разрешён, так как mileage - protected
    }
};
```

### 6. Виды наследования: public, protected, private (обзорно)

Помимо публичного наследования (`class Car : public Vehicle`), C++ также поддерживает `protected` и `private` наследование, изменяющие уровень доступа унаследованных членов в дочернем классе. В подавляющем большинстве случаев используется именно публичное наследование, а два других вида применяются значительно реже и выходят за рамки базового уровня курса.

### Резюме урока

- Наследование в C++ задаётся синтаксисом `class Дочерний : public Родитель`, конструктор родителя явно вызывается в списке инициализации дочернего класса.
- Для полноценного переопределения метода родительский метод должен быть объявлен `virtual`, а переопределение в наследнике рекомендуется помечать `override`.
- Модификатор `protected` обеспечивает доступ к члену класса из дочерних классов, но не извне, в отличие от `private`.

## Урок 7.5. Полиморфизм и виртуальные функции
**Время: 45 минут**

### Цель урока
Изучить полиморфизм в C++ через механизм виртуальных функций.

### 1. Проблема без virtual

Без ключевого слова `virtual` вызов метода через указатель на базовый класс всегда вызывает версию метода именно базового класса, даже если фактический объект относится к дочернему классу:

```cpp
class Animal {
public:
    void makeSound() {
        cout << "Какой-то звук" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() {
        cout << "Мяу" << endl;
    }
};

void playSound(Animal *animal) {
    animal->makeSound();
}

int main() {
    Cat cat;
    playSound(&cat);   // выведет "Какой-то звук" (НЕ "Мяу"), так как makeSound не virtual!
    return 0;
}
```

### 2. Решение: virtual функции

```cpp
class Animal {
public:
    virtual void makeSound() {
        cout << "Какой-то звук" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override {
        cout << "Мяу" << endl;
    }
};

void playSound(Animal *animal) {
    animal->makeSound();
}

int main() {
    Cat cat;
    playSound(&cat);   // теперь корректно выведет "Мяу"
    return 0;
}
```

Когда метод объявлен `virtual`, C++ определяет, какую именно версию метода вызвать, на основе **фактического типа объекта во время выполнения программы** (динамическая диспетчеризация), а не типа указателя/ссылки, через который к нему обращаются.

### 3. Полиморфизм с коллекцией объектов

```cpp
#include <vector>

class Animal {
public:
    virtual void makeSound() {
        cout << "Какой-то звук" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override { cout << "Мяу" << endl; }
};

class Dog : public Animal {
public:
    void makeSound() override { cout << "Гав" << endl; }
};

int main() {
    vector<Animal*> animals;
    animals.push_back(new Cat());
    animals.push_back(new Dog());
    
    for (Animal *animal : animals) {
        animal->makeSound();   // вызывает правильную версию для каждого объекта
    }
    
    return 0;
}
```

(Использование `new` для динамического создания объектов подробно разберём в Модуле 8.)

### 4. Абстрактные классы и чисто виртуальные функции

Абстрактный класс содержит хотя бы одну «чисто виртуальную» функцию (объявленную с `= 0`) и не может быть инстанцирован напрямую — он служит только базой для наследования:

```cpp
class Shape {
public:
    virtual double area() = 0;   // чисто виртуальная функция - "контракт" для наследников
};

class Circle : public Shape {
public:
    double radius;
    Circle(double r) : radius(r) {}
    
    double area() override {
        return 3.14159 * radius * radius;
    }
};

int main() {
    // Shape shape;   // ОШИБКА: нельзя создать объект абстрактного класса
    Circle circle(5);
    cout << circle.area() << endl;
    return 0;
}
```

### 5. Виртуальный деструктор (важная практика, обзорно)

При работе с полиморфизмом и указателями на базовый класс важно объявлять деструктор базового класса виртуальным — это гарантирует правильное освобождение памяти для объектов дочерних классов (подробнее о деструкторах в Модуле 8):

```cpp
class Animal {
public:
    virtual ~Animal() {}   // виртуальный деструктор
};
```

### 6. Зачем нужен полиморфизм на практике

Полиморфизм позволяет писать функции и структуры данных, работающие единообразно с объектами разных, но родственных классов (как в примере с `vector<Animal*>`), не зная заранее точный тип каждого конкретного объекта — это основа многих архитектурных паттернов и делает код более расширяемым.

### Резюме урока

- Для полноценного полиморфного поведения метод должен быть объявлен `virtual` в базовом классе — иначе вызывается версия метода в зависимости от типа указателя/ссылки, а не фактического объекта.
- Чисто виртуальные функции (`= 0`) делают класс абстрактным — такой класс нельзя инстанцировать, он служит лишь «контрактом» для наследников.
- При использовании полиморфизма с динамически созданными объектами важно объявлять деструктор базового класса виртуальным.

## Урок 7.6. Перегрузка операторов
**Время: 40 минут**

### Цель урока
Изучить перегрузку операторов — механизм, позволяющий определить поведение стандартных операторов (`+`, `==` и др.) для собственных классов.

### 1. Что такое перегрузка операторов

Перегрузка операторов позволяет определить, как стандартные операторы C++ (`+`, `-`, `==`, `<<` и др.) должны вести себя применительно к объектам пользовательских классов — аналог магических методов в Python (Урок 7.6 курса Python).

### 2. Перегрузка оператора + (сложение)

```cpp
class Point {
public:
    double x, y;
    
    Point(double x, double y) : x(x), y(y) {}
    
    Point operator+(const Point &other) {
        return Point(x + other.x, y + other.y);
    }
};

int main() {
    Point p1(1, 2);
    Point p2(3, 4);
    Point p3 = p1 + p2;   // вызывает перегруженный operator+
    
    cout << p3.x << ", " << p3.y << endl;   // 4, 6
    return 0;
}
```

### 3. Перегрузка оператора == (сравнение)

```cpp
class Point {
public:
    double x, y;
    Point(double x, double y) : x(x), y(y) {}
    
    bool operator==(const Point &other) {
        return x == other.x && y == other.y;
    }
};

int main() {
    Point p1(1, 2);
    Point p2(1, 2);
    cout << (p1 == p2) << endl;   // 1 (true), без перегрузки сравнивались бы адреса объектов
    return 0;
}
```

### 4. Перегрузка оператора << для вывода через cout

Один из самых полезных случаев перегрузки — возможность выводить объект пользовательского класса напрямую через `cout`, аналогично встроенным типам:

```cpp
class Point {
public:
    double x, y;
    Point(double x, double y) : x(x), y(y) {}
    
    friend ostream& operator<<(ostream &os, const Point &p) {
        os << "(" << p.x << ", " << p.y << ")";
        return os;
    }
};

int main() {
    Point p(3, 4);
    cout << p << endl;   // (3, 4)
    return 0;
}
```

Ключевое слово `friend` предоставляет функции доступ к приватным членам класса, даже если сама функция не является методом этого класса — эта функция должна быть внешней (не методом класса), так как первым операндом оператора `<<` является поток `cout` (тип `ostream`), а не сам объект `Point`.

### 5. Какие операторы можно перегружать

В C++ можно перегружать большинство операторов: арифметические (`+`, `-`, `*`, `/`), сравнения (`==`, `<`, `>`), индексации (`[]`), вызова функции (`()`) и многие другие. Нельзя перегружать некоторые специальные операторы, например `::` или тернарный оператор `?:`.

### 6. Рекомендации по использованию перегрузки операторов

Перегрузка операторов — мощный инструмент, но им не стоит злоупотреблять: оператор должен вести себя интуитивно понятным образом, соответствующим его обычному смыслу (например, `+` для класса `Point` логично складывает координаты, но было бы странно, если бы `+` для какого-то класса неожиданно удалял данные).

### Резюме урока

- Перегрузка операторов позволяет определить поведение стандартных операторов C++ для объектов пользовательских классов.
- Оператор `operator+`, `operator==` и другие объявляются как специальные методы класса (или внешние функции с ключевым словом `friend` для операторов вроде `<<`).
- Перегруженные операторы должны вести себя интуитивно понятным образом, соответствующим их обычному математическому или логическому смыслу.

## Урок 7.7. Практика: разработка мини-приложения на ООП
**Время: 65 минут**

### Цель урока
Закрепить весь материал модуля, разработав небольшое приложение «Автопарк» с использованием классов, наследования, инкапсуляции и полиморфизма — по аналогии с итоговой практикой Модуля 7 курса Python.

### 1. Постановка задачи

Как и в аналогичном проекте на Python, разработаем систему учёта автопарка: базовый класс `Vehicle`, дочерние классы `Car` и `Truck`, и класс `Fleet` для управления коллекцией транспортных средств.

### 2. Базовый класс Vehicle

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

class Vehicle {
protected:
    string brand;
    string model;
    int year;
    
public:
    Vehicle(string b, string m, int y) : brand(b), model(m), year(y) {}
    
    virtual string info() {
        return brand + " " + model + " (" + to_string(year) + ")";
    }
    
    virtual ~Vehicle() {}
};
```

### 3. Дочерние классы

```cpp
class Car : public Vehicle {
private:
    int seats;
    
public:
    Car(string b, string m, int y, int s) : Vehicle(b, m, y), seats(s) {}
    
    string info() override {
        return Vehicle::info() + " — легковой автомобиль, " + to_string(seats) + " мест";
    }
};

class Truck : public Vehicle {
private:
    double loadCapacity;
    
public:
    Truck(string b, string m, int y, double lc) : Vehicle(b, m, y), loadCapacity(lc) {}
    
    string info() override {
        return Vehicle::info() + " — грузовик, грузоподъёмность " + to_string(loadCapacity) + " т";
    }
};
```

Вызов `Vehicle::info()` внутри переопределённого метода — способ вызвать реализацию метода родительского класса, дополнив её собственной логикой, аналогично `super()` в Python.

### 4. Класс Fleet — управление коллекцией

```cpp
class Fleet {
private:
    vector<Vehicle*> vehicles;
    
public:
    void addVehicle(Vehicle *vehicle) {
        vehicles.push_back(vehicle);
        cout << "Добавлено: " << vehicle->info() << endl;
    }
    
    void showAll() {
        cout << "\n--- Список автопарка ---" << endl;
        for (Vehicle *vehicle : vehicles) {
            cout << vehicle->info() << endl;   // полиморфизм: у каждого свой info()
        }
    }
    
    ~Fleet() {
        for (Vehicle *vehicle : vehicles) {
            delete vehicle;   // освобождение памяти для каждого динамически созданного объекта
        }
    }
};
```

(Оператор `delete` и управление динамической памятью подробно разберём в Модуле 8.)

### 5. Использование приложения

```cpp
int main() {
    Fleet fleet;
    
    fleet.addVehicle(new Car("Toyota", "Camry", 2022, 5));
    fleet.addVehicle(new Truck("Volvo", "FH16", 2020, 20.0));
    fleet.addVehicle(new Car("Honda", "Civic", 2023, 5));
    
    fleet.showAll();
    
    return 0;
}
```

### 6. Что демонстрирует этот пример

- **Наследование:** `Car` и `Truck` наследуют общие атрибуты и логику от `Vehicle`.
- **Полиморфизм:** виртуальный метод `info()` работает по-разному для разных классов, но вызывается единообразно в `showAll()`.
- **Инкапсуляция:** атрибуты классов защищены модификаторами `private`/`protected`.
- **Управление памятью:** класс `Fleet` берёт на себя ответственность за освобождение памяти объектов, созданных через `new` — важная тема, которую подробно разберём в следующем модуле.

### Резюме урока

- Реальные приложения на C++ с использованием ООП обычно комбинируют наследование, полиморфизм, инкапсуляцию и управление динамической памятью вместе.
- Использование `vector<Vehicle*>` с полиморфными объектами — распространённый паттерн для хранения коллекций разнотипных, но родственных объектов.
- Этот пример закладывает основу для перехода к Модулю 8, где подробно разбирается управление памятью, уже частично затронутое здесь через `new`/`delete`.
