## Урок 2.1. Переменные и типы данных (int, double, char, bool, string)
**Время: 45 минут**

### Цель урока
Изучить основные типы данных C++ и научиться объявлять переменные с явным указанием типа.

### 1. Статическая типизация

В отличие от Python и JavaScript, C++ использует статическую типизацию — тип каждой переменной должен быть указан явно при объявлении и не может измениться в дальнейшем.

```cpp
int age = 20;
age = "текст";   // ошибка компиляции: нельзя присвоить строку переменной типа int
```

### 2. Основные типы данных

| Тип | Описание | Пример |
|---|---|---|
| `int` | целое число | `int age = 20;` |
| `double` | дробное число двойной точности | `double price = 19.99;` |
| `float` | дробное число одинарной точности (менее точное, занимает меньше памяти) | `float rate = 3.14f;` |
| `char` | один символ | `char grade = 'A';` |
| `bool` | логическое значение | `bool isActive = true;` |
| `string` | строка (требует `#include <string>`) | `string name = "Алина";` |

### 3. Объявление переменных

```cpp
int age;              // объявление без инициализации
age = 20;                // присваивание значения отдельно

int height = 175;      // объявление с одновременной инициализацией (рекомендуется)
```

### 4. Особенности char

Тип `char` хранит ровно один символ и записывается в одинарных кавычках, в отличие от `string`, использующего двойные кавычки:

```cpp
char letter = 'A';        // правильно
char letter2 = "A";       // ошибка: "A" - это строка, а не символ
```

### 5. Разница между int и double при делении

```cpp
int a = 5;
int b = 2;
cout << a / b << endl;         // 2 (целочисленное деление, дробная часть отбрасывается)

double c = 5.0;
double d = 2.0;
cout << c / d << endl;         // 2.5
```

Это частый источник ошибок у новичков: деление двух `int` всегда даёт `int`, даже если результат должен быть дробным.

### 6. Работа со строками (std::string)

```cpp
#include <string>

string firstName = "Алина";
string lastName = "Иванова";
string fullName = firstName + " " + lastName;    // конкатенация через +

cout << fullName << endl;
cout << fullName.length() << endl;   // длина строки
```

### 7. Именование переменных

Правила именования схожи с другими языками: буквы, цифры, знак подчёркивания, не может начинаться с цифры. Общепринятый стиль в C++ — `camelCase` (`userName`) или `snake_case` (`user_name`) в зависимости от конвенций конкретного проекта.

### Резюме урока

- C++ использует статическую типизацию — тип переменной задаётся явно при объявлении и не может измениться.
- Основные типы: `int`, `double`, `float`, `char` (один символ, одинарные кавычки), `bool`, `string` (требует подключения `<string>`).
- Деление двух переменных типа `int` всегда даёт целочисленный результат, даже если ожидается дробное значение.

## Урок 2.2. Ввод и вывод: cin, cout
**Время: 40 минут**

### Цель урока
Научиться получать данные от пользователя через `cin` и выводить результат через `cout`.

### 1. Повторение: вывод через cout

```cpp
cout << "Привет, мир!" << endl;
cout << "Число: " << 42 << endl;
```

### 2. Ввод данных: cin

`cin` («console input») — стандартный поток ввода, считывающий данные, введённые пользователем в терминале:

```cpp
#include <iostream>
using namespace std;

int main() {
    int age;
    cout << "Введите ваш возраст: ";
    cin >> age;
    cout << "Вам " << age << " лет" << endl;
    return 0;
}
```

Оператор `>>` («извлечь из потока») передаёт введённые данные в указанную переменную.

### 3. Важная особенность: cin и типы данных

`cin` автоматически пытается преобразовать введённые данные к типу переменной. Если пользователь вводит некорректные данные (например, текст вместо числа), это может привести к ошибке состояния потока, что выходит за рамки базового уровня, но важно знать об этой особенности.

### 4. Ввод нескольких значений

```cpp
int a, b;
cout << "Введите два числа через пробел: ";
cin >> a >> b;
cout << "Сумма: " << a + b << endl;
```

### 5. Ввод строк: особенность с пробелами

```cpp
string name;
cout << "Введите имя: ";
cin >> name;    // считывает только до первого пробела!
```

Оператор `cin >>` считывает строку только до первого пробела. Для считывания целой строки, включая пробелы, используется функция `getline()`:

```cpp
string fullName;
cout << "Введите полное имя: ";
getline(cin, fullName);   // считывает всю строку целиком, включая пробелы
```

### 6. Смешивание cin >> и getline() — частая ловушка

```cpp
int age;
string name;

cout << "Введите возраст: ";
cin >> age;

cout << "Введите имя: ";
getline(cin, name);   // проблема! считает пустую строку

// Решение: очистить оставшийся символ переноса строки перед getline
cin.ignore();
getline(cin, name);
```

Эта особенность связана с тем, что `cin >> age` оставляет в потоке ввода символ переноса строки (нажатие Enter), который затем сразу «съедается» следующим вызовом `getline()`, создавая иллюзию пустого ввода.

### 7. Практический пример

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string name;
    int age;
    
    cout << "Введите имя: ";
    getline(cin, name);
    
    cout << "Введите возраст: ";
    cin >> age;
    
    cout << "Привет, " << name << "! Тебе " << age << " лет." << endl;
    return 0;
}
```

### Резюме урока

- `cin >>` считывает данные из терминала до первого пробела и автоматически приводит их к типу переменной.
- `getline(cin, переменная)` считывает всю строку целиком, включая пробелы.
- Смешивание `cin >>` и `getline()` требует особого внимания из-за особенности с оставшимся символом переноса строки в потоке.

## Урок 2.3. Операторы: арифметические, сравнения, логические
**Время: 40 минут**

### Цель урока
Изучить основные группы операторов C++.

### 1. Арифметические операторы

| Оператор | Значение | Пример | Результат |
|---|---|---|---|
| `+` | сложение | `5 + 2` | `7` |
| `-` | вычитание | `5 - 2` | `3` |
| `*` | умножение | `5 * 2` | `10` |
| `/` | деление | `5 / 2` | `2` (для int!) |
| `%` | остаток от деления | `5 % 2` | `1` |

### 2. Операторы инкремента и декремента

```cpp
int count = 5;
count++;         // count = count + 1
count--;          // count = count - 1

int a = 5;
cout << a++ << endl;   // выведет 5, ЗАТЕМ увеличит a (постфиксная форма)
cout << ++a << endl;   // СНАЧАЛА увеличит a, ЗАТЕМ выведет (префиксная форма)
```

### 3. Операторы сравнения

```cpp
cout << (5 > 3) << endl;    // 1 (true)
cout << (5 < 3) << endl;    // 0 (false)
cout << (5 == 5) << endl;   // 1
cout << (5 != 3) << endl;   // 1
```

Обрати внимание: в C++ значения типа `bool` при выводе через `cout` по умолчанию отображаются как `1` (true) и `0` (false), а не как слова «true»/«false».

### 4. Логические операторы

| Оператор | Значение |
|---|---|
| `&&` | логическое И |
| `\|\|` | логическое ИЛИ |
| `!` | логическое НЕ |

```cpp
int age = 20;
bool hasId = true;

cout << (age >= 18 && hasId) << endl;   // 1
cout << (age < 18 || hasId) << endl;     // 1
cout << !hasId << endl;                    // 0
```

### 5. Операторы составного присваивания

```cpp
int x = 10;
x += 5;    // x = x + 5, теперь x = 15
x -= 3;    // x = x - 3, теперь x = 12
x *= 2;    // x = x * 2, теперь x = 24
x /= 4;    // x = x / 4, теперь x = 6
```

### 6. Приоритет операторов

Как и в других языках, скобки `()` позволяют явно задать порядок выполнения:

```cpp
int result = (2 + 3) * 4;   // 20, а не 14
```

### Резюме урока

- Деление `int / int` всегда даёт целочисленный результат — важно учитывать это при математических вычислениях.
- `bool`-значения при выводе через `cout` отображаются как `1`/`0`, а не как слова.
- Постфиксный (`x++`) и префиксный (`++x`) инкремент отличаются моментом, когда происходит увеличение значения относительно его использования в выражении.

## Урок 2.4. Приведение типов (явное и неявное)
**Время: 35 минут**

### Цель урока
Понять механизмы явного и неявного приведения типов в C++.

### 1. Неявное приведение типов

C++ иногда автоматически преобразует один тип в другой при выполнении операций:

```cpp
int a = 5;
double b = 2.0;
double result = a / b;    // int неявно преобразуется в double
cout << result << endl;    // 2.5
```

### 2. Проблема целочисленного деления

```cpp
int a = 5;
int b = 2;
double result = a / b;   // ОШИБКА логики: деление произойдёт как int / int = 2, и только потом 2 будет преобразовано в double
cout << result << endl;   // 2, а не 2.5!
```

Чтобы получить корректный дробный результат, нужно явно привести хотя бы один из операндов к `double` **до** выполнения деления.

### 3. Явное приведение типов: static_cast

Современный, рекомендуемый способ явного приведения типов в C++ — `static_cast`:

```cpp
int a = 5;
int b = 2;
double result = static_cast<double>(a) / b;
cout << result << endl;   // 2.5
```

### 4. Устаревший способ приведения типов (для общего понимания)

```cpp
double result = (double)a / b;    // "C-style" приведение типов, унаследовано от языка C
```

Такой способ работает, но считается менее безопасным и менее явным по сравнению с `static_cast`, так как не всегда очевидно, какое именно преобразование происходит — в современном коде C++ рекомендуется использовать `static_cast`.

### 5. Потеря точности при приведении типов

```cpp
double price = 19.99;
int roundedPrice = static_cast<int>(price);
cout << roundedPrice << endl;   // 19 (дробная часть просто отбрасывается, а не округляется!)
```

Важно: приведение `double` к `int` не округляет число, а отбрасывает дробную часть (усечение).

### 6. Приведение char и int

```cpp
char letter = 'A';
int code = static_cast<int>(letter);
cout << code << endl;   // 65 (ASCII-код символа 'A')

int number = 66;
char newLetter = static_cast<char>(number);
cout << newLetter << endl;   // B
```

Символы в C++ внутренне хранятся как числовые коды (по таблице ASCII), поэтому приведение между `char` и `int` — обычная и часто полезная операция.

### Резюме урока

- Неявное приведение типов происходит автоматически, но при делении `int / int` результат остаётся целочисленным до момента присваивания.
- `static_cast<тип>(значение)` — рекомендуемый современный способ явного приведения типов в C++.
- Приведение `double` к `int` отбрасывает дробную часть числа, а не округляет его.

## Урок 2.5. Константы и модификатор const
**Время: 35 минут**

### Цель урока
Научиться объявлять неизменяемые значения с помощью ключевого слова const и понять его роль в написании надёжного кода.

### 1. Что такое константа

Константа — значение, которое не может быть изменено после инициализации. Использование констант делает код более безопасным (защита от случайного изменения) и понятным (сразу ясно, что значение не должно меняться).

### 2. Объявление константы: const

```cpp
const double PI = 3.14159;
const int MAX_ATTEMPTS = 3;

PI = 3.14;   // ошибка компиляции: нельзя изменить значение константы
```

### 3. Соглашение об именовании констант

Хотя формально это необязательно, распространённое соглашение — писать имена констант заглавными буквами с разделением через нижнее подчёркивание, аналогично Python и другим языкам:

```cpp
const int MAX_USERS = 100;
const double TAX_RATE = 0.12;
```

### 4. Зачем использовать const, если можно просто не менять переменную

Ключевое отличие от «просто не менять переменную по договорённости» в том, что `const` — это гарантия, проверяемая компилятором. Если по ошибке в коде где-то будет попытка изменить константу, компилятор сразу укажет на эту ошибку, а не позволит ей остаться незамеченной до момента выполнения программы.

### 5. const в параметрах функций (забегая вперёд, подробнее в Модуле 4)

```cpp
void printMessage(const string& message) {
    cout << message << endl;
    // message = "новое значение";   // ошибка: message нельзя изменить внутри функции
}
```

Использование `const` в параметрах функций — важная практика, гарантирующая, что функция не изменит переданные ей данные (подробнее об этом поговорим в контексте передачи параметров по ссылке в Уроке 4.2).

### 6. Разница между const и #define (для общего понимания)

В языке C для объявления констант часто использовалась директива препроцессора `#define`:

```cpp
#define PI 3.14159   // устаревший способ, унаследованный от C
```

В современном C++ использование `const` (а для более сложных случаев — `constexpr`) считается предпочтительным, так как `#define` не имеет типа и не проверяется компилятором так же строго, как `const`.

### Резюме урока

- `const` объявляет неизменяемое значение, попытка его изменения приводит к ошибке компиляции.
- Использование `const` — это защита кода от случайных ошибок, проверяемая на этапе компиляции, а не просто соглашение между разработчиками.
- В современном C++ `const` предпочтительнее устаревшей директивы препроцессора `#define` для объявления констант.
