## Урок 1.1. Что такое Java, принцип «write once, run anywhere», JVM
**Время: 35 минут**

### Цель урока
Понять, что такое Java, в чём заключается её ключевая особенность переносимости, и как устроен процесс выполнения Java-программ.

### 1. Что такое Java

Java — один из самых распространённых языков программирования в мире, созданный компанией Sun Microsystems в 1995 году (сейчас развивается компанией Oracle). Java — объектно-ориентированный, статически типизированный язык, спроектированный с особым акцентом на переносимость между разными операционными системами.

### 2. Принцип «Write Once, Run Anywhere»

Девиз Java — «напиши один раз, запусти где угодно» (Write Once, Run Anywhere, WORA). Это означает, что одна и та же скомпилированная Java-программа может без изменений выполняться на Windows, macOS, Linux и других платформах — в отличие от C++, где скомпилированный исполняемый файл обычно привязан к конкретной операционной системе.

### 3. Как это работает: JVM и байт-код

Java достигает переносимости за счёт двухэтапного процесса выполнения:

1. Исходный код (`.java` файл) компилируется не напрямую в машинный код конкретного процессора, а в промежуточный **байт-код** (`.class` файл) — универсальный набор инструкций, не привязанный к конкретной операционной системе.
2. Байт-код выполняется специальной программой — **JVM (Java Virtual Machine, виртуальная машина Java)**, которая уже установлена конкретно под операционную систему пользователя и переводит байт-код в машинный код «на лету».

```
Код.java → javac (компилятор) → Код.class (байт-код) → JVM → выполнение на конкретной ОС
```

### 4. JDK, JRE и JVM: в чём разница

| Компонент | Назначение |
|---|---|
| **JVM (Java Virtual Machine)** | выполняет байт-код на конкретной платформе |
| **JRE (Java Runtime Environment)** | JVM + стандартные библиотеки, необходимые только для запуска готовых Java-программ |
| **JDK (Java Development Kit)** | JRE + инструменты разработки (компилятор `javac`, отладчик и др.), необходим для написания и компиляции кода |

Для разработки на Java необходим именно JDK — он включает в себя всё необходимое, включая JRE.

### 5. Где применяется Java

- **Корпоративные приложения (Enterprise).** Java — стандарт индустрии для крупных бизнес-систем: банковское ПО, системы управления предприятием.
- **Android-разработка.** Долгое время Java была основным языком разработки приложений под Android (сегодня наравне с Kotlin).
- **Веб-бэкенд.** Фреймворк Spring — один из самых популярных инструментов для серверной разработки на Java.
- **Большие данные.** Многие инструменты экосистемы Big Data (Hadoop, Kafka) написаны на Java.

### 6. Почему Java остаётся востребованной спустя почти 30 лет

Java сочетает надёжность статической типизации, зрелую экосистему инструментов и библиотек, обратную совместимость (код, написанный много лет назад, как правило, продолжает работать на новых версиях), и огромный рынок труда, особенно в корпоративном секторе — эти факторы делают её одним из самых стабильных выборов языка программирования для долгосрочной карьеры.

### Резюме урока

- Java компилируется не напрямую в машинный код, а в промежуточный байт-код, выполняемый виртуальной машиной JVM — это обеспечивает переносимость программ между операционными системами.
- Для разработки на Java необходим JDK, включающий компилятор и другие инструменты, в отличие от JRE, предназначенного только для запуска готовых программ.
- Java широко применяется в корпоративной разработке, Android-приложениях и веб-бэкенде.

## Урок 1.2. Установка JDK и настройка IntelliJ IDEA
**Время: 45 минут**

### Цель урока
Установить JDK и настроить среду разработки IntelliJ IDEA для написания и запуска Java-программ.

### 1. Установка JDK

1. Перейти на сайт официального дистрибутива JDK (например, Oracle JDK или бесплатную сборку Eclipse Temurin/OpenJDK).
2. Скачать установщик для своей операционной системы (рекомендуется использовать актуальную LTS-версию — Long Term Support, версию с длительной поддержкой).
3. Установить, следуя инструкциям установщика.

Проверка установки через терминал:

```bash
java --version
javac --version
```

Если обе команды выводят номер версии, JDK установлен корректно.

### 2. Что такое IntelliJ IDEA

IntelliJ IDEA — одна из самых популярных и мощных сред разработки (IDE) для Java, разработанная компанией JetBrains. Существует бесплатная Community-версия (достаточная для учебных целей) и платная Ultimate-версия с дополнительными возможностями для веб- и корпоративной разработки.

### 3. Установка IntelliJ IDEA

1. Скачать IntelliJ IDEA Community Edition с сайта jetbrains.com.
2. Установить, следуя инструкциям установщика.
3. При первом запуске IDE автоматически предложит настроить базовые параметры.

### 4. Создание первого проекта

1. В IntelliJ IDEA выбрать «New Project».
2. Указать имя проекта и убедиться, что выбран установленный JDK в качестве SDK проекта.
3. IDE автоматически создаст базовую структуру проекта.

### 5. Структура типичного Java-проекта

```
MyProject/
├── src/                # исходный код (.java файлы)
│   └── Main.java
└── out/                  # скомпилированные .class файлы (создаются автоматически)
```

### 6. Первый запуск программы

IntelliJ IDEA автоматически создаёт файл `Main.java` с заготовкой кода при создании нового проекта. Запуск программы осуществляется нажатием зелёной кнопки «Run» рядом с методом `main` или сочетанием клавиш `Shift+F10`.

### 7. Компиляция и запуск через терминал (для понимания процесса)

Хотя IntelliJ IDEA берёт этот процесс на себя автоматически, полезно понимать, что происходит «под капотом»:

```bash
javac Main.java     # компиляция: создаёт файл Main.class (байт-код)
java Main              # запуск: имя класса указывается БЕЗ расширения .class
```

### Резюме урока

- JDK необходим для компиляции и разработки на Java, устанавливается отдельно перед началом работы.
- IntelliJ IDEA — мощная и удобная среда разработки, широко используемая как профессионалами, так и новичками при изучении Java.
- Компиляция (`javac`) создаёт файл байт-кода (`.class`), который затем выполняется командой `java` (без указания расширения файла).

## Урок 1.3. Первая программа: класс, метод main, компиляция
**Время: 35 минут**

### Цель урока
Разобрать структуру первой программы на Java и понять обязательные элементы синтаксиса.

### 1. Первая программа

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

### 2. Разбор структуры программы

- **`public class Main`** — объявление класса с именем `Main`. Важнейшая особенность Java: **каждый класс, содержащий код, должен быть определён внутри файла с точно таким же именем**, что и класс (в данном случае файл обязательно должен называться `Main.java`).
- **`public static void main(String[] args)`** — метод `main`, точка входа в программу, аналогичная функции `main()` в C++. Именно с вызова этого метода JVM начинает выполнение программы.
- **`System.out.println(...)`** — вывод текста на экран с переносом строки в конце.

### 3. Разбор модификаторов метода main

| Часть | Значение |
|---|---|
| `public` | метод доступен из любого места программы (JVM должна иметь доступ к нему извне класса) |
| `static` | метод принадлежит самому классу, а не конкретному объекту — можно вызвать без создания экземпляра класса |
| `void` | метод не возвращает значения |
| `String[] args` | массив строковых аргументов командной строки, переданных при запуске программы |

Эта сигнатура метода `main` строго фиксирована — JVM ожидает найти метод именно с такими модификаторами для запуска программы.

### 4. Вывод текста: System.out.println и System.out.print

```java
System.out.println("Строка с переносом");   // добавляет перенос строки в конце
System.out.print("Без переноса ");             // не добавляет перенос строки
System.out.print("продолжение той же строки");
```

### 5. Компиляция и запуск (повторение процесса)

```bash
javac Main.java   # создаёт Main.class
java Main            # запускает программу
```

### 6. Обязательность фигурных скобок и точки с запятой

Как и в C++, в Java блоки кода обозначаются фигурными скобками `{}`, а каждая инструкция обязательно завершается точкой с запятой `;`.

### 7. Комментарии

```java
// Однострочный комментарий

/*
Многострочный
комментарий
*/

/**
 * Javadoc-комментарий - специальный формат для документирования кода,
 * из которого можно автоматически генерировать документацию
 */
```

Javadoc-комментарии (начинающиеся с `/**`) — особенность экосистемы Java, широко используемая для документирования классов и методов в реальных проектах.

### Резюме урока

- Каждый публичный класс в Java должен находиться в файле с точно таким же именем, как и сам класс.
- Метод `main` со строго определённой сигнатурой (`public static void main(String[] args)`) — обязательная точка входа в программу.
- `System.out.println()` выводит текст с переносом строки, `System.out.print()` — без переноса.

## Урок 1.4. Структура Java-проекта
**Время: 30 минут**

### Цель урока
Понять принципы организации Java-проектов: пакеты, классы, файловую структуру.

### 1. Что такое пакет (package)

Пакет в Java — механизм организации связанных классов в логические группы, аналогичный папкам в файловой системе (и фактически соответствующий структуре папок проекта):

```java
package com.codeway.myapp;

public class Main {
    public static void main(String[] args) {
        System.out.println("Программа в пакете");
    }
}
```

Объявление `package` должно быть первой непустой строкой файла (кроме комментариев).

### 2. Соответствие пакетов и структуры папок

```
src/
└── com/
    └── codeway/
        └── myapp/
            └── Main.java   # содержит package com.codeway.myapp;
```

Имя пакета в объявлении `package` должно строго соответствовать пути к файлу относительно корневой папки исходного кода (`src`).

### 3. Соглашение об именовании пакетов

Общепринятое соглашение — использовать обратный домен организации в качестве базового имени пакета (например, `com.codeway.myapp`) для обеспечения уникальности имён пакетов в разных проектах и организациях по всему миру.

### 4. Импорт классов из других пакетов

```java
import java.util.Scanner;         // импорт конкретного класса
import java.util.*;                  // импорт всех классов пакета (использовать с осторожностью)

public class Main {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        // ...
    }
}
```

### 5. Несколько классов в одном проекте

Хотя каждый **публичный** класс должен находиться в отдельном файле, соответствующем его имени, один файл может содержать один публичный класс и произвольное число непубличных (package-private) вспомогательных классов:

```java
// Main.java
public class Main {
    public static void main(String[] args) {
        Helper helper = new Helper();
        helper.doSomething();
    }
}

class Helper {   // не public, находится в том же файле Main.java
    void doSomething() {
        System.out.println("Вспомогательный класс работает");
    }
}
```

Тем не менее, распространённая и рекомендуемая практика — размещать каждый класс в собственном отдельном файле для лучшей читаемости и организации проекта.

### 6. Стандартная структура более крупного проекта

```
MyApp/
├── src/
│   └── com/codeway/myapp/
│       ├── Main.java
│       ├── model/
│       │   └── User.java
│       └── service/
│           └── UserService.java
└── out/
```

Такая структура с разделением на подпакеты (`model`, `service` и др.) отражающие роль классов — стандартная практика организации более крупных Java-приложений, особенно в контексте фреймворков вроде Spring.

### Резюме урока

- Пакеты в Java организуют классы в логические группы, соответствующие структуре папок проекта.
- Имя пакета традиционно строится на основе обратного домена организации для обеспечения уникальности.
- `import` подключает классы из других пакетов для использования в текущем файле.
