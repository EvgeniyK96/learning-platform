## Урок 2.1. Переменные и примитивные типы данных
**Время: 45 минут**

### Цель урока
Изучить примитивные типы данных Java и научиться объявлять переменные со статической типизацией.

### 1. Статическая типизация в Java

Как и C++, Java использует статическую типизацию — тип переменной указывается явно при объявлении и не может измениться в дальнейшем.

```java
int age = 20;
age = "текст";   // ошибка компиляции
```

### 2. Восемь примитивных типов данных

| Тип | Описание | Пример |
|---|---|---|
| `byte` | целое число, 8 бит | `byte b = 100;` |
| `short` | целое число, 16 бит | `short s = 1000;` |
| `int` | целое число, 32 бита (самый распространённый) | `int age = 20;` |
| `long` | целое число, 64 бита | `long bigNumber = 10000000000L;` |
| `float` | дробное число одинарной точности | `float price = 19.99f;` |
| `double` | дробное число двойной точности (по умолчанию для дробных) | `double pi = 3.14159;` |
| `char` | один символ (Unicode) | `char grade = 'A';` |
| `boolean` | логическое значение | `boolean isActive = true;` |

### 3. Суффиксы для литералов

```java
long bigNumber = 10000000000L;   // суффикс L обязателен для long-литералов, превышающих диапазон int
float price = 19.99f;               // суффикс f (или F) обязателен для float-литералов
```

Без суффикса `L` или `f` компилятор по умолчанию трактует целочисленный литерал как `int`, а дробный — как `double`, что может привести к ошибке компиляции при попытке присвоить его переменной другого типа без явного приведения.

### 4. Объявление переменных

```java
int age;              // объявление без инициализации
age = 20;                // присваивание отдельно

int height = 175;      // объявление с инициализацией (рекомендуется)
```

В отличие от C++, локальные переменные в Java не имеют «мусорных» значений при отсутствии инициализации — компилятор просто не позволит использовать неинициализированную локальную переменную, выдав ошибку компиляции.

### 5. Ссылочный тип String

Строки в Java, в отличие от восьми примитивных типов, являются ссылочным типом (объектом класса `String`), но благодаря особому синтаксическому удобству объявляются почти как примитивы:

```java
String name = "Алина";
String fullName = "Алина" + " " + "Иванова";   // конкатенация через +
System.out.println(name.length());                // 5
```

### 6. Диапазоны значений и переполнение

Каждый числовой тип имеет фиксированный диапазон допустимых значений (например, `int` — от примерно -2.1 до 2.1 миллиарда). Превышение этого диапазона приводит к переполнению (overflow) — значение «перескакивает» на противоположный край диапазона, а не выдаёт ошибку:

```java
int maxValue = Integer.MAX_VALUE;   // максимальное значение int
System.out.println(maxValue + 1);      // переполнение! Результат станет отрицательным
```

### 7. Именование переменных

Общепринятый стиль в Java — `camelCase` для переменных и методов (`userName`, `calculateTotal`), `PascalCase` для классов (`UserProfile`), заглавные буквы с подчёркиванием для констант (`MAX_SPEED`).

### Резюме урока

- Java использует статическую типизацию с восемью примитивными типами данных, каждый со своим фиксированным диапазоном значений и размером в памяти.
- Литералы `long` и `float` требуют явных суффиксов (`L` и `f` соответственно) при превышении диапазона типов по умолчанию.
- `String` — ссылочный тип (класс), а не примитив, но синтаксически используется почти как обычная переменная.

## Урок 2.2. Ввод и вывод: Scanner, System.out.println
**Время: 40 минут**

### Цель урока
Научиться получать данные от пользователя с помощью класса Scanner и выводить результат.

### 1. Класс Scanner

Для считывания пользовательского ввода в Java используется класс `Scanner` из пакета `java.util`:

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        System.out.print("Введите имя: ");
        String name = scanner.nextLine();
        
        System.out.println("Привет, " + name + "!");
    }
}
```

### 2. Основные методы Scanner

| Метод | Считывает |
|---|---|
| `nextLine()` | всю строку целиком (включая пробелы) |
| `next()` | одно слово (до пробела) |
| `nextInt()` | целое число |
| `nextDouble()` | дробное число |
| `nextBoolean()` | логическое значение |

```java
Scanner scanner = new Scanner(System.in);

System.out.print("Введите возраст: ");
int age = scanner.nextInt();

System.out.print("Введите рост: ");
double height = scanner.nextDouble();

System.out.println("Возраст: " + age + ", рост: " + height);
```

### 3. Особенность смешивания nextInt() и nextLine()

Аналогично особенности `cin >>` и `getline()` в C++ (Урок 2.2 курса C++), у Scanner есть похожая ловушка: `nextInt()` не считывает символ переноса строки после числа, из-за чего следующий вызов `nextLine()` может вернуть пустую строку:

```java
Scanner scanner = new Scanner(System.in);

System.out.print("Введите возраст: ");
int age = scanner.nextInt();

scanner.nextLine();   // "поглощаем" оставшийся символ переноса строки

System.out.print("Введите имя: ");
String name = scanner.nextLine();   // теперь работает корректно
```

### 4. Вывод данных

```java
System.out.println("Обычный текст");
System.out.println(42);
System.out.println(3.14);

int age = 20;
System.out.println("Возраст: " + age);   // конкатенация числа со строкой
```

### 5. Форматированный вывод: printf

Аналогично `printf` в других C-подобных языках, Java предоставляет метод `System.out.printf()` для более гибкого форматирования вывода:

```java
double price = 1999.5;
System.out.printf("Цена: %.2f тг%n", price);   // Цена: 1999.50 тг
```

`%.2f` — спецификатор форматирования дробного числа с двумя знаками после запятой, `%n` — переносимый на разные ОС символ переноса строки (предпочтительнее жёстко заданного `\n` для платформонезависимого кода).

### 6. Практический пример

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        System.out.print("Введите имя: ");
        String name = scanner.nextLine();
        
        System.out.print("Введите возраст: ");
        int age = scanner.nextInt();
        
        System.out.println("Привет, " + name + "! Через 5 лет тебе будет " + (age + 5) + " лет.");
    }
}
```

### Резюме урока

- Класс `Scanner` из `java.util` — стандартный способ получения пользовательского ввода в консольных Java-программах.
- Разные методы Scanner (`nextInt()`, `nextLine()`, `nextDouble()`) считывают разные типы данных, и их смешивание требует внимания к особенности с символом переноса строки.
- `printf()` с форматными спецификаторами (`%.2f` и др.) обеспечивает более гибкий контроль над выводом чисел, чем обычная конкатенация.

## Урок 2.3. Операторы: арифметические, сравнения, логические
**Время: 40 минут**

### Цель урока
Изучить основные группы операторов Java.

### 1. Арифметические операторы

```java
System.out.println(5 + 2);    // 7
System.out.println(5 - 2);    // 3
System.out.println(5 * 2);    // 10
System.out.println(5 / 2);    // 2 (целочисленное деление для int)
System.out.println(5 % 2);    // 1
System.out.println(5.0 / 2);  // 2.5 (если хотя бы один операнд double)
```

### 2. Операторы инкремента и декремента

```java
int count = 5;
count++;
count--;

int a = 5;
System.out.println(a++);   // выведет 5, затем увеличит a (постфиксная форма)
System.out.println(++a);   // сначала увеличит a, затем выведет (префиксная форма)
```

### 3. Операторы сравнения

```java
System.out.println(5 > 3);     // true
System.out.println(5 == 5);    // true
System.out.println(5 != 3);    // true
```

### 4. Важная особенность: сравнение объектов через ==

Для примитивных типов оператор `==` сравнивает значения напрямую и работает ожидаемым образом. Но для ссылочных типов (включая `String`) `==` сравнивает не содержимое, а то, ссылаются ли переменные на один и тот же объект в памяти:

```java
String a = new String("привет");
String b = new String("привет");

System.out.println(a == b);          // false! Это разные объекты в памяти
System.out.println(a.equals(b));     // true - .equals() сравнивает именно содержимое
```

Эта особенность — частый источник ошибок у новичков, переходящих в Java из других языков. Для сравнения содержимого строк и других объектов всегда следует использовать метод `.equals()`, а не `==`.

### 5. Логические операторы

```java
int age = 20;
boolean hasId = true;

System.out.println(age >= 18 && hasId);   // true
System.out.println(age < 18 || hasId);     // true
System.out.println(!hasId);                    // false
```

### 6. Операторы составного присваивания

```java
int x = 10;
x += 5;    // x = 15
x -= 3;    // x = 12
x *= 2;    // x = 24
x /= 4;    // x = 6
```

### Резюме урока

- Арифметические и логические операторы в Java синтаксически схожи с C++ и JavaScript.
- Оператор `==` для ссылочных типов (в том числе `String`) сравнивает ссылки на объекты, а не их содержимое.
- Для сравнения содержимого объектов, включая строки, необходимо использовать метод `.equals()`.

## Урок 2.4. Приведение типов (widening/narrowing)
**Время: 35 минут**

### Цель урока
Изучить механизмы неявного и явного приведения типов в Java.

### 1. Расширяющее преобразование (widening) — неявное

Java автоматически преобразует значение меньшего типа в больший без явного указания, если это преобразование не приводит к потере данных:

```java
int intValue = 100;
double doubleValue = intValue;   // неявное расширяющее преобразование: int → double

System.out.println(doubleValue);   // 100.0
```

Иерархия расширения (упрощённо): `byte → short → int → long → float → double`.

### 2. Сужающее преобразование (narrowing) — явное

Обратное преобразование (из большего типа в меньший) может привести к потере данных, поэтому требует явного указания через приведение типов:

```java
double doubleValue = 9.99;
int intValue = (int) doubleValue;   // явное сужающее преобразование

System.out.println(intValue);   // 9 (дробная часть отброшена, а не округлена)
```

### 3. Синтаксис приведения типов в Java

В отличие от C++, где рекомендуется `static_cast<тип>()`, в Java для явного приведения используется более простой C-style синтаксис `(тип)значение`:

```java
double price = 19.99;
int roundedDown = (int) price;   // 19
```

### 4. Проблема целочисленного деления (повторение знакомой темы)

```java
int a = 5;
int b = 2;
double result = a / b;    // ОШИБКА логики: сначала произойдёт целочисленное деление (2), и только потом приведение к double
System.out.println(result);   // 2.0, а не 2.5!

double correctResult = (double) a / b;   // правильно: сначала приводим к double, потом делим
System.out.println(correctResult);   // 2.5
```

### 5. Автоупаковка и распаковка (autoboxing/unboxing) — обзорно

Java предоставляет так называемые классы-обёртки (wrapper classes) для каждого примитивного типа (`Integer` для `int`, `Double` для `double` и т.д.), необходимые, например, при работе с коллекциями (Модуль 7), которые могут хранить только объекты, а не примитивы напрямую:

```java
int primitiveInt = 42;
Integer wrapperInt = primitiveInt;   // автоупаковка (autoboxing): int → Integer

int backToPrimitive = wrapperInt;      // автораспаковка (unboxing): Integer → int
```

Java выполняет эти преобразования автоматически начиная с Java 5, что значительно упростило работу с коллекциями по сравнению с более ранними версиями языка.

### 6. Практический пример

```java
Scanner scanner = new Scanner(System.in);
System.out.print("Введите два целых числа: ");
int a = scanner.nextInt();
int b = scanner.nextInt();

double average = (a + b) / 2.0;   // деление на 2.0 (double) гарантирует дробный результат
System.out.println("Среднее: " + average);
```

### Резюме урока

- Расширяющее преобразование (из меньшего типа в больший) в Java происходит неявно, сужающее — требует явного приведения через `(тип)значение`.
- Деление `int / int` в Java, как и в C++, всегда даёт целочисленный результат — важно явно приводить хотя бы один операнд к дробному типу для получения точного результата.
- Классы-обёртки (`Integer`, `Double` и др.) автоматически преобразуются в примитивы и обратно через механизм автоупаковки/распаковки.

## Урок 2.5. Обёрточные классы (Wrapper classes) и константы
**Время: 40 минут**

### Цель урока
Углубить понимание обёрточных классов и научиться объявлять константы в Java.

### 1. Полный список обёрточных классов

| Примитив | Класс-обёртка |
|---|---|
| `byte` | `Byte` |
| `short` | `Short` |
| `int` | `Integer` |
| `long` | `Long` |
| `float` | `Float` |
| `double` | `Double` |
| `char` | `Character` |
| `boolean` | `Boolean` |

### 2. Зачем нужны обёрточные классы

- Позволяют использовать примитивные значения там, где Java требует именно объект (например, в коллекциях, Модуль 7).
- Предоставляют полезные статические методы и константы (например, `Integer.parseInt()`, `Integer.MAX_VALUE`).
- Позволяют представить «отсутствие значения» через `null`, что невозможно для обычных примитивов.

### 3. Полезные методы обёрточных классов

```java
String numberStr = "42";
int number = Integer.parseInt(numberStr);   // преобразование строки в int

String doubleStr = "3.14";
double doubleValue = Double.parseDouble(doubleStr);

System.out.println(Integer.MAX_VALUE);   // максимальное значение int
System.out.println(Integer.MIN_VALUE);   // минимальное значение int

System.out.println(Integer.toString(42));   // преобразование int в строку
```

### 4. Осторожность с null и автораспаковкой

```java
Integer wrapperValue = null;
int primitiveValue = wrapperValue;   // NullPointerException! Нельзя распаковать null в примитив
```

Эта ситуация — частый источник ошибок времени выполнения (`NullPointerException`) в реальных Java-программах, особенно при работе с данными, полученными из базы данных или внешнего API, где значение может отсутствовать.

### 5. Константы в Java: final

Ключевое слово для объявления констант в Java — `final` (в отличие от `const` в C++):

```java
final double PI = 3.14159;
final int MAX_ATTEMPTS = 3;

PI = 3.14;   // ошибка компиляции: нельзя изменить значение final-переменной
```

### 6. Константы уровня класса: static final

Для констант, общих для всех экземпляров класса (а не относящихся к конкретному объекту), используется сочетание `static final`:

```java
public class Circle {
    static final double PI = 3.14159;
    double radius;
    
    Circle(double r) {
        radius = r;
    }
    
    double area() {
        return PI * radius * radius;
    }
}
```

Соглашение об именовании — заглавные буквы с подчёркиванием (`MAX_SPEED`, `DEFAULT_TIMEOUT`), аналогично константам в других изученных языках курса.

### 7. Практический пример: безопасное преобразование строки в число

```java
String input = "42abc";

try {
    int number = Integer.parseInt(input);
    System.out.println(number);
} catch (NumberFormatException e) {
    System.out.println("Некорректный формат числа: " + e.getMessage());
}
```

(Обработку исключений подробно разберём в Модуле 8.)

### Резюме урока

- Обёрточные классы (`Integer`, `Double` и др.) предоставляют объектное представление примитивов вместе с полезными статическими методами и константами.
- Попытка автораспаковки `null`-значения обёрточного типа в примитив приводит к `NullPointerException`.
- Константы в Java объявляются через ключевое слово `final`, а константы уровня класса — через `static final`.
