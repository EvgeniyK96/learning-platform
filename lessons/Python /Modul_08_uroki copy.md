## Урок 8.1. Модули и пакеты, импорт кода
**Время: 35 минут**

### Цель урока
Понять, зачем нужно разделять код на несколько файлов, и научиться импортировать код из одного файла в другой.

### 1. Что такое модуль

Модуль в Python — это просто файл с расширением `.py`, содержащий код (функции, классы, переменные), который можно подключить (импортировать) в другой файл.

Допустим, у нас есть файл `operations.py`:

```python
# operations.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b
```

В другом файле мы можем использовать эти функции:

```python
# main.py
import operations

print(operations.add(5, 3))         # 8
print(operations.subtract(5, 3))    # 2
```

### 2. Разные способы импорта

```python
import operations                       # импорт всего модуля
from operations import add               # импорт конкретной функции
from operations import add, subtract     # импорт нескольких функций
from operations import add as sum_func   # импорт с переименованием
```

При использовании `from operations import add` функцию можно вызывать напрямую, без указания имени модуля:

```python
from operations import add
print(add(5, 3))   # 8
```

### 3. Что такое пакет

Пакет — это папка, содержащая несколько связанных модулей и специальный файл `__init__.py` (может быть пустым), который сообщает Python, что эту папку нужно рассматривать как пакет.

```
my_package/
├── __init__.py
├── math_operations.py
└── string_operations.py
```

```python
from my_package import math_operations
```

### 4. Импорт встроенных модулей Python

Помимо собственных файлов, можно импортировать модули стандартной библиотеки Python, которые устанавливаются вместе с самим Python:

```python
import math
print(math.sqrt(16))    # 4.0
print(math.pi)            # 3.141592653589793
```

### 5. Условие if __name__ == "__main__"

Часто в файлах-модулях можно встретить такую конструкцию:

```python
def add(a, b):
    return a + b

if __name__ == "__main__":
    print(add(2, 3))    # выполнится, только если файл запущен напрямую
```

Это позволяет писать в модуле тестовый код, который выполнится только при прямом запуске файла, но не при его импорте в другой файл.

### 6. Практическая рекомендация по структуре проекта

По мере роста программы стоит выносить логически связанный код в отдельные файлы — например, все функции для работы с файлами в `file_utils.py`, а функции калькулятора в `calculator.py`. Это делает проект понятнее и облегчает командную разработку.

### Резюме урока

- Модуль — это файл `.py`, код из которого можно подключить в другой файл через `import`.
- Пакет — это папка с несколькими модулями и файлом `__init__.py`.
- Конструкция `if __name__ == "__main__":` позволяет отделить код для тестирования модуля от кода, предназначенного для импорта.

## Урок 8.2. Стандартная библиотека (os, sys, datetime, random)
**Время: 45 минут**

### Цель урока
Познакомиться с наиболее полезными модулями стандартной библиотеки Python, которые не требуют отдельной установки.

### 1. Что такое стандартная библиотека

Стандартная библиотека — это набор модулей, которые устанавливаются вместе с самим Python и доступны сразу после установки без дополнительных команд. Это одно из главных преимуществ Python — «батарейки уже включены» (batteries included).

### 2. Модуль os — работа с операционной системой

```python
import os

print(os.getcwd())              # текущая рабочая папка
print(os.listdir("."))          # список файлов в текущей папке
os.mkdir("new_folder")          # создать новую папку
print(os.path.exists("main.py"))  # проверка существования файла
```

### 3. Модуль sys — взаимодействие с интерпретатором

```python
import sys

print(sys.version)     # версия установленного Python
print(sys.argv)         # аргументы, переданные при запуске скрипта из терминала
```

### 4. Модуль datetime — работа с датой и временем

```python
from datetime import datetime, timedelta

now = datetime.now()
print(now)                             # текущая дата и время
print(now.strftime("%d.%m.%Y"))     # форматированный вывод: 12.07.2026

tomorrow = now + timedelta(days=1)
print(tomorrow.strftime("%d.%m.%Y"))
```

### 5. Модуль random — генерация случайных значений

```python
import random

print(random.randint(1, 100))          # случайное целое число от 1 до 100
print(random.choice(["орёл", "решка"]))  # случайный элемент списка
print(random.random())                    # случайное число от 0 до 1

numbers = [1, 2, 3, 4, 5]
random.shuffle(numbers)                    # перемешивает список "на месте"
print(numbers)
```

### 6. Практический пример: генератор пароля

```python
import random
import string

def generate_password(length=10):
    characters = string.ascii_letters + string.digits
    password = "".join(random.choice(characters) for _ in range(length))
    return password

print(generate_password())
print(generate_password(16))
```

### Резюме урока

- Стандартная библиотека Python содержит готовые модули на все случаи жизни — от работы с файлами до генерации случайных чисел.
- `os` и `sys` — для взаимодействия с операционной системой и интерпретатором.
- `datetime` — для работы с датами, `random` — для генерации случайных значений.

## Урок 8.3. Установка сторонних библиотек через pip
**Время: 30 минут**

### Цель урока
Научиться устанавливать, обновлять и удалять сторонние библиотеки с помощью менеджера пакетов pip.

### 1. Что такое pip

`pip` — это стандартный менеджер пакетов Python, который позволяет устанавливать библиотеки, написанные сторонними разработчиками и опубликованные в репозитории PyPI (Python Package Index). Pip устанавливается автоматически вместе с Python.

### 2. Проверка версии pip

```bash
pip --version
```

### 3. Установка библиотеки

```bash
pip install requests
```

Эта команда скачает и установит библиотеку `requests` вместе со всеми её зависимостями.

### 4. Установка конкретной версии библиотеки

```bash
pip install requests==2.31.0
```

### 5. Просмотр установленных библиотек

```bash
pip list                    # список всех установленных библиотек
pip show requests           # подробная информация о конкретной библиотеке
```

### 6. Обновление и удаление библиотек

```bash
pip install --upgrade requests    # обновить до последней версии
pip uninstall requests             # удалить библиотеку
```

### 7. Файл requirements.txt

В реальных проектах принято сохранять список всех используемых библиотек в файл `requirements.txt`, чтобы другой разработчик (или ты сам на другом компьютере) мог быстро установить все зависимости одной командой:

```bash
pip freeze > requirements.txt      # сохранить текущие зависимости в файл
pip install -r requirements.txt    # установить все зависимости из файла
```

### Резюме урока

- `pip` — стандартный инструмент для установки сторонних библиотек Python.
- Основные команды: `pip install`, `pip uninstall`, `pip list`, `pip show`.
- Файл `requirements.txt` позволяет зафиксировать и быстро воспроизвести набор зависимостей проекта.

## Урок 8.4. Виртуальное окружение (venv)
**Время: 30 минут**

### Цель урока
Понять, зачем нужны виртуальные окружения, и научиться создавать и использовать их для изоляции зависимостей проекта.

### 1. Проблема, которую решает виртуальное окружение

Если устанавливать все библиотеки глобально на компьютер, со временем возникает риск конфликта версий: один проект требует `requests==2.20`, а другой — `requests==2.31`. Виртуальное окружение решает эту проблему, создавая изолированное пространство для каждого проекта со своим набором библиотек.

### 2. Создание виртуального окружения

Python включает встроенный модуль `venv` для создания виртуальных окружений:

```bash
python -m venv venv
```

Эта команда создаст папку `venv` внутри проекта, содержащую отдельную копию интерпретатора Python и место для установки библиотек.

### 3. Активация виртуального окружения

**Windows:**
```bash
venv\Scripts\activate
```

**macOS / Linux:**
```bash
source venv/bin/activate
```

После активации в начале строки терминала появится `(venv)` — это означает, что окружение активно, и все последующие команды `pip install` будут устанавливать библиотеки именно в это изолированное окружение, а не глобально.

### 4. Установка библиотек внутри виртуального окружения

```bash
(venv) pip install requests
```

### 5. Деактивация окружения

```bash
deactivate
```

### 6. Рекомендуемая практика

Для каждого нового Python-проекта рекомендуется создавать отдельное виртуальное окружение. Папку `venv` обычно не публикуют вместе с кодом проекта (например, в Git) — вместо неё используют файл `requirements.txt`, из которого окружение можно легко воссоздать на любом компьютере.

### Резюме урока

- Виртуальное окружение изолирует зависимости одного проекта от других, предотвращая конфликты версий библиотек.
- Создаётся командой `python -m venv venv`, активируется отдельной командой в зависимости от операционной системы.
- Хорошая практика — создавать отдельное окружение для каждого проекта.

## Урок 8.5. Введение в работу с API (библиотека requests)
**Время: 50 минут**

### Цель урока
Понять, что такое API, и научиться получать данные с внешних сервисов с помощью библиотеки requests.

### 1. Что такое API

API (Application Programming Interface) — это способ, которым разные программы могут «общаться» друг с другом. Веб-API чаще всего работает через HTTP-запросы: программа отправляет запрос на определённый адрес (URL), а сервер возвращает данные, обычно в формате JSON.

### 2. Установка библиотеки requests

```bash
pip install requests
```

### 3. Простой GET-запрос

GET-запрос используется для получения данных с сервера:

```python
import requests

response = requests.get("https://api.example.com/data")
print(response.status_code)    # код ответа сервера, 200 означает "успешно"
print(response.text)              # текст ответа
```

### 4. Работа с JSON-ответом

Большинство современных API возвращают данные в формате JSON, который библиотека requests умеет автоматически преобразовывать в словарь Python:

```python
import requests

response = requests.get("https://api.example.com/users/1")

if response.status_code == 200:
    data = response.json()
    print(data["name"])
else:
    print("Ошибка при получении данных:", response.status_code)
```

### 5. Передача параметров в запросе

```python
import requests

params = {"city": "Almaty", "units": "metric"}
response = requests.get("https://api.example.com/weather", params=params)
data = response.json()
print(data)
```

### 6. Коды ответа сервера

| Код | Значение |
|---|---|
| 200 | успешно |
| 400 | некорректный запрос |
| 401 | требуется авторизация |
| 404 | ресурс не найден |
| 500 | ошибка на сервере |

### 7. Практический пример: получение курса валют

```python
import requests

response = requests.get("https://api.example.com/currency/latest")

if response.status_code == 200:
    data = response.json()
    print(f"USD/KZT: {data.get('USD', 'нет данных')}")
else:
    print("Не удалось получить данные о курсе валют")
```

> Обрати внимание: реальный адрес API и структура ответа зависят от конкретного сервиса — перед использованием любого API важно ознакомиться с его документацией.

### Резюме урока

- API позволяет программам обмениваться данными через HTTP-запросы.
- Библиотека `requests` — стандарт де-факто для работы с API в Python.
- Ответ API чаще всего приходит в формате JSON, который легко преобразуется в словарь методом `.json()`.

## Урок 8.6. Работа с базами данных SQLite
**Время: 55 минут**

### Цель урока
Научиться создавать базу данных SQLite, а также добавлять, изменять, удалять и выбирать данные из таблиц с помощью SQL-запросов из Python.

### 1. Что такое SQLite

SQLite — это лёгкая встроенная база данных, которая хранится в одном файле и не требует установки отдельного сервера. Модуль для работы с SQLite (`sqlite3`) входит в стандартную библиотеку Python.

### 2. Подключение к базе данных

```python
import sqlite3

connection = sqlite3.connect("school.db")   # создаст файл, если он не существует
cursor = connection.cursor()
```

### 3. Создание таблицы

```python
cursor.execute("""
CREATE TABLE IF NOT EXISTS students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    age INTEGER,
    course TEXT
)
""")
connection.commit()
```

### 4. Добавление данных

```python
cursor.execute(
    "INSERT INTO students (name, age, course) VALUES (?, ?, ?)",
    ("Алина", 20, "Python с нуля")
)
connection.commit()
```

> Важно: значения передаются через параметр `(?, ?, ?)`, а не через f-строку — это защищает от SQL-инъекций и является стандартной практикой безопасности.

### 5. Выборка данных

```python
cursor.execute("SELECT * FROM students")
rows = cursor.fetchall()

for row in rows:
    print(row)
```

Можно также выбирать данные с условием:

```python
cursor.execute("SELECT * FROM students WHERE age > ?", (19,))
rows = cursor.fetchall()
```

### 6. Обновление и удаление данных

```python
cursor.execute("UPDATE students SET age = ? WHERE name = ?", (21, "Алина"))
connection.commit()

cursor.execute("DELETE FROM students WHERE name = ?", ("Алина",))
connection.commit()
```

### 7. Закрытие соединения

```python
connection.close()
```

Как и с файлами, рекомендуется использовать конструкцию `with` (в актуальных версиях `sqlite3` она автоматически управляет транзакцией, но не закрывает соединение — его по-прежнему стоит закрывать явно или через отдельный контекстный менеджер).

### 8. Практический пример: полный цикл работы с базой данных

```python
import sqlite3

connection = sqlite3.connect("school.db")
cursor = connection.cursor()

cursor.execute("""
CREATE TABLE IF NOT EXISTS students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    age INTEGER
)
""")

students = [("Алина", 20), ("Данияр", 19), ("Аружан", 22)]
cursor.executemany("INSERT INTO students (name, age) VALUES (?, ?)", students)
connection.commit()

cursor.execute("SELECT * FROM students ORDER BY age")
for row in cursor.fetchall():
    print(row)

connection.close()
```

### Резюме урока

- SQLite — встроенная база данных, идеальная для обучения и небольших проектов, не требующая отдельной установки сервера.
- Модуль `sqlite3` входит в стандартную библиотеку Python.
- Значения в SQL-запросы всегда следует передавать через параметры (`?`), а не через конкатенацию строк, — это защищает от SQL-инъекций.
