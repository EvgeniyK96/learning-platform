## Урок 2.1. Переменные и типы данных
**Время: 45 минут**

### Цель урока
Понять, что такое переменная, как её создавать, какие базовые типы данных существуют в Python и как определить тип любого значения.

### 1. Что такое переменная

Переменная — это «именованная коробка» в памяти компьютера, в которую можно положить значение и обращаться к нему по имени.

```python
name = "Алина"
age = 19
city = "Алматы"
```

Python сам определяет тип значения в момент присваивания — это называется **динамической типизацией**.

### 2. Правила именования переменных

- Имя может состоять из букв, цифр и знака подчёркивания `_`.
- Имя не может начинаться с цифры.
- Python чувствителен к регистру: `age` и `Age` — разные переменные.
- Нельзя использовать зарезервированные слова (`if`, `for`, `class` и т.д.).
- По PEP 8 имена переменных пишут в стиле `snake_case`: `user_name`, а не `userName`.

```python
user_age = 20      # хорошо
2age = 20           # ошибка
```

### 3. Основные типы данных

| Тип | В Python | Пример | Описание |
|-----|-----------|--------|----------|
| Целое число | `int` | `10`, `-5` | Числа без дробной части |
| Дробное число | `float` | `3.14` | Числа с плавающей точкой |
| Строка | `str` | `"Привет"` | Текстовые данные |
| Логический тип | `bool` | `True`, `False` | Истина/ложь |

Проверить тип переменной можно функцией `type()`:

```python
name = "Алина"
age = 19
print(type(name))   # <class 'str'>
print(type(age))    # <class 'int'>
```

### 4. Изменение значения и типа переменной

```python
x = 5
x = "теперь я строка"   # тип x изменился с int на str
```

### 5. Множественное присваивание

```python
a, b, c = 1, 2, 3
x = y = z = 0
```

### 6. Типичные ошибки новичков

- Использование переменной до её создания → `NameError`.
- Опечатки в имени переменной создают новую переменную вместо обращения к старой.
- Путаница строки и числа: `"5" + "3"` даст `"53"`, а не `8`.

```python
print("5" + "3")   # 53
print(5 + 3)        # 8
```

### Резюме урока

- Переменная — это имя, привязанное к значению в памяти.
- Тип переменной определяется автоматически при присваивании.
- Основные типы: `int`, `float`, `str`, `bool`.
- Проверить тип можно функцией `type()`.

## Урок 2.2. Ввод и вывод данных (input, print)
**Время: 40 минут**

### Цель урока
Научиться получать данные от пользователя через `input()` и грамотно выводить результат с помощью `print()`.

### 1. Функция input()

`input()` приостанавливает выполнение программы и ждёт, пока пользователь введёт текст и нажмёт Enter. Введённое значение **всегда возвращается в виде строки** (`str`), даже если пользователь ввёл число.

```python
name = input("Как тебя зовут? ")
print("Привет,", name)
```

### 2. Преобразование ввода в число

Так как `input()` всегда возвращает строку, для математических операций её нужно преобразовать:

```python
age = input("Сколько тебе лет? ")
age = int(age)          # преобразуем строку в целое число
print("Через 5 лет тебе будет", age + 5)
```

Это можно записать компактнее в одну строку:

```python
age = int(input("Сколько тебе лет? "))
```

### 3. Дополнительные параметры print()

- `sep` — разделитель между значениями (по умолчанию — пробел).
- `end` — что печатать в конце строки (по умолчанию — перенос строки).

```python
print("яблоко", "банан", "груша", sep=", ")
# яблоко, банан, груша

print("Загрузка", end="")
print("...")
# Загрузка...
```

### 4. Форматирование вывода с помощью f-строк

f-строки — самый удобный способ вставлять значения переменных внутрь текста:

```python
name = "Данияр"
age = 20
print(f"Меня зовут {name}, мне {age} лет")
```

Внутри фигурных скобок можно писать не только переменные, но и выражения:

```python
print(f"Через год мне будет {age + 1} лет")
```

### 5. Практический пример: программа-приветствие

```python
name = input("Введите имя: ")
age = int(input("Введите возраст: "))
print(f"Привет, {name}! Через 5 лет тебе будет {age + 5} лет.")
```

### 6. Типичные ошибки новичков

- Забывают преобразовать результат `input()` в число перед арифметическими операциями → `TypeError`.
- Пытаются сложить число и строку напрямую: `"Мне " + age + " лет"` вызовет ошибку, если `age` — число. Нужно использовать f-строку или `str(age)`.

### Резюме урока

- `input()` всегда возвращает строку — для чисел её нужно преобразовывать через `int()` или `float()`.
- `print()` поддерживает параметры `sep` и `end` для управления форматом вывода.
- f-строки (`f"..."`) — самый удобный способ вставки переменных в текст.

## Урок 2.3. Операторы: арифметические, сравнения, логические
**Время: 45 минут**

### Цель урока
Изучить основные группы операторов Python и научиться комбинировать условия с помощью логических операторов.

### 1. Арифметические операторы

| Оператор | Значение | Пример | Результат |
|---|---|---|---|
| `+` | сложение | `5 + 2` | `7` |
| `-` | вычитание | `5 - 2` | `3` |
| `*` | умножение | `5 * 2` | `10` |
| `/` | деление (всегда float) | `5 / 2` | `2.5` |
| `//` | целочисленное деление | `5 // 2` | `2` |
| `%` | остаток от деления | `5 % 2` | `1` |
| `**` | возведение в степень | `5 ** 2` | `25` |

### 2. Операторы сравнения

Результатом операторов сравнения всегда является `bool` (`True` или `False`):

| Оператор | Значение |
|---|---|
| `==` | равно |
| `!=` | не равно |
| `>` | больше |
| `<` | меньше |
| `>=` | больше или равно |
| `<=` | меньше или равно |

```python
print(5 == 5)    # True
print(5 != 3)    # True
print(5 > 10)    # False
```

> Частая ошибка новичков — путать `=` (присваивание) и `==` (сравнение).

### 3. Логические операторы

| Оператор | Значение |
|---|---|
| `and` | истина, если оба условия истинны |
| `or` | истина, если хотя бы одно условие истинно |
| `not` | инвертирует значение |

```python
age = 20
has_id = True

print(age >= 18 and has_id)   # True
print(age < 18 or has_id)      # True
print(not has_id)               # False
```

### 4. Порядок выполнения операторов (приоритет)

Как и в математике, у операторов есть приоритет: сначала `**`, затем `*`, `/`, `//`, `%`, затем `+`, `-`, затем операторы сравнения, и в конце — логические операторы. Скобки `()` позволяют явно задать нужный порядок:

```python
result = (2 + 3) * 4      # 20, а не 14
```

### 5. Оператор принадлежности in

Отдельно стоит запомнить оператор `in`, который проверяет, содержится ли значение в последовательности (строке, списке и т.д.):

```python
print("a" in "banana")      # True
print(10 in [1, 2, 3])       # False
```

### Резюме урока

- Арифметические операторы выполняют вычисления, операторы сравнения возвращают `bool`.
- `and`, `or`, `not` позволяют комбинировать несколько условий.
- Важно не путать `=` (присваивание) и `==` (сравнение).
- Приоритет операторов можно явно менять с помощью скобок.

## Урок 2.4. Приведение типов данных
**Время: 35 минут**

### Цель урока
Научиться осознанно преобразовывать данные из одного типа в другой и понимать, какие преобразования допустимы, а какие вызывают ошибку.

### 1. Зачем нужно приведение типов

Иногда данные приходят в «неудобном» типе — например, число вводится пользователем как строка. Чтобы выполнить над ним математические операции, тип нужно изменить. Этот процесс называется **приведением типов** (type casting).

### 2. Основные функции преобразования

| Функция | Преобразует в | Пример |
|---|---|---|
| `int()` | целое число | `int("10")` → `10` |
| `float()` | дробное число | `float("3.14")` → `3.14` |
| `str()` | строку | `str(10)` → `"10"` |
| `bool()` | логическое значение | `bool(0)` → `False` |

```python
age_str = "25"
age_int = int(age_str)
print(age_int + 5)   # 30
```

### 3. Какие преобразования вызывают ошибку

Не любую строку можно превратить в число:

```python
int("двадцать")   # ValueError: invalid literal for int()
```

Такие ситуации в реальных программах обычно обрабатывают с помощью конструкции `try/except`, с которой мы познакомимся в Модуле 6.

### 4. Особенности bool()

Практически любое значение можно преобразовать в `bool`. При этом действует правило: «пустое» или нулевое значение → `False`, всё остальное → `True`.

```python
print(bool(0))        # False
print(bool(1))        # True
print(bool(""))       # False (пустая строка)
print(bool("текст"))  # True
print(bool([]))         # False (пустой список)
```

### 5. Неявное приведение типов

В некоторых случаях Python сам преобразует типы — например, при сложении `int` и `float` результат автоматически станет `float`:

```python
result = 5 + 2.5
print(result)          # 7.5
print(type(result))   # <class 'float'>
```

Это называется **неявным приведением типов** и происходит автоматически по внутренним правилам Python.

### 6. Практический пример

```python
first_num = float(input("Введите первое число: "))
second_num = float(input("Введите второе число: "))

print(f"Сумма: {first_num + second_num}")
print(f"Среднее: {(first_num + second_num) / 2}")
```

### Резюме урока

- Явное приведение типов выполняется функциями `int()`, `float()`, `str()`, `bool()`.
- Не всякая строка может быть преобразована в число — Python выбросит ошибку `ValueError`.
- При смешивании `int` и `float` в операциях Python автоматически приводит результат к `float`.

## Урок 2.5. Комментарии и стиль кода (PEP 8)
**Время: 30 минут**

### Цель урока
Научиться писать понятные комментарии и придерживаться общепринятого стиля оформления кода на Python.

### 1. Что такое комментарии и зачем они нужны

Комментарий — это часть кода, которую интерпретатор Python игнорирует при выполнении программы. Комментарии нужны для того, чтобы объяснить логику кода — себе в будущем или другим разработчикам.

```python
# Это однострочный комментарий
age = 18  # Возраст пользователя, полученный от input()
```

### 2. Многострочные комментарии

Формально в Python нет отдельного синтаксиса для многострочных комментариев, но для этого часто используют тройные кавычки (это технически многострочная строка, которая не присваивается переменной):

```python
"""
Эта программа рассчитывает
среднее значение списка чисел.
"""
```

### 3. Хорошие и плохие комментарии

**Плохой комментарий** дублирует то, что и так понятно из кода:

```python
x = x + 1   # прибавили 1 к x
```

**Хороший комментарий** объясняет, зачем это делается:

```python
x = x + 1   # компенсируем нумерацию с нуля при выводе для пользователя
```

### 4. Что такое PEP 8

**PEP 8** — это официальный документ-руководство по стилю написания кода на Python. Следование PEP 8 делает код более читаемым и понятным другим разработчикам.

Основные правила PEP 8:

- Отступ — 4 пробела (не табуляция).
- Имена переменных и функций — в стиле `snake_case`.
- Имена классов — в стиле `PascalCase` (например, `UserProfile`).
- Имена констант — заглавными буквами: `MAX_SPEED = 120`.
- Не более одной инструкции на строку.
- Пробелы вокруг операторов: `x = 5`, а не `x=5`.
- Пустая строка между логическими блоками кода.
- Длина строки — не более 79–99 символов (в зависимости от команды/проекта).

### 5. Пример «до» и «после» по PEP 8

```python
# До
def calc(a,b):
    result=a+b
    return result

# После
def calculate_sum(a, b):
    result = a + b
    return result
```

### 6. Инструменты для проверки стиля

В реальной разработке используются линтеры (например, `flake8`, `pylint`) и форматтеры (`black`), которые автоматически проверяют или приводят код к стандарту PEP 8. В VS Code эти инструменты можно подключить как расширения.

### Резюме урока

- Комментарии объясняют логику кода и не влияют на его выполнение.
- Хороший комментарий отвечает на вопрос «зачем», а не «что» — это и так видно из кода.
- PEP 8 — стандарт оформления кода на Python, которого стоит придерживаться с первых уроков.
- Аккуратный, единообразный код гораздо проще читать и поддерживать — как себе, так и другим разработчикам.
